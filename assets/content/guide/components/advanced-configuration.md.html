edit

ChangeDetectionStrategy

@Component 装饰器接受一个 changeDetection 选项，该选项控制组件的变更检测模式。有两种变更检测模式选项。

ChangeDetectionStrategy.Default 如其名，是默认策略。在此模式下，每当应用范围内可能发生任何活动时，Angular 都会检查组件的 DOM 是否需要更新。触发此检查的活动包括用户交互、网络响应、计时器等。

ChangeDetectionStrategy.OnPush 是一种可选模式，可减少 Angular 需要执行的检查量。在此模式下，框架仅在以下情况检查组件的 DOM 是否需要更新：

  • 组件输入由于模板中的绑定而发生更改，或者
  • 此组件中的事件监听器运行
  • 组件被显式标记为待检查，通过 ChangeDetectorRef.markForCheck 或包装它的东西，例如 AsyncPipe。

此外，当检查 OnPush 组件时，Angular 还会检查其所有祖先组件，并在应用树中向上遍历。

PreserveWhitespaces

默认情况下，Angular 会删除和折叠模板中多余的空格，最常见的是来自换行符和缩进的空格。你可以通过在组件的元数据中显式地将 preserveWhitespaces 设置为 true 来更改此设置。

自定义元素 schema

默认情况下，Angular 在遇到未知 HTML 元素时会抛出错误。您可以通过在组件元数据中的 schemas 属性中包含 CUSTOM_ELEMENTS_SCHEMA 来为组件禁用此行为。

import {Component, CUSTOM_ELEMENTS_SCHEMA} from '@angular/core';@Component({  ...,  schemas: [CUSTOM_ELEMENTS_SCHEMA],  template: '<some-unknown-component></some-unknown-component>'})export class ComponentWithCustomElements { }

目前，Angular 不支持任何其他 schema。
