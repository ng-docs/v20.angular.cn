了解你正在测试多少代码

edit

Angular CLI 可以运行单元测试并创建代码覆盖率报告。 代码覆盖率报告会显示你的代码库中可能未被单元测试充分测试到的任何部分。

要在你的项目根目录下生成覆盖率报告，请运行以下命令。

ng test --no-watch --code-coverage

当测试完成时，该命令会在项目中创建一个新的 /coverage 目录。 打开 index.html 文件以查看包含你的源代码和代码覆盖率值的报告。

如果你想在每次测试时都创建代码覆盖率报告，请在 Angular CLI 配置文件 angular.json 中设置以下选项：

"test": {  "options": {    "codeCoverage": true  }}

代码覆盖率强制执行

代码覆盖率百分比让你估计你的代码有多少被测试过。 如果你的团队决定了要进行单元测试的最小代码量，请使用 Angular CLI 强制执行此最小值。

例如，假设你希望代码库的代码覆盖率至少达到 80%。 要启用此功能，请打开 Karma 测试平台配置文件 karma.conf.js，并在 coverageReporter: 键中添加 check 属性。

coverageReporter: {  dir: require('path').join(__dirname, './coverage/<project-name>'),  subdir: '.',  reporters: [    { type: 'html' },    { type: 'text-summary' }  ],  check: {    global: {      statements: 80,      branches: 80,      functions: 80,      lines: 80    }  }}

提示： 请在测试指南中阅读更多关于创建和微调 Karma 配置的信息。

当在项目中运行单元测试时，check 属性会导致该工具强制执行至少 80% 的代码覆盖率。

请在 karma coverage 文档中阅读更多关于覆盖率配置选项的信息。
