使用自定义 ID 管理标记文本

edit

对于以下每种情况，Angular 提取器都会生成一个包含翻译单元条目的文件。

  • 组件模板中的每个 i18n 属性
  • 组件代码中每个带有 $localize 标记的消息字符串

如 含义如何控制文本提取和合并 中所述，Angular 会为每个翻译单元分配一个唯一的 ID。

以下范例展示了带有唯一 ID 的翻译单元。

当你更改可翻译文本时，提取器会为该翻译单元生成一个新的 ID。 在大多数情况下，源文本的更改也需要更改翻译。 因此，使用新的 ID 可以使文本更改与翻译保持同步。

但是，某些翻译系统对 ID 的格式或语法有特定要求。 为了满足此要求，可以使用自定义 ID 来标记文本。 大多数开发人员不需要使用自定义 ID。 如果你想使用独特的语法来传达额外的元数据，请使用自定义 ID。 额外的元数据可能包括文本出现位置的应用的库、组件或区域。

要在 i18n 属性或带有 $localize 标记的消息字符串中指定自定义 ID，请使用 @@ 前缀。 以下示例在标题元素中定义了 introductionHeader 自定义 ID。

app/app.component.html

<h1>Hello i18n!</h1><h1 i18n>Hello i18n!</h1><h1 i18n="An introduction header for this sample">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample">Hello i18n!</h1><h1 i18n="An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="@@introductionHeader">Hello i18n!</h1><img [src]="logo" title="Angular logo" alt="Angular logo"><h3 i18n="@@myId">Hello</h3><!-- ... --><p i18n="@@myId">Good bye</p>

以下范例为一个变量定义了 introductionHeader 自定义 ID。

variableText1 = $localize`:@@introductionHeader:Hello i18n!`;

当你指定自定义 ID 时，提取器会生成一个带有该自定义 ID 的翻译单元。

如果更改文本，提取器不会更改 ID。 因此，你不必采取额外的步骤来更新翻译。 使用自定义 ID 的缺点是，如果更改文本，你的翻译可能与新更改的源文本不同步。

将自定义 ID 与描述一起使用

将自定义 ID 与描述和含义结合使用，以进一步帮助翻译人员。

以下范例包含描述，后跟自定义 ID。

app/app.component.html

<h1>Hello i18n!</h1><h1 i18n>Hello i18n!</h1><h1 i18n="An introduction header for this sample">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample">Hello i18n!</h1><h1 i18n="An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="@@introductionHeader">Hello i18n!</h1><img [src]="logo" title="Angular logo" alt="Angular logo"><h3 i18n="@@myId">Hello</h3><!-- ... --><p i18n="@@myId">Good bye</p>

以下范例为一个变量定义了 introductionHeader 自定义 ID 和描述。

variableText2 = $localize`:An introduction header for this sample@@introductionHeader:Hello i18n!`;

以下范例添加了一个含义。

app/app.component.html

<h1>Hello i18n!</h1><h1 i18n>Hello i18n!</h1><h1 i18n="An introduction header for this sample">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample">Hello i18n!</h1><h1 i18n="An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="@@introductionHeader">Hello i18n!</h1><img [src]="logo" title="Angular logo" alt="Angular logo"><h3 i18n="@@myId">Hello</h3><!-- ... --><p i18n="@@myId">Good bye</p>

以下范例为一个变量定义了 introductionHeader 自定义 ID。

variableText3 = $localize`:site header|An introduction header for this sample@@introductionHeader:Hello i18n!`;

定义唯一的自定义 ID

请务必定义唯一的自定义 ID。 如果对两个不同的文本元素使用相同的 ID，则提取工具只会提取第一个，并且 Angular 会使用该翻译来代替这两个原始文本元素。

例如，在以下代码片段中，为两个不同的文本元素定义了相同的 myId 自定义 ID。

app/app.component.html

<h1>Hello i18n!</h1><h1 i18n>Hello i18n!</h1><h1 i18n="An introduction header for this sample">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample">Hello i18n!</h1><h1 i18n="An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="site header|An introduction header for this sample@@introductionHeader">Hello i18n!</h1><h1 i18n="@@introductionHeader">Hello i18n!</h1><img [src]="logo" title="Angular logo" alt="Angular logo"><h3 i18n="@@myId">Hello</h3><!-- ... --><p i18n="@@myId">Good bye</p>
