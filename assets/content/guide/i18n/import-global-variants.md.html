导入语言环境数据的全局变体

edit

当您使用 --localize 选项运行 ng build 命令时，Angular CLI 会自动包含语言环境数据。

ng build --localize

有帮助： Angular 的初始安装已包含美国英语 (en-US) 的语言环境数据。 当您将 --localize 选项与 ng build 命令一起使用时，Angular CLI 会自动包含语言环境数据并设置 LOCALE_ID 值。

npm 上的 @angular/common 包包含语言环境数据文件。 语言环境数据的全局变体在 @angular/common/locales/global 中可用。

法语的 import 范例

例如，你可以在引导启动应用的 main.ts 中导入法语 (fr) 的全局变体。

src/main.ts (导入语言环境)

import '@angular/common/locales/global/fr';import {provideProtractorTestingSupport} from '@angular/platform-browser';import {bootstrapApplication} from '@angular/platform-browser';import {AppComponent} from './app/app.component';bootstrapApplication(AppComponent, {  providers: [    provideProtractorTestingSupport(), // essential for e2e testing  ],});

提示： 在 NgModules 应用中，你需要在你的 app.module 中导入它。
