通过 ID 引用语言环境

edit

Angular 使用 Unicode 语言环境标识符（Unicode 语言环境 ID）来查找正确的语言环境数据，以便对文本字符串进行国际化。

Unicode 语言环境 ID

  • 语言环境 ID 符合Unicode Common Locale Data Repository (CLDR) 核心规范。 有关语言环境 ID 的更多信息，请参阅 [Unicode Language and Locale Identifiers][UnicodeCldrDevelopmentCoreSpecificationLocaleIDs]。

  • CLDR 和 Angular 使用BCP 47 标签作为语言环境 ID 的基础

语言环境 ID 指定了语言、国家以及用于进一步变体或细分的可选代码。 语言环境 ID 由语言标识符、连字符 (-) 和语言环境扩展名组成。

{language_id}-{locale_extension}

提示： 要准确翻译你的 Angular 项目，你必须确定要将哪些语言和语言环境作为国际化的目标。

许多国家/地区使用相同的语言，但在用法上有所不同。 差异包括语法、标点符号、货币格式、十进制数字、日期等等。

对于本指南中的示例，请使用以下语言和语言环境。

语言 语言环境 Unicode 语言环境 ID
英语 加拿大 en-CA
英语 美利坚合众国 en-US
法语 加拿大 fr-CA
法语 法国 fr-FR

Angular 代码库包含常用语言环境。

有关语言代码列表，请参阅 ISO 639-2。

设置源语言环境 ID

使用 Angular CLI 设置你正在编写组件模板和代码的源语言。

默认情况下，Angular 使用 en-US 作为项目的源语言环境。

要更改项目中用于构建的源语言环境，请完成以下操作。

  1. 打开 angular.json 工作区构建配置文件。
  2. 在 i18n 部分内添加或修改 sourceLocale 字段：
{  "projects": {    "your-project": {      "i18n": {        "sourceLocale": "ca"  // Use your desired locale code      }    }  }}
