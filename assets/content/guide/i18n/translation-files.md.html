使用翻译文件

edit

准备好组件以供翻译后，使用 extract-i18n Angular CLI 命令将组件中标记的文本提取到源语言文件中。

标记的文本包括用 i18n 标记的文本、用 i18n-attribute 标记的属性，以及用 $localize 标记的文本，具体描述请参见 为组件准备翻译。

要为你的项目创建和更新翻译文件，请完成以下步骤。

  1. 提取源语言文件。
    1. （可选）更改位置、格式和名称。
  2. 将源语言文件复制到 为每种语言创建翻译文件。
  3. 翻译每个翻译文件。
  4. 分别翻译复数和备用表达式。
    1. 翻译复数。
    2. 翻译替代表达式。
    3. 翻译嵌套表达式。

提取源语言文件

要提取源语言文件，请完成以下操作。

  1. 打开终端窗口。
  2. 切换到你的项目根目录。
  3. 运行以下 CLI 命令。
    ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

extract-i18n 命令会在你的项目根目录中创建一个名为 messages.xlf 的源语言文件。 有关 XML Localization Interchange File Format (XLIFF, version 1.2) 的更多信息，请参见 XLIFF。

使用以下 extract-i18n 命令选项来更改源语言文件的位置、格式和文件名。

命令选项 详情
--format 设置输出文件的格式
--out-file 设置输出文件的名称
--output-path 设置输出目录的路径

更改源语言文件位置

要在 src/locale 目录中创建文件，请将输出路径指定为一个选项。

extract-i18n --output-path 范例

以下范例将输出路径指定为一个选项。

ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

更改源语言文件格式

extract-i18n 命令以以下翻译格式创建文件。

翻译格式 详情 文件扩展名
ARB 应用资源捆绑包 .arb
JSON JavaScript 对象表示法 .json
XLIFF 1.2 XML 本地化交换文件格式，1.2 版 .xlf
XLIFF 2 XML 本地化交换文件格式，第 2 版 .xlf
XMB XML 消息捆绑包 .xmb (.xtb)

使用 --format 命令选项显式指定翻译格式。

提示： XMB 格式生成 .xmb 源语言文件，但使用 .xtb 翻译文件。

extract-i18n --format 范例

以下范例演示了几种翻译格式。

ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

更改源语言文件名

要更改提取工具生成的源语言文件的名称，请使用 --out-file 命令选项。

extract-i18n --out-file 范例

以下范例演示了如何命名输出文件。

ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

为每种语言创建翻译文件

要为语言环境或语言创建翻译文件，请完成以下操作。

  1. 提取源语言文件。

  2. 制作源语言文件的副本，以便为每种语言创建一个翻译文件。

  3. 重命名翻译文件以添加语言环境。

    messages.xlf --> messages.{locale}.xlf

  4. 在项目根目录下创建一个名为 locale 的新目录。

    src/locale

  5. 将翻译文件移动到新目录。

  6. 将翻译文件发送给你的翻译人员。

  7. 对于要添加到应用中的每种语言，重复上述步骤。

extract-i18n 法语范例

例如，要创建法语翻译文件，请完成以下操作。

  1. 运行 extract-i18n 命令。
  2. 制作 messages.xlf 源语言文件的副本。
  3. 将副本重命名为 messages.fr.xlf 以进行法语 (fr) 翻译。
  4. 将 fr 翻译文件移动到 src/locale 目录。
  5. 将 fr 翻译文件发送给翻译人员。

翻译每个翻译文件

除非你精通该语言并有时间编辑翻译，否则你很可能需要完成以下步骤。

  1. 将每个翻译文件发送给翻译人员。
  2. 翻译人员使用 XLIFF 文件编辑器来完成以下操作。
    1. 创建翻译。
    2. 编辑翻译。

翻译复数

根据每种语言的需要添加或删除复数情况。

有帮助：有关语言复数规则，请参阅 CLDR 复数规则。

minute plural 范例

要翻译 plural，请翻译 ICU 格式匹配值。

  • just now
  • one minute ago
  • <x id="INTERPOLATION" equiv-text="{{minutes}}"/> minutes ago
