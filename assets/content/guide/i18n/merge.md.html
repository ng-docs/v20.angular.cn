将翻译合并到应用中

edit

要将完成的翻译合并到你的项目中，请完成以下操作

  1. 使用 Angular CLI 来构建项目的可分发文件副本
  2. 使用 "localize" 选项将所有 i18n 消息替换为有效的翻译，并构建一个本地化变体应用。 变体应用是为单个语言环境翻译的应用的可分发文件副本。

合并翻译后，使用服务端语言检测或不同的子目录为应用的每个可分发副本启动服务器。

提示： 有关如何为应用的每个可分发副本启动服务器的更多信息，请参阅 部署多种语言环境。

对于应用的编译时翻译，构建过程使用预先编译 (AoT) 来生成一个小的、快速的、可立即运行的应用。

有帮助：有关构建过程的详细说明，请参阅构建和提供 Angular 应用。 构建过程适用于 .xlf 格式的翻译文件，或其他 Angular 可识别的格式，例如 .xtb。 有关 Angular 使用的翻译文件格式的更多信息，请参阅更改源语言文件格式

要为每个语言环境构建应用的可分发文件副本，请在项目的 angular.json 工作区构建配置文件中定义语言环境。

此方法通过消除为每个语言环境执行完整应用构建的需求来缩短构建过程。

要为每个语言环境生成应用变体，请在 angular.json 工作区构建配置文件中使用 "localize" 选项。 此外，要从命令行构建，请使用带有 --localize 选项的 build Angular CLI 命令。

有帮助：可选地，为自定义语言环境配置仅为一种语言环境应用特定的构建选项。

在构建配置中定义语言环境

在项目的 angular.json 工作区构建配置文件中使用 i18n 项目选项来为项目定义语言环境。

以下子选项标识源语言，并告知编译器在哪里可以找到项目的受支持翻译。

子选项 详细信息
sourceLocale 你在应用源码中使用的语言环境（默认为 en-US）
locales 语言环境标识符到翻译文件的映射表

为每个语言环境生成应用变体

要在构建配置中使用您的语言环境定义，请在 angular.json 工作区构建配置文件中使用 "localize" 选项，告知 CLI 为构建配置生成哪些语言环境。

  • 对于构建配置中先前定义的所有语言环境，将 "localize" 设置为 true。
  • 将 "localize" 设置为先前定义的语言环境标识符子集的数组，以仅构建这些语言环境版本。
  • 将 "localize" 设置为 false 以禁用本地化，并且不生成任何特定于语言环境的版本。

提示： 本地化组件模板需要预先编译 (AoT)。

如果你更改了此设置，请将 "aot" 设置为 true 以使用 AoT。

提示： 由于 i18n 的部署复杂性以及需要最大限度地缩短重建时间，开发服务器一次仅支持本地化单个语言环境。 如果你将 "localize" 选项设置为 true，定义了多个语言环境，并使用 ng serve；则会发生错误。 如果你想针对特定语言环境进行开发，请将 "localize" 选项设置为特定语言环境。 例如，对于法语 (fr)，请指定 "localize": ["fr"]。

CLI 加载并注册语言环境数据，将每个生成的版本放在特定于语言环境的目录中，以使其与其他语言环境版本分开，并将这些目录放在项目中配置的 outputPath 中。 对于每个应用变体，html 元素的 lang 属性都设置为语言环境。 CLI 还会通过将语言环境添加到配置的 baseHref 中来调整每个应用版本的 HTML base HREF。

将 "localize" 属性设置为共享配置，以便有效地为所有配置继承。 此外，设置该属性以覆盖其他配置。

从命令行构建

此外，在 production 配置和 ng build 命令一起使用 --localize 选项。 CLI 会构建构建配置中定义的所有语言环境。 如果在构建配置中设置了语言环境，这与将 "localize" 选项设置为 true 时类似。

有帮助：有关如何设置语言环境的更多信息，请参阅为每个语言环境生成应用变体。

ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

仅为一个语言环境应用特定的构建选项

要仅为一个语言环境应用特定的构建选项，请指定单个语言环境以创建自定义的特定于语言环境的配置。

重要：请仅使用单个语言环境的 Angular CLI 开发服务器（ng serve）。

报告缺失的翻译

当翻译缺失时，构建会成功，但会生成警告，例如 Missing translation for message "{translation_text}"。 要配置 Angular 编译器生成的警告级别，请指定以下级别之一。

警告级别 详细信息 输出
error 抛出错误并且构建失败 不适用
ignore 不执行任何操作 不适用
warning 在控制台或 Shell 中显示默认警告 Missing translation for message "{translation_text}"

在你的 angular.json 工作区构建配置文件中，为 build 目标指定 options 部分的警告级别。

有帮助：当你将 Angular 项目编译成 Angular 应用时，i18n 属性的实例会被 $localize 标记的消息字符串实例替换。 这意味着你的 Angular 应用是在编译后进行翻译的。 这也意味着你可以创建你 Angular 应用的本地化版本，而无需为每个语言环境重新编译整个 Angular 项目。

当你翻译你的 Angular 应用时，翻译转换会用来自翻译集合的字符串替换和重新排序模板字面量字符串的（静态字符串和表达式）部分。 有关更多信息，请参阅 $localize。

概括地说： 编译一次，然后为每种语言环境进行翻译。
