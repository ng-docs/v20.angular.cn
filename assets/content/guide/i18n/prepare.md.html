准备要翻译的组件

edit

要准备你的项目以进行翻译，请完成以下操作。

  • 使用 i18n 属性标记组件模板中的文本
  • 使用 i18n- 属性标记组件模板中的属性文本字符串
  • 使用 $localize 标记的消息字符串标记组件代码中的文本字符串

标记组件模板中的文本

在组件模板中，i18n 元数据是 i18n 属性的值。

<element i18n="{i18n_metadata}">{string_to_translate}</element>

使用 i18n 属性在组件模板中标记静态文本消息以进行翻译。 将其放在每个包含要翻译的固定文本的元素标签上。

提示： i18n 属性是 Angular 工具和编译器可识别的自定义属性。

i18n 范例

以下 <h1> 标签显示了一个简单的英语问候语“Hello i18n!”。

src/app/app.component.html

src/app/app.component.ts

import {Component, computed, signal} from '@angular/core';import {$localize} from '@angular/localize/init';@Component({  selector: 'app-root',  templateUrl: './app.component.html',})export class AppComponent {  minutes = 0;  gender = 'female';  fly = true;  logo = '${this.baseUrl}/angular.svg';  readonly toggle = signal(false);  readonly toggleAriaLabel = computed(() => {    return this.toggle()      ? $localize`:Toggle Button|A button to toggle status:Show`      : $localize`:Toggle Button|A button to toggle status:Hide`;  });  inc(i: number) {    this.minutes = Math.min(5, Math.max(0, this.minutes + i));  }  male() {    this.gender = 'male';  }  female() {    this.gender = 'female';  }  other() {    this.gender = 'other';  }  toggleDisplay() {    this.toggle.update((toggle) => !toggle);  }}

标记元素属性以进行翻译

在组件模板中，i18n 元数据是 i18n-{attribute_name} 属性的值。

<element i18n-{attribute_name}="{i18n_metadata}" {attribute_name}="{attribute_value}" />

HTML 元素的属性包含应与组件模板中其余显示的文本一起翻译的文本。

将 i18n-{attribute_name} 与任何元素的任何属性一起使用，并将 {attribute_name} 替换为属性的名称。 使用以下语法来赋予含义、描述和自定义 ID。

i18n-{attribute_name}="{meaning}|{description}@@{id}"

标记组件代码中的文本

在组件代码中，翻译源文本和元数据都用反引号 (`) 字符括起来。

使用 $localize 标记的消息字符串来标记代码中的字符串以供翻译。

$localize`string_to_translate`;

i18n 元数据用冒号 (:) 字符括起来，并位于翻译源文本的前面。

$localize`:{i18n_metadata}:string_to_translate`

包含插值文本

在 $localize 标记的消息字符串中包含 插值。

$localize`string_to_translate ${variable_name}`;

命名插值占位符 

$localize`string_to_translate ${variable_name}:placeholder_name:`;

用于翻译的条件语法 

return this.show ? $localize`Show Tabs` : $localize`Hide tabs`;

用于翻译的 i18n 元数据 

{meaning}|{description}@@{custom_id}

以下参数提供上下文和其他信息，以减少翻译人员的困惑。

元数据参数 详细信息
自定义 ID 提供自定义标识符
描述 提供其他信息或上下文
含义 在特定上下文中提供文本的含义或意图

有关自定义 ID 的更多信息，请参阅 使用自定义 ID 管理标记文本。

添加有用的描述和含义

为了准确地翻译文本消息，请为翻译人员提供其他信息或上下文。

将文本消息的描述添加为 i18n 属性的值，或添加到 $localize 标记的消息字符串中。

翻译人员可能还需要知道此特定应用上下文中文本消息的含义或意图，以便将其翻译成与其他具有相同含义的文本相同的方式。 使用含义开始 i18n 属性值，并使用 | 字符将其与描述分隔开：{meaning}|{description}。

meaning 如何控制文本的提取和合并

Angular 提取工具为模板中每个 i18n 属性生成一个翻译单元条目。 Angular 提取工具根据 meaning 和 description 为每个翻译单元赋予一个唯一的 ID。

提示： 有关 Angular 提取工具的更多信息，请参阅使用翻译文件。

具有不同 meanings 的相同文本元素会使用不同的 ID 提取。 例如，如果单词 "right" 在两个不同的位置使用以下两个定义，则该单词将被不同地翻译，并作为不同的翻译条目合并回应用中。

  • correct，如“你是对的”
  • direction，如“向右转”

如果相同的文本元素满足以下条件，则这些文本元素仅提取一次并使用相同的 ID。

  • 相同的 meaning 或定义
  • 不同的 description

无论在应用中何处出现相同的文本元素，该翻译条目都会被合并回应用中。

ICU 表达式

ICU 表达式可帮助你在组件模板中标记备用文本以满足条件。 ICU 表达式包括组件属性（Property）、ICU 子句以及用左花括号 ({) 和右花括号 (}) 字符括起来的 case 语句。

{ component_property, icu_clause, case_statements }

组件属性定义变量。 ICU 子句定义条件文本的类型。

ICU 子句 详细信息
plural 标记复数数字的使用
select 根据你定义的字符串值，标记备用文本的选项

为了简化翻译，请将 Unicode 国际组件子句（ICU 子句）与正则表达式结合使用。

有帮助：ICU 子句遵循 ICU 消息格式，该格式在 CLDR 複數化规则 中指定。

标记复数

不同的语言有不同的复数规则，这增加了翻译的难度。 由于其他语言环境表示基数的方式不同，你可能需要设置与英语不一致的复数类别。 使用 plural 子句来标记逐字翻译可能没有意义的表达式。

{ component_property, plural, pluralization_categories }

在复数类别之后，输入用左花括号 ({) 和右花括号 (}) 字符括起来的默认文本（英语）。

pluralization_category { }

以下复数类别适用于英语，并且可能会根据语言环境而变化。

复数类别 详细信息 范例
zero 数量为零 =0 { }
zero { }
one 数量为 1 =1 { }
one { }
two 数量为 2 =2 { }
two { }
few 数量为 2 或更多 few { }
many 数量很大 many { }
other 默认数量 other { }

如果所有复数类别都不匹配，Angular 将使用 other 来匹配缺失类别的标准回退。

other { default_quantity }

提示：有关复数化类别的更多信息，请参阅 CLDR - Unicode 通用区域设置数据存储库 中的“选择复数类别名称”。

许多语言环境不支持某些复数化类别。 默认语言环境 (en-US) 使用一个非常简单的 plural() 函数，该函数不支持 few 复数化类别。 另一个具有简单 plural() 函数的语言环境是 es。 以下代码示例显示了 en-US plural() 函数。

function plural(n: number): number {  let i = Math.floor(Math.abs(n)),    v = n.toString().replace(/^[^.]*\.?/, '').length;  if (i === 1 && v === 0) return 1;  return 5;}

plural() 函数仅返回 1 (one) 或 5 (other)。 few 类别永远不匹配。

标记备选项和嵌套表达式

select 子句根据你定义的字符串值标记备用文本的选项。

{ component_property, select, selection_categories }

翻译所有的备选项，以根据变量的值显示备选文本。

在选择类别之后，输入用左花括号 ({) 和右花括号 (}) 字符包围的文本（英文）。

selection_category { text }

不同的语言环境有不同的语法结构，这增加了翻译的难度。 使用 HTML 标记。 如果没有选择类别匹配，Angular 将使用 other 来匹配缺失类别的标准回退。

other { default_value }
