添加 localize 包

edit

要利用 Angular 的本地化特性，请使用 Angular CLI 将 @angular/localize 包添加到你的项目中。

要添加 @angular/localize 包，请使用以下命令来更新项目中的 package.json 和 TypeScript 配置文件。

ng add @angular/localizeng extract-i18nng extract-i18n --output-path src/localeng extract-i18n --format=xlfng extract-i18n --format=xlf2ng extract-i18n --format=xmbng extract-i18n --format=jsonng extract-i18n --format=arbng extract-i18n --out-file source.xlfng build --localizeng serve --configuration=frng build --configuration=production,fr

它会在 TypeScript 配置文件中添加 types: ["@angular/localize"]。 它还会在 main.ts 文件的顶部添加 /// <reference types="@angular/localize" /> 行，这是对类型定义的引用。

有帮助：有关 package.json 和 tsconfig.json 文件的更多信息，请参阅 工作区 npm 依赖项 和 TypeScript 配置。要了解三斜杠指令，请访问 TypeScript 手册。

如果未安装 @angular/localize 并且您尝试构建项目的本地化版本（例如，在模板中使用 i18n 属性时），Angular CLI 将生成一个错误，其中包含您可以为项目启用 i18n 的步骤。

选项

OPTION DESCRIPTION 值类型 默认值
--project 项目的名称。 string
--use-at-runtime 如果设置，则可以在运行时使用 $localize。此外，@angular/localize 会包含在 package.json 的 dependencies 部分，而不是默认的 devDependencies 部分。 boolean false

有关更多可用选项，请参阅 Angular CLI 中的 ng add。
