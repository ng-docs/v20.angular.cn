定义路由

edit

路由是 Angular 应用中导航的基本构建块。

什么是路由？

在 Angular 中，路由是一个对象，它定义了在特定 URL 路径或模式下应渲染哪个组件，以及当用户导航到该 URL 时会发生什么的其他配置选项。

这是一个路由的基本示例：

import { AdminPage } from './app-admin/app-admin.component';const adminPage = {  path: 'admin',  component: AdminPage}

对于此路由，当用户访问 /admin 路径时，应用将显示 AdminPage 组件。

管理应用中的路由

大多数项目在包含文件名中带有 routes 的单独文件中定义路由。

路由集合看起来像这样：

import { Routes } from '@angular/router';import { HomePage } from './home-page/home-page.component';import { AdminPage } from './about-page/admin-page.component';export const routes: Routes = [  {    path: '',    component: HomePage,  },  {    path: 'admin',    component: AdminPage,  },];

提示：如果您使用 Angular CLI 生成了项目，您的路由定义在 src/app/app.routes.ts 中。

将路由器添加到您的应用

在不使用 Angular CLI 引导 Angular 应用时，您可以传递一个包含 providers 数组的配置对象。

在 providers 数组中，您可以通过调用带有路由的 provideRouter 函数，将 Angular 路由器添加到您的应用中。

import { ApplicationConfig } from '@angular/core';import { provideRouter } from '@angular/router';import { routes } from './app.routes';export const appConfig: ApplicationConfig = {  providers: [    provideRouter(routes),    // ...  ]};

路由 URL 路径

静态 URL 路径

静态 URL 路径是指具有预定义路径的路由，这些路径不会根据动态参数而改变。这些路由会精确匹配一个 path 字符串并产生固定的结果。

这包括的示例有：

  • "/admin"
  • "/blog"
  • "/settings/account"

使用路由参数定义 URL 路径

参数化 URL 允许您定义动态路径，这些路径允许多个 URL 指向同一个组件，同时根据 URL 中的参数动态显示数据。

您可以通过在路由的 path 字符串中添加参数，并在每个参数前加上冒号 (:) 字符来定义此类模式。

重要提示：参数与 URL 的 查询字符串中的信息不同。 在本指南中了解更多关于 Angular 中的查询参数。

以下示例根据通过 URL 传入的用户 ID 显示用户配置文件组件。

import { Routes } from '@angular/router';import { UserProfile } from './user-profile/user-profile;const routes: Routes = [  { path: 'user/:id', component: UserProfile }];

在此示例中，诸如 /user/leeroy 和 /user/jenkins 之类的 URL 会渲染 UserProfile 组件。然后，该组件可以读取 id 参数并使用它来执行其他工作，例如获取数据。有关读取路由参数的详细信息，请参阅 读取路由状态指南。

有效的路由参数名称必须以字母 (a-z, A-Z) 开头，并且只能包含：

  • 字母 (a-z, A-Z)
  • 数字 (0-9)
  • 下划线 (_)
  • 连字符 (-)

您还可以定义带有多个参数的路径：

import { Routes } from '@angular/router';import { UserProfile } from './user-profile/user-profile.component';import { SocialMediaFeed } from './user-profile/social–media-feed.component';const routes: Routes = [  { path: 'user/:id/:social-media', component: SocialMediaFeed },  { path: 'user/:id/', component: UserProfile },];

使用此新路径，用户可以访问 /user/leeroy/youtube 和 /user/leeroy/bluesky，并根据用户 leeroy 的参数查看各自的社交媒体信息流。

有关读取路由参数的详细信息，请参阅 读取路由状态。

通配符

当您需要捕获特定路径的所有路由时，解决方案是使用双星号 (**) 定义的通配符路由。

一个常见的示例是定义一个“未找到页面”组件。

import { Home } from './home/home.component';import { UserProfile } from './user-profile/user-profile.component';import { NotFound } from './not-found/not-found.component';const routes: Routes = [  { path: 'home', component: Home },  { path: 'user/:id', component: UserProfile },  { path: '**', component: NotFound }];

在此路由数组中，当用户访问 home 和 user/:id 之外的任何路径时，应用将显示 NotFound 组件。

提示：通配符路由通常放在路由数组的末尾。

Angular 如何匹配 URL

定义路由时，顺序很重要，因为 Angular 使用“首次匹配获胜”策略。这意味着一旦 Angular 将 URL 与路由 path 匹配，它就会停止检查任何进一步的路由。因此，请始终将更具体的路由放在不太具体的路由之前。

以下示例显示了从最具体到最不具体的路由定义：

const routes: Routes = [  { path: '', component: HomeComponent },              // Empty path  { path: 'users/new', component: NewUserComponent },  // Static, most specific  { path: 'users/:id', component: UserDetailComponent }, // Dynamic  { path: 'users', component: UsersComponent },        // Static, less specific  { path: '**', component: NotFoundComponent }         // Wildcard - always last];

如果用户访问 /users/new，Angular 路由器将执行以下步骤：

  1. 检查 '' - 不匹配
  2. 检查 users/new - 匹配！在此停止
  3. 即使可以匹配，也永远不会到达 users/:id
  4. 永远不会到达 users
  5. 永远不会到达 **

加载路由组件策略

了解组件在 Angular 路由中如何以及何时加载对于构建响应式 Web 应用至关重要。Angular 提供了两种主要策略来控制组件加载行为：

  1. 急性加载：立即加载的组件
  2. 惰性加载：仅在需要时加载的组件

每种方法都为不同的场景提供了独特的优势。

急性加载的组件

当您使用 component 属性定义路由时，引用的组件将作为路由配置的同一 JavaScript 捆绑包的一部分被急性加载。

import { Routes } from "@angular/router";import { HomePage } from "./components/home/home-page"import { LoginPage } from "./components/auth/login-page"export const routes: Routes = [  // HomePage and LoginPage are both directly referenced in this config,  // so their code is eagerly included in the same JavaScript bundle as this file.  {    path: "",    component: HomePage  },  {    path: "login",    component: LoginPage  }]

像这样急性加载路由组件意味着浏览器必须在初始页面加载时下载和解析这些组件的所有 JavaScript，但这些组件会立即提供给 Angular。

虽然在初始页面加载中包含更多 JavaScript 会导致更慢的初始加载时间，但这可以为用户在应用程序中导航时提供更无缝的过渡。

惰性加载的组件

您可以使用 loadComponent 属性仅在路由变为活动状态时惰性加载该路由的 JavaScript。

import { Routes } from "@angular/router";export const routes: Routes = [  // The HomePage and LoginPage components are loaded lazily at the point at which  // their corresponding routes become active.  {    path: 'login',    loadComponent: () => import('./components/auth/login-page').then(m => m.LoginPage)  },  {    path: '',    loadComponent: () => import('./components/home/home-page').then(m => m.HomePage)  }]

loadComponent 属性接受一个加载器函数，该函数返回一个解析为 Angular 组件的 Promise。在大多数情况下，此函数使用标准的 JavaScript 动态导入 API。但是，您可以使用任何任意的异步加载器函数。

惰性加载路由可以通过从初始捆绑包中删除大量 JavaScript 来显著提高 Angular 应用程序的加载速度。这些代码部分会编译成单独的 JavaScript “块”，路由器仅在用户访问相应路由时才请求它们。

我应该使用急性路由还是惰性路由？

在决定路由应该是急性的还是惰性的时，有许多因素需要考虑。

总的来说，建议对主要登陆页面进行急性加载，而其他页面则进行惰性加载。

注意：虽然惰性路由具有减少用户初始请求数据量的前期性能优势，但它会增加未来可能不希望出现的数据请求。在处理多层嵌套惰性加载时尤其如此，这会严重影响性能。

重定向

您可以定义一个路由，该路由重定向到另一个路由，而不是渲染组件：

import { BlogComponent } from './home/blog.component';const routes: Routes = [  {    path: 'articles',    redirectTo: '/blog',  },  {    path: 'blog',    component: BlogComponent  },];

如果您修改或删除了路由，一些用户仍可能点击过时的链接或书签。您可以添加重定向，将这些用户引导到合适的替代路由，而不是“未找到”页面。

页面标题

您可以为每个路由关联一个 title。当路由激活时，Angular 会自动更新 页面标题。请始终为您的应用程序定义合适的页面标题，因为这些标题对于创建无障碍体验是必需的。

import { Routes } from '@angular/router';import { HomeComponent } from './home/home.component';import { AboutComponent } from './about/about.component';import { ProductsComponent } from './products/products.component';const routes: Routes = [  {    path: '',    component: HomeComponent,    title: 'Home Page'  },  {    path: 'about',    component: AboutComponent,    title: 'About Us'  },];

页面 title 属性可以使用 ResolveFn 动态设置为解析器函数。

const titleResolver: ResolveFn<string> = (route) => route.queryParams['id'];const routes: Routes = [   ...  {    path: 'products',    component: ProductsComponent,    title: titleResolver,  }];

路由标题也可以通过扩展 TitleStrategy 抽象类的服务来设置。默认情况下，Angular 使用 DefaultTitleStrategy。

用于依赖注入的路由级提供者

每个路由都有一个 providers 属性，允许您通过 依赖注入 为该路由的内容提供依赖项。

这可能很有帮助的常见场景包括根据用户是否为管理员而具有不同服务的应用程序。

export const ROUTES: Route[] = [  {    path: 'admin',    providers: [      AdminService,      {provide: ADMIN_API_KEY, useValue: '12345'},    ],    children: [      {path: 'users', component: AdminUsersComponent},      {path: 'teams', component: AdminTeamsComponent},    ],  },  // ... other application routes that don't  //     have access to ADMIN_API_KEY or AdminService.];

在此代码示例中，admin 路径包含一个受保护的 ADMIN_API_KEY 数据属性，该属性仅对该部分内的子项可用。因此，其他路径将无法访问通过 ADMIN_API_KEY 提供的数据。

有关 Angular 中提供者和注入的更多信息，请参阅 依赖注入指南。

将数据与路由关联

路由数据使您能够将附加信息附加到路由。您可以根据这些数据配置组件的行为方式。

有两种方法可以处理路由数据：保持不变的静态数据，以及根据运行时条件可能发生变化的动态数据。

静态数据

您可以通过 data 属性将任意静态数据与路由关联，以集中处理诸如特定于路由的元数据（例如，分析跟踪、权限等）之类的内容：

import { Routes } from '@angular/router';import { HomeComponent } from './home/home.component';import { AboutComponent } from './about/about.component';import { ProductsComponent } from './products/products.component';const routes: Routes = [  {    path: 'about',    component: AboutComponent,    data: { analyticsId: '456' }  },  {    path: '',    component: HomeComponent,    data: { analyticsId: '123' }  }];

在此代码示例中，主页和关于页面都配置了特定的 analyticsId，这些 ID 将在各自的组件中用于页面跟踪分析。

您可以通过注入 ActivatedRoute 来读取此静态数据。有关详细信息，请参阅 读取路由状态。

带数据解析器的动态数据

当您需要为路由提供动态数据时，请查看 路由数据解析器指南。

嵌套路由

嵌套路由，也称为子路由，是一种管理更复杂导航路由的常用技术，其中组件具有基于 URL 更改的子视图。

您可以使用 children 属性将子路由添加到任何路由定义中：

const routes: Routes = [  {    path: 'product/:id',    component: ProductComponent,    children: [      {        path: 'info',        component: ProductInfoComponent      },      {        path: 'reviews',        component: ProductReviewsComponent      }    ]  }]

上面的示例为产品页面定义了一个路由，该路由允许用户根据 URL 更改显示产品信息还是评论。

children 属性接受一个 Route 对象数组。

要显示子路由，父组件（上面示例中的 ProductComponent）需要包含自己的 <router-outlet>。

<!-- ProductComponent --><article>  <h1>Product {{ id }}</h1>  <router-outlet /></article>

在将子路由添加到配置并向组件添加 <router-outlet> 后，匹配子路由的 URL 之间的导航仅会更新嵌套的出口。
