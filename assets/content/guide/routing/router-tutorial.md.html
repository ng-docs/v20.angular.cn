在单页应用中使用 Angular 路由

edit

本教程介绍如何构建一个使用多个 Angular 路由的单页应用（SPA）。

在单页应用（SPA）中，你的应用的所有功能都存在于单个 HTML 页面中。 当用户访问应用的功能时，浏览器只需要渲染对用户重要的部分，而不是加载新页面。 这种模式可以显著改善应用的用户体验。

要定义用户如何在你的应用中导航，你可以使用路由。 添加路由以定义用户如何从应用的某一部分导航到另一部分。 你也可以配置路由以防止意外或未经授权的行为。

目标

  • 将示例应用的功能组织成模块。
  • 定义如何导航到组件。
  • 使用参数将信息传递给组件。
  • 通过嵌套多个路由来构建路由结构。
  • 检查用户是否可以访问路由。
  • 控制应用是否可以放弃未保存的更改。
  • 通过预取路由数据和惰性加载特性模块来提高性能。
  • 要求满足特定条件才能加载组件。

创建一个示例应用

使用 Angular CLI，创建一个新的应用，angular-router-sample。 此应用将有两个组件：crisis-list 和 heroes-list。

  1. 创建一个新的 Angular 项目，angular-router-sample。

    ng new angular-router-sample

    当提示 Would you like to add Angular routing? 时，选择 N。

    当提示 Which stylesheet format would you like to use? 时，选择 CSS。

    稍等片刻，新的项目 angular-router-sample 就准备好了。

  2. 从终端中，导航到 angular-router-sample 目录。

  3. 创建一个组件，crisis-list。

    ng generate component crisis-list

  4. 在你的代码编辑器中，找到文件 crisis-list.component.html，并将占位符内容替换为以下 HTML。

    src/app/crisis-list/crisis-list.component.html

    <h3>CRISIS CENTER</h3><p>Get your crisis here</p>

  5. 创建第二个组件，heroes-list。

    ng generate component heroes-list

  6. 在你的代码编辑器中，找到文件 heroes-list.component.html，并将占位符内容替换为以下 HTML。

    src/app/heroes-list/heroes-list.component.html

    <h3>HEROES</h3><p>Get your heroes here</p>

  7. 在你的代码编辑器中，打开文件 app.component.html，并将其内容替换为以下 HTML。

    src/app/app.component.html

    <h1>Angular Router Sample</h1><nav>  <a class="button"     routerLink="/crisis-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Crisis Center  </a> |  <a class="button"     routerLink="/heroes-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Heroes  </a></nav><router-outlet></router-outlet><div style="display: none;"><!-- This HTML represents the initial state for the tutorial. It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>

  8. 通过运行 ng serve 命令，验证你的新应用是否按预期运行。

    ng serve

  9. 在浏览器中打开 http://localhost:4200。

    你应该看到一个单页，其中包含标题和你两个组件的 HTML。

定义你的路由

在本节中，你将定义两个路由：

  • 路由 /crisis-center 打开 crisis-center 组件。
  • 路由 /heroes-list 打开 heroes-list 组件。

路由定义是一个 JavaScript 对象。 每个路由通常有两个属性。 第一个属性 path 是一个字符串，用于指定路由的 URL 路径。 第二个属性 component 是一个字符串，用于指定你的应用应该为该路径显示的组件。

  1. 在你的代码编辑器中，创建并打开 app.routes.ts 文件。

  2. 为你的应用创建并导出一个路由列表：

    import {Routes} from '@angular/router';export const routes = [];

  3. 为你的前两个组件添加两个路由：

    {path: 'crisis-list', component: CrisisListComponent},{path: 'heroes-list', component: HeroesListComponent},

此路由列表是一个 JavaScript 对象数组，每个对象都定义了一个路由的属性。

从 @angular/router 导入 provideRouter

路由允许您根据 URL 路径显示应用的特定视图。 要将此功能添加到您的示例应用中，您需要更新 app.config.ts 文件以使用路由提供者函数 provideRouter。 您从 @angular/router 导入此提供者函数。

  1. 在你的代码编辑器中，打开 app.config.ts 文件。

  2. 添加以下导入语句：

    import { provideRouter } from '@angular/router';import { routes } from './app.routes';

  3. 更新 appConfig 中的 providers：

    providers: [provideRouter(routes)]

对于基于 NgModule 的应用，将 provideRouter 放在 AppModule 的 providers 列表中，或者放在传递给应用中 bootstrapModule 的任何模块的 providers 列表中。

使用 router-outlet 更新你的组件

此时，你已经为你的应用定义了两个路由。 但是，你的应用仍然在 app.component.html 模板中硬编码了 crisis-list 和 heroes-list 组件。 为了使你的路由工作，你需要更新你的模板以根据 URL 路径动态加载组件。

要实现此功能，你需要将 router-outlet 指令添加到你的模板文件中。

  1. 在你的代码编辑器中，打开 app.component.html 文件。

  2. 删除以下行。

    src/app/app.component.html

It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>

  3. 添加 router-outlet 指令。

    src/app/app.component.html

    <h1>Angular Router Sample</h1><nav>  <a class="button"     routerLink="/crisis-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Crisis Center  </a> |  <a class="button"     routerLink="/heroes-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Heroes  </a></nav><router-outlet></router-outlet><div style="display: none;"><!-- This HTML represents the initial state for the tutorial. It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>

  4. 在 app.component.ts 文件中的 AppComponent 的 imports 中添加 RouterOutlet。

    imports: [RouterOutlet],

在浏览器中查看你更新后的应用。 你应该只看到应用标题。 要查看 crisis-list 组件，请将 crisis-list 添加到浏览器地址栏中路径的末尾。 例如：

http://localhost:4200/crisis-list

请注意，crisis-list 组件会显示出来。 Angular 正在使用你定义的路由来动态加载该组件。 你可以用同样的方式加载 heroes-list 组件：

http://localhost:4200/heroes-list

使用 UI 元素控制导航

目前，你的应用支持两个路由。 然而，使用这些路由的唯一方法是让用户在浏览器的地址栏中手动输入路径。 在本节中，你将添加两个链接，用户可以单击这些链接在 heroes-list 和 crisis-list 组件之间导航。 你还将添加一些 CSS 样式。 虽然这些样式不是必需的，但它们可以更容易地识别当前显示组件的链接。 你将在下一节中添加该功能。

  1. 打开 app.component.html 文件，并在标题下方添加以下 HTML。

    src/app/app.component.html

    <h1>Angular Router Sample</h1><nav>  <a class="button"     routerLink="/crisis-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Crisis Center  </a> |  <a class="button"     routerLink="/heroes-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Heroes  </a></nav><router-outlet></router-outlet><div style="display: none;"><!-- This HTML represents the initial state for the tutorial. It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>

    此 HTML 使用了一个 Angular 指令 routerLink。 此指令将你定义的路由连接到你的模板文件。

  2. 在 app.component.ts 文件中的 AppComponent 的 imports 列表中添加 RouterLink 指令。

  3. 打开 app.component.css 文件，并添加以下样式。

    src/app/app.component.css

    .button {    box-shadow: inset 0 1px 0 0 #ffffff;    background: #ffffff linear-gradient(to bottom, #ffffff 5%, #f6f6f6 100%);    border-radius: 6px;    border: 1px solid #dcdcdc;    display: inline-block;    cursor: pointer;    color: #666666;    font-family: Arial, sans-serif;    font-size: 15px;    font-weight: bold;    padding: 6px 24px;    text-decoration: none;    text-shadow: 0 1px 0 #ffffff;    outline: 0;}.activebutton {    box-shadow: inset 0 1px 0 0 #dcecfb;    background: #bddbfa linear-gradient(to bottom, #bddbfa 5%, #80b5ea 100%);    border: 1px solid #84bbf3;    color: #ffffff;    text-shadow: 0 1px 0 #528ecc;}

如果在浏览器中查看你的应用，你应该会看到这两个链接。 当你单击链接时，相应的组件就会出现。

识别活动路由

虽然用户可以使用你在上一节中添加的链接来导航你的应用，但他们没有直接的方式来识别哪个是活动路由。 使用 Angular 的 routerLinkActive 指令添加此功能。

  1. 在你的代码编辑器中，打开 app.component.html 文件。
  2. 更新这些锚点标签，以包含 routerLinkActive 指令。

    src/app/app.component.html

    <h1>Angular Router Sample</h1><nav>  <a class="button"     routerLink="/crisis-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Crisis Center  </a> |  <a class="button"     routerLink="/heroes-list"     routerLinkActive="activebutton"     ariaCurrentWhenActive="page">    Heroes  </a></nav><router-outlet></router-outlet><div style="display: none;"><!-- This HTML represents the initial state for the tutorial. It is not intended to appear in the app. --><app-crisis-list></app-crisis-list><app-heroes-list></app-heroes-list><!-- This HTML snippet is for when the user first adds the routerlink navigation. --><nav>  <a class="button" routerLink="/crisis-list">Crisis Center</a> |  <a class="button" routerLink="/heroes-list">Heroes</a></nav></div>
  3. 在 app.component.ts 文件中的 AppComponent 的 imports 列表中添加 RouterLinkActive 指令。

再次查看你的应用。 当你单击其中一个按钮时，该按钮的样式会自动更新，从而向用户标识出活动的组件。 通过添加 routerLinkActive 指令，你可以通知你的应用将特定的 CSS 类应用于活动路由。 在本教程中，该 CSS 类是 activebutton，但你可以使用任何你想要的类。

请注意，我们还为 routerLinkActive 的 ariaCurrentWhenActive 指定了一个值。这确保了视觉障碍用户（他们可能无法感知到正在应用的不同样式）也可以识别出活动按钮。有关更多信息，请参阅无障碍功能最佳实践的活动链接识别部分。

添加重定向

在本教程的这一步中，你将添加一个路由，该路由会将用户重定向以显示 /heroes-list 组件。

  1. 在你的代码编辑器中，打开 app.routes.ts 文件。

  2. 按如下所示更新 routes 部分。

    {path: '', redirectTo: '/heroes-list', pathMatch: 'full'},

    请注意，这个新路由使用一个空字符串作为其路径。 此外，它使用两个新属性替换了 component 属性：

    属性 详情
    redirectTo 此属性指示 Angular 从空路径重定向到 heroes-list 路径。
    pathMatch 此属性指示 Angular 如何匹配 URL 的多少。对于本教程，你应该将此属性设置为 full。当你的路径为空字符串时，建议使用此策略。有关此属性的更多信息，请参阅 Route API 文档。

现在，当你打开你的应用时，它默认会显示 heroes-list 组件。

添加 404 页面

用户有可能尝试访问你尚未定义的路由。 为了应对这种行为，最佳实践是显示 404 页面。 在本节中，你将创建一个 404 页面并更新你的路由配置，以便为任何未指定的路由显示该页面。

  1. 从终端中，创建一个新组件 PageNotFound。

    ng generate component page-not-found

  2. 在你的代码编辑器中，打开 page-not-found.component.html 文件，并将其内容替换为以下 HTML。

    src/app/page-not-found/page-not-found.component.html

    <h2>Page Not Found</h2><p>We couldn't find that page! Not even with x-ray vision.</p>

  3. 打开 app.routes.ts 文件，并将以下路由添加到路由列表中：

    {path: '**', component: PageNotFoundComponent}

    新路由使用路径 **。 此路径是 Angular 识别通配符路由的方式。 任何与你的配置中现有路由不匹配的路由都将使用此路由。

重要提示： 请注意，通配符路由位于数组的末尾。 你的路由顺序非常重要，因为 Angular 会按顺序应用路由并使用它找到的第一个匹配项。

尝试导航到你的应用上不存在的路由，例如 http://localhost:4200/powers。 此路由与你在 app.routes.ts 文件中定义的任何内容都不匹配。 但是，由于你定义了一个通配符路由，因此应用会自动显示你的 PageNotFound 组件。
