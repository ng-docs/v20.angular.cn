显示带出口的路由

edit

RouterOutlet 指令是一个占位符，它标记了路由应该渲染当前 URL 所对应组件的位置。

<app-header /><router-outlet />  <!-- Angular inserts your route content here --><app-footer />
import { Component } from '@angular/core';import { RouterOutlet } from '@angular/router';@Component({  selector: 'app-root',  imports: [RouterOutlet],  templateUrl: './app.component.html',  styleUrl: './app.component.css'})export class AppComponent {}

在此示例中，如果一个应用定义了以下路由：

import { Routes } from '@angular/router';import { HomeComponent } from './home/home.component';import { ProductsComponent } from './products/products.component';const routes: Routes = [  {    path: '',    component: HomeComponent,    title: 'Home Page'  },  {    path: 'products',    component: ProductsComponent,    title: 'Our Products'  }];

当用户访问 /products 时，Angular 渲染如下：

<app-header /><app-products /><app-footer />

如果用户返回主页，则 Angular 渲染如下：

<app-header /><app-home /><app-footer />

显示路由时，<router-outlet> 元素会作为未来导航的参考点保留在 DOM 中。Angular 将路由内容作为兄弟节点插入到出口元素之后。

<!-- Contents of the component's template --><app-header /><router-outlet /><app-footer />
<!-- Content rendered on the page when the user visits /admin --><app-header>...</app-header><router-outlet></router-outlet><app-admin-page>...</app-admin-page><app-footer>...</app-footer>

使用子路由嵌套路由

随着您的应用程序变得越来越复杂，您可能希望创建相对于根组件以外的组件的路由。这使您能够创建这样的体验：当 URL 更改时，只有应用程序的一部分会发生变化，而不是让用户感觉整个页面都刷新了。

这些类型的嵌套路由称为子路由。这意味着您正在向您的应用添加第二个 <router-outlet>，因为它是在 AppComponent 中的 <router-outlet> 之外添加的。

在此示例中，Settings 组件将根据用户的选择显示所需的面板。您会注意到子路由的一个独特之处是，该组件通常有自己的 <nav> 和 <router-outlet>。

<h1>Settings</h1><nav>  <ul>    <li><a routerLink="profile">Profile</a></li>    <li><a routerLink="security">Security</a></li>  </ul></nav><router-outlet />

子路由与其他路由一样，需要 path 和 component。唯一的区别是您将子路由放在父路由的 children 数组中。

const routes: Routes = [  {    path: 'settings-component',    component: SettingsComponent, // this is the component with the <router-outlet> in the template    children: [      {        path: 'profile', // child route path        component: ProfileComponent, // child route component that the router renders      },      {        path: 'security',        component: SecurityComponent, // another child route component that the router renders      },    ],  },];

一旦 routes 和 <router-outlet> 都配置正确，您的应用程序现在就可以使用嵌套路由了！

带命名出口的次级路由

页面可以有多个出口—您可以为每个出口分配一个名称，以指定哪个内容属于哪个出口。

<app-header /><router-outlet /><router-outlet name='read-more' /><router-outlet name='additional-actions' /><app-footer />

每个出口都必须有一个唯一的名称。名称不能动态设置或更改。默认情况下，名称是 'primary'。

Angular 将出口的名称与每个路由上定义的 outlet 属性进行匹配：

{  path: 'user/:id',  component: UserDetails,  outlet: 'additional-actions'}

出口生命周期事件

路由出口可以发出四种生命周期事件：

事件 描述
activate 当实例化新组件时
deactivate 当组件被销毁时
attach 当 RouteReuseStrategy 指示出口附加子树时
detach 当 RouteReuseStrategy 指示出口分离子树时

您可以使用标准的事件绑定语法添加事件监听器：

<router-outlet  (activate)='onActivate($event)'  (deactivate)='onDeactivate($event)'  (attach)='onAttach($event)'  (detach)='onDetach($event)'/>

如果您想了解更多信息，请查看 RouterOutlet 的 API 文档。
