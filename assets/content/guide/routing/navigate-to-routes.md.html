导航到路由

edit

RouterLink 指令是 Angular 声明式导航的方法。它允许你使用标准的锚点元素 (<a>)，这些元素可以与 Angular 的路由系统无缝集成。

如何使用 RouterLink

与其使用带有 href 属性的常规锚点元素 <a>，不如添加带有相应路径的 RouterLink 指令，以利用 Angular 路由。

import {RouterLink} from '@angular/router';@Component({  template: `    <nav>      <a routerLink="/user-profile">User profile</a>      <a routerLink="/settings">Settings</a>    </nav>  `  imports: [RouterLink],  ...})export class App {}

使用绝对或相对链接

Angular 路由中的相对 URL 允许你定义相对于当前路由位置的导航路径。这与绝对 URL 不同，绝对 URL 包含带协议（例如 http://）和根域（例如 google.com）的完整路径。

<!-- Absolute URL --><a href="https://www.angular.dev/essentials">Angular Essentials Guide</a><!-- Relative URL --><a href="/essentials">Angular Essentials Guide</a>

在此示例中，第一个示例为 essentials 页面明确定义了包含协议（即 https://）和根域（即 angular.dev）的完整路径。相比之下，第二个示例假定用户已在 /essentials 的正确根域上。

总的来说，相对 URL 更受青睐，因为它们在跨应用程序时更易于维护，因为它们不需要知道其在路由层级结构中的绝对位置。

相对 URL 的工作原理

Angular 路由有两种定义相对 URL 的语法：字符串和数组。

<!-- Navigates user to /dashboard --><a routerLink="dashboard">Dashboard</a><a [routerLink]="['dashboard']">Dashboard</a>

有帮助：传递字符串是定义相对 URL 最常见的方式。

当你需要定义相对 URL 中的动态参数时，请使用数组语法：

<a [routerLink]="['user', currentUserId]">Current User</a>

此外，Angular 路由允许你根据相对路径是否以斜杠（/）为前缀来指定路径是相对于当前 URL 还是根域。

例如，如果用户在 example.com/settings 上，以下是针对各种场景定义不同相对路径的方法：

<!-- Navigates to /settings/notifications --><a routerLink="notifications">Notifications</a><a routerLink="/settings/notifications">Notifications</a><!-- Navigates to /team/:teamId/user/:userId --><a routerLink="/team/123/user/456">User 456</a><a [routerLink]="['/team', teamId, 'user', userId]">Current User</a>”

以编程方式导航到路由

虽然 RouterLink 处理模板中的声明式导航，但 Angular 为需要基于逻辑、用户操作或应用程序状态进行导航的场景提供了以编程方式导航的功能。通过注入 Router，你可以在 TypeScript 代码中动态导航到路由、传递参数以及控制导航行为。

router.navigate()

你可以使用 router.navigate() 方法通过指定 URL 路径数组来以编程方式在路由之间导航。

import { Router } from '@angular/router';@Component({  selector: 'app-dashboard',  template: `    <button (click)="navigateToProfile()">View Profile</button>  `})export class AppDashboard {  private router = inject(Router);  navigateToProfile() {    // Standard navigation    this.router.navigate(['/profile']);    // With route parameters    this.router.navigate(['/users', userId]);    // With query parameters    this.router.navigate(['/search'], {      queryParams: { category: 'books', sort: 'price' }    });  }}

router.navigate() 支持简单和复杂的路由场景，允许你传递路由参数、查询参数以及控制导航行为。

你还可以使用 relativeTo 选项根据组件在路由树中的位置构建动态导航路径。

import { Router, ActivatedRoute } from '@angular/router';@Component({  selector: 'app-user-detail',  template: `    <button (click)="navigateToEdit()">Edit User</button>    <button (click)="navigateToParent()">Back to List</button>  `})export class UserDetailComponent {  private route = inject(ActivatedRoute);  private router = inject(Router);  constructor() {}  // Navigate to a sibling route  navigateToEdit() {    // From: /users/123    // To:   /users/123/edit    this.router.navigate(['edit'], { relativeTo: this.route });  }  // Navigate to parent  navigateToParent() {    // From: /users/123    // To:   /users    this.router.navigate(['..'], { relativeTo: this.route });  }}

router.navigateByUrl()

router.navigateByUrl() 方法提供了一种直接的方式，可以使用 URL 路径字符串而不是数组段来进行以编程方式导航。当你拥有完整的 URL 路径并需要执行绝对导航时，此方法非常理想，尤其是在处理外部提供的 URL 或深度链接场景时。

// Standard route navigationrouter.navigateByUrl('/products);// Navigate to nested routerouter.navigateByUrl('/products/featured');// Complete URL with parameters and fragmentrouter.navigateByUrl('/products/123?view=details#reviews');// Navigate with query parametersrouter.navigateByUrl('/search?category=books&sortBy=price');

如果你需要在历史记录中替换当前 URL，navigateByUrl 还接受一个包含 replaceUrl 选项的配置对象。

// Replace current URL in historyrouter.navigateByUrl('/checkout', {  replaceUrl: true});
