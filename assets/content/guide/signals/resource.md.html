使用资源实现异步响应性

edit

重要：resource 是实验性的。它已准备好供您试用，但在稳定之前可能会发生变化。

大多数信号 API 是同步的——signal、computed、input 等。然而，应用程序通常需要处理异步可用的数据。 Resource 为您提供了一种将异步数据整合到应用程序的基于信号的代码中的方法。

您可以使用 Resource 来执行任何类型的异步操作，但 Resource 最常见的用例是从服务器获取数据。以下示例创建一个资源来获取一些用户数据。

创建 Resource 最简单的方法是使用 resource 函数。

import {resource, Signal} from '@angular/core';const userId: Signal<string> = getUserId();const userResource = resource({  // Define a reactive computation.  // The params value recomputes whenever any read signals change.  params: () => ({id: userId()}),  // Define an async loader that retrieves data.  // The resource calls this function every time the `params` value changes.  loader: ({params}) => fetchUser(params),});// Create a computed signal based on the result of the resource's loader function.const firstName = computed(() => {  if (userResource.hasValue()) {    // `hasValue` serves 2 purposes:    // - It acts as type guard to strip `undefined` from the type    // - If protects against reading a throwing `value` when the resource is in error state    return userResource.value().firstName;  }  // fallback in case the resource value is `undefined` or if the resource is in error state  return undefined;});

resource函数接受一个ResourceOptions对象，该对象具有两个主要属性：params和loader。

params属性定义了一个生成参数值的响应式计算。每当此计算中读取的信号发生更改时，资源就会生成一个新的参数值，这与computed类似。

loader属性定义了一个ResourceLoader——一个检索某些状态的异步函数。每当params计算生成新值时，资源都会调用加载器，并将该值传递给加载器。有关更多详细信息，请参见下面的资源加载器。

Resource有一个value信号，其中包含加载器的结果。

资源加载器

创建资源时，您需要指定一个ResourceLoader。此加载器是一个异步函数，它接受单个参数——一个ResourceLoaderParams对象——并返回一个值。

ResourceLoaderParams对象包含三个属性：params、previous和abortSignal。

属性 描述
params 资源params计算的值。
previous 一个具有status属性的对象，包含之前的ResourceStatus。
abortSignal 一个 AbortSignal。有关详细信息，请参见下文的中止请求。

如果params计算返回undefined，则加载器函数不会运行，资源状态将变为'idle'。

中止请求

如果资源在加载过程中params计算发生更改，则资源会中止挂起的加载操作。

您可以在ResourceLoaderParams中使用abortSignal来响应被中止的请求。例如，原生的fetch函数接受一个AbortSignal：

const userId: Signal<string> = getUserId();const userResource = resource({  params: () => ({id: userId()}),  loader: ({params, abortSignal}): Promise<User> => {    // fetch cancels any outstanding HTTP requests when the given `AbortSignal`    // indicates that the request has been aborted.    return fetch(`users/${params.id}`, {signal: abortSignal});  },});

有关使用 AbortSignal 取消请求的更多详细信息，请参见 MDN 上的 AbortSignal。

重新加载

你可以通过调用 reload 方法以编程方式触发资源的 loader。

const userId: Signal<string> = getUserId();const userResource = resource({  params: () => ({id: userId()}),  loader: ({params}) => fetchUser(params),});// ...userResource.reload();

资源状态

资源对象具有多个 Signal 属性，用于读取异步加载器的状态。

属性 描述
value 资源的最新值；如果未收到任何值，则为 undefined。
hasValue 资源是否具有值。
error 在运行资源加载器时遇到的最新错误；如果未发生错误，则为 undefined。
isLoading 资源加载器当前是否正在运行。
status 资源的特定ResourceStatus，如下所述。

status信号提供了一个特定的ResourceStatus，它使用字符串常量来描述资源的状态。

状态 value() 描述
'idle' undefined 资源没有有效的请求，并且加载器尚未运行。
'error' undefined 加载器遇到错误。
'loading' undefined 由于 params 值发生更改，加载器正在运行。
'reloading' 先前的值 加载器正在运行，原因是调用了资源的 reload 方法。
'resolved' 已求解的值 加载器已完成。
'local' 本地设置的值 资源的值已通过 .set() 或 .update() 在本地设置

你可以使用此状态信息有条件地显示用户界面元素，例如加载指示器和错误消息。

使用 httpResource 进行响应式数据获取

httpResource 是 HttpClient 的封装器，可将请求状态和响应作为信号提供给您。它通过 Angular HTTP 堆栈（包括拦截器）发出 HTTP 请求。
