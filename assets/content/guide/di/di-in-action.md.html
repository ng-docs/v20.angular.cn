依赖注入的应用

edit

本指南探讨了 Angular 中依赖注入的其他特性。

使用 @Inject 的自定义提供者

使用自定义提供者可以为隐式依赖（例如内置浏览器 API）提供具体的实现。 下面的示例使用一个 InjectionToken 来将浏览器的 localStorage API 作为依赖提供给 BrowserStorageService：

src/app/storage.service.ts

import { Inject, Injectable, InjectionToken } from '@angular/core';export const BROWSER_STORAGE = new InjectionToken<Storage>('Browser Storage', {  providedIn: 'root',  factory: () => localStorage});@Injectable({  providedIn: 'root'})export class BrowserStorageService {  public storage = inject(BROWSER_STORAGE);  get(key: string) {    return this.storage.getItem(key);  }  set(key: string, value: string) {    this.storage.setItem(key, value);  }}

factory 函数返回附着在浏览器 window 对象上的 localStorage 属性。 inject 函数使用令牌的实例初始化 storage 属性。

现在，可以在测试期间使用 localStorage 的模拟 API 来重写此自定义提供者，而无需与真实的浏览器 API 交互。

注入组件的 DOM 元素

尽管开发人员力求避免这种情况，但某些视觉效果和第三方工具还是需要直接访问 DOM。 因此，你可能需要访问组件的 DOM 元素。

Angular 通过使用 @Component 或 @Directive 的注入令牌 ElementRef 来暴露其底层元素：

import { Directive, ElementRef } from '@angular/core';@Directive({  selector: '[appHighlight]'})export class HighlightDirective {  private element = inject(ElementRef)  update() {    this.element.nativeElement.style.color = 'red';  }}

使用 forward reference 求解循环依赖

类的对象声明顺序在 TypeScript 中很重要。 在类被定义之前，你不能直接引用它。

这通常不是问题，特别是当你遵循推荐的每个文件一个类规则时。 但有时循环引用是不可避免的。 例如，当类“A”引用类“B”，而“B”又引用“A”时，其中一个必须先定义。

Angular 的 forwardRef() 函数创建一个间接引用，Angular 稍后可以解析该引用。

当一个类引用自身时，你会遇到类似的问题。 例如，在其 providers 数组中。 providers 数组是 @Component() 装饰器函数的属性，它必须出现在类定义之前。 你可以通过使用 forwardRef 来打破这种循环引用。

app.component.ts

providers: [  {    provide: PARENT_MENU_ITEM,    useExisting: forwardRef(() => MenuItem),  },],
