模板中的变量

edit

Angular 在模板中有两种类型的变量声明：局部模板变量和模板引用变量。

使用带有 @let 的本地模板变量

Angular 的 @let 语法允许你定义一个局部变量并在整个模板中复用它，类似于 JavaScript 的 let 语法。

使用 @let

使用 @let 来声明一个变量，其值基于模板表达式的结果。Angular 会自动使变量的值与给定的表达式保持同步，类似于绑定。

@let name = user.name;@let greeting = 'Hello, ' + name;@let data = data$ | async;@let pi = 3.14159;@let coordinates = {x: 50, y: 100};@let longExpression = 'Lorem ipsum dolor sit amet, consectetur adipiscing elit ' +                      'sed do eiusmod tempor incididunt ut labore et dolore magna ' +                      'Ut enim ad minim veniam...';

每个 @let 控制块只能声明一个变量。你不能在同一个控制块中用逗号声明多个变量。

引用 @let 的值

一旦你用 @let 声明了一个变量，你就可以在同一个模板中复用它：

@let user = user$ | async;@if (user) {  <h1>Hello, {{user.name}}</h1>  <user-avatar [photo]="user.photo"/>  <ul>    @for (snack of user.favoriteSnacks; track snack.id) {      <li>{{snack.name}}</li>    }  </ul>  <button (click)="update(user)">Update profile</button>}

可赋值性

@let 和 JavaScript 的 let 之间的一个主要区别是，@let 在声明后不能被重新赋值。但是，Angular 会自动使变量的值与给定的表达式保持同步。

@let value = 1;<!-- Invalid - This does not work! --><button (click)="value = value + 1">Increment the value</button>

变量作用域

@let 对象声明的作用域限定为当前视图及其后代。Angular 在组件边界以及模板可能包含动态内容（例如控制块、@defer 块或结构型指令）的任何位置创建新视图。

由于 @let 声明不会被提升，因此父视图或兄弟视图 无法 访问它们：

@let topLevel = value;<div>  @let insideDiv = value;</div>{{topLevel}} <!-- Valid -->{{insideDiv}} <!-- Valid -->@if (condition) {  {{topLevel + insideDiv}} <!-- Valid -->  @let nested = value;  @if (condition) {    {{topLevel + insideDiv + nested}} <!-- Valid -->  }}{{nested}} <!-- Error, not hoisted from @if -->

完整语法

@let 语法的正式定义如下：

  • @let 关键字。
  • 后跟一个或多个空格，不包括换行符。
  • 后跟一个有效的 JavaScript 名称和零个或多个空格。
  • 后跟 = 符号和零个或多个空格。
  • 后跟一个 Angular 表达式，该表达式可以是多行的。
  • 以 ; 符号结尾。

模板引用变量

模板引用变量为你提供了一种声明变量的方法，该变量引用模板中元素的值。

模板引用变量可以引用以下内容：

  • 模板中的 DOM 元素（包括 自定义元素）
  • 一个 Angular 组件或指令
  • 来自 ng-template 的 TemplateRef

你可以使用模板引用变量从同一模板的一部分读取信息到同一模板的另一部分。

声明模板引用变量

你可以在模板中的元素上声明一个变量，方法是添加一个以井号字符 (#) 开头的属性，后跟变量名。

<!-- Create a template reference variable named "taskInput", referring to the HTMLInputElement. --><input #taskInput placeholder="Enter task name">

为模板引用变量赋值

Angular 根据声明变量的元素为模板变量赋值。

如果你在 Angular 组件上声明变量，则该变量引用组件实例。

<!-- The `startDate` variable is assigned the instance of `MyDatepicker`. --><my-datepicker #startDate />

如果你在 <ng-template> 元素上声明变量，该变量将引用一个代表该模板的 TemplateRef 实例。有关更多信息，请参阅 结构型指令 中的 Angular 如何使用星号 * 语法。

<!-- The `myFragment` variable is assigned the `TemplateRef` instance corresponding to this template fragment. --><ng-template #myFragment>  <p>This is a template fragment</p></ng-template>

如果你在任何其他显示的元素上声明变量，则该变量引用 HTMLElement 实例。

<!-- The "taskInput" variable refers to the HTMLInputElement instance. --><input #taskInput placeholder="Enter task name">

为 Angular 指令赋值引用

Angular 指令可能具有 exportAs 属性，该属性定义了在模板中引用该指令的名称：

@Directive({  selector: '[dropZone]',  exportAs: 'dropZone',})export class DropZone { /* ... */ }

当你在元素上声明模板变量时，你可以通过指定此 exportAs 名称为该变量赋值一个指令实例：

<!-- The `firstZone` variable refers to the `DropZone` directive instance. --><section dropZone #firstZone="dropZone"> ... </section>

你不能引用未指定 exportAs 名称的指令。

将模板引用变量与查询一起使用

除了使用模板变量从同一模板的另一部分读取值之外，你还可以使用这种变量声明样式来“标记”一个元素，以用于组件和指令查询。

当你想在模板中查询特定元素时，你可以在该元素上声明一个模板变量，然后根据变量名查询该元素。

 <input #description value="Original description">
@Component({  /* ... */,  template: `<input #description value="Original description">`,})export class AppComponent {  // Query for the input element based on the template variable name.  @ViewChild('description') input: ElementRef | undefined;}

有关查询的更多信息，请参阅使用查询引用子组件。
