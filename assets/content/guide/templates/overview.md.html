模板语法

edit

在 Angular 中，模板是一段 HTML。 在模板中使用特殊的语法可以充分利用 Angular 的许多特性。

提示： 在深入学习本综合指南之前，请查看 Angular 的 Essentials。

每个 Angular 组件都有一个模板，用于定义组件渲染到页面上的 DOM。通过使用模板，Angular 能够在数据更改时自动保持页面最新。

模板通常位于 *.component.ts 文件的 template 属性或 *.component.html 文件中。要了解更多信息，请查看深入组件指南。

模板是如何工作的？

模板基于 HTML 语法，并具有其他特性，例如内置模板函数、数据绑定、事件监听、变量等等。

Angular 将模板编译成 JavaScript，以便建立对你的应用的内部理解。这样做的好处之一是 Angular 会自动将内置的渲染优化应用于你的应用。

与标准 HTML 的差异

模板和标准 HTML 语法之间的一些差异包括：

  • 模板源码中的注释不包含在渲染后的输出中
  • 组件和指令元素可以是自闭合的（例如，<UserProfile />）
  • 具有某些字符（即 []、() 等）的 Attribute 对 Angular 具有特殊含义。有关更多信息，请参阅绑定文档和添加事件监听器文档。
  • @ 字符对于 Angular 具有特殊含义，用于向模板添加动态行为，例如控制块。你可以通过将其转义为 HTML 实体代码（&commat; 或 &#64;）来包含字面量 @ 字符。
  • Angular 会忽略和折叠不必要的空白字符。有关更多详细信息，请参阅模板中的空白。
  • Angular 可能会向页面添加注释节点，作为动态内容的占位符，但开发人员可以忽略这些注释节点。

此外，虽然大多数 HTML 语法都是有效的模板语法，但 Angular 不支持模板中的 <script> 元素。有关更多信息，请参阅安全性页面。
