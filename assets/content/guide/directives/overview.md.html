内置指令

edit

指令是一些类，它们为 Angular 应用中的元素添加额外的行为。

使用 Angular 的内置指令来管理表单、列表、样式以及用户所见的内容。

Angular 指令的不同类型如下：

指令类型 详情
组件 与模板一起使用。这种类型的指令是最常见的指令类型。
属性型指令 更改元素、组件或其他指令的外观或行为。
结构型指令 通过添加和移除 DOM 元素来更改 DOM 布局。

本指南涵盖了内置的属性型指令。

内置属性型指令

属性型指令监听并修改其他 HTML 元素、Attribute、Property 和组件的行为。

最常见的属性型指令如下：

通用指令 详情
NgClass 添加和移除一组 CSS 类。
NgStyle 添加和移除一组 HTML 样式。
NgModel 向 HTML 表单元素添加双向数据绑定。

提示： 内置指令仅使用公共 API。它们无权访问其他指令无法访问的任何私有 API。

使用 NgClass 添加和移除类

使用 ngClass 同时添加或移除多个 CSS 类。

有帮助：要添加或移除单个类，请使用类绑定而不是NgClass。

在组件中导入 NgClass

要使用 NgClass，请将其添加到组件的 imports 列表中。

src/app/app.component.ts (NgClass 导入)

 isSpecial = true;  isUnchanged = true;  isActive = true;  nullCustomer: string | null = null;  currentCustomer = {    name: 'Laura',  };  item!: Item; // defined to demonstrate template context precedence  items: Item[] = [];  currentItem!: Item;  // trackBy change counting  itemsNoTrackByCount = 0;  itemsWithTrackByCount = 0;  itemsWithTrackByCountReset = 0;  itemIdIncrement = 1;  currentClasses: Record<string, boolean> = {}; 将 NgClass 与方法结合使用

  1. 要将NgClass与方法结合使用，请将该方法添加到组件类中。 在下面的示例中，setCurrentClasses() 使用一个对象来设置 currentClasses 属性，该对象根据另外三个组件属性的 true 或 false 状态来添加或移除三个类。

使用 NgStyle 设置内联样式

有帮助：要添加或移除单个样式，请使用样式绑定而不是NgStyle。

在组件中导入 NgStyle

要使用NgStyle，请将其添加到组件的 imports 列表中。

  1. 要使用 NgStyle，请向组件类添加一个方法。

在没有 DOM 元素的情况下宿主指令

Angular 的 <ng-container> 是一个分组元素，它不会干扰样式或布局，因为 Angular 不会将其放入 DOM 中。

当没有单个元素来宿主指令时，请使用 <ng-container>。
