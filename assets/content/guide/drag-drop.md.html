拖放

edit

概览

本页介绍拖放指令，它能让你快速创建拖放界面，支持以下功能：

  • 自由拖动
  • 创建可重新排序的可拖动元素列表
  • 在列表之间传输可拖动元素
  • 拖动动画
  • 沿轴或元素锁定可拖动元素
  • 添加自定义拖动柄
  • 拖动时添加预览
  • 添加自定义拖动占位符

有关完整的 API 参考，请参阅 Angular CDK 的拖放 API 参考页。

开始之前

CDK 安装

该 组件开发工具包 (CDK) 是一组用于构建组件的行为基元。要使用拖放指令，请先从 npm 安装 @angular/cdk。你可以使用 Angular CLI 在终端中执行此操作：

ng add @angular/cdk

导入拖放

要使用拖放，请在你的组件中导入所需的指令。

import {Component} from '@angular/core';import {CdkDrag} from '@angular/cdk/drag-drop';@Component({  selector: 'my-custom-component',  templateUrl: 'my-custom-component.html',  standalone: true,  imports: [CdkDrag],})export class DragDropExample {}

创建可拖动元素

你可以通过添加 cdkDrag 指令使任何元素可拖动。默认情况下，所有可拖动元素都支持自由拖动。

<div class="example-box" cdkDrag>  Drag me around</div>

创建可重新排序的可拖动元素列表

将 cdkDropList 指令添加到父元素，将可拖动元素分组到一个可重新排序的集合中。这定义了可拖动元素可以放置的位置。当元素移动时，拖放列表组中的可拖动元素会自动重新排列。

拖放指令不会更新你的数据模型。要更新数据模型，请监听 cdkDropListDropped 事件（在用户完成拖动后），然后手动更新数据模型。

app/app.component.html

<div cdkDropList class="example-list" (cdkDropListDropped)="drop($event)">  @for (movie of movies; track movie) {    <div class="example-box" cdkDrag>{{movie}}</div>  }</div>

在列表之间传输可拖动元素

cdkDropList 指令支持在连接的放置列表之间传输可拖动元素。有两种方法可以将一个或多个 cdkDropList 实例连接在一起：

  • 将 cdkDropListConnectedTo 属性设置为另一个放置列表。
  • 将元素包装在具有 cdkDropListGroup 属性的元素中。

cdkDropListConnectedTo 指令既可以与另一个 cdkDropList 的直接引用一起工作，也可以通过引用另一个放置容器的 ID 来工作。

<!-- This is valid --><div cdkDropList #listOne="cdkDropList" [cdkDropListConnectedTo]="[listTwo]"></div><div cdkDropList #listTwo="cdkDropList" [cdkDropListConnectedTo]="[listOne]"></div><!-- This is valid as well --><div cdkDropList id="list-one" [cdkDropListConnectedTo]="['list-two']"></div><div cdkDropList id="list-two" [cdkDropListConnectedTo]="['list-one']"></div>

选择性拖动

默认情况下，用户可以将 cdkDrag 元素从一个容器移动到另一个连接的容器。要更精细地控制哪些元素可以放置到容器中，请使用 cdkDropListEnterPredicate。每当可拖动元素进入新容器时，Angular 都会调用该谓词。根据谓词返回 true 或 false，该项可能允许或不允许进入新容器。

app/app.component.html

<div class="example-container">  <h2>Available numbers</h2>  <div    id="all"    cdkDropList    [cdkDropListData]="all"    cdkDropListConnectedTo="even"    class="example-list"    (cdkDropListDropped)="drop($event)"    [cdkDropListEnterPredicate]="noReturnPredicate">    @for (number of all; track number) {      <div          class="example-box"          [cdkDragData]="number"          cdkDrag>{{number}}</div>    }  </div></div><div class="example-container">  <h2>Even numbers</h2>  <div    id="even"    cdkDropList    [cdkDropListData]="even"    cdkDropListConnectedTo="all"    class="example-list"    (cdkDropListDropped)="drop($event)"    [cdkDropListEnterPredicate]="evenPredicate">    @for (number of even; track number) {      <div          class="example-box"          cdkDrag          [cdkDragData]="number">{{number}}</div>    }  </div></div>

附加数据

你可以通过分别设置 cdkDragData 或 cdkDropListData 来将任意数据与 cdkDrag 和 cdkDropList 相关联。你可以绑定到这两个指令触发的事件，这些事件将包含此数据，从而让你轻松识别拖动或放置交互的来源。

@for (list of lists; track list) {  <div cdkDropList [cdkDropListData]="list" (cdkDropListDropped)="drop($event)">    @for (item of list; track item) {      <div cdkDrag [cdkDragData]="item"></div>    }  </div>}

自定义拖动预览

当 cdkDrag 元素被拖动时，预览元素会变得可见。默认情况下，预览是原始元素的克隆，位于用户光标旁边。

要自定义预览，请通过 *cdkDragPreview 提供自定义模板。自定义预览的大小将与原始拖动元素的大小不匹配，因为不会对元素的内容做出假设。要使拖动预览的大小与元素匹配，请将 true 传递给 matchSize 输入。

克隆元素会移除其 id 属性，以避免页面上出现多个具有相同 id 的元素。这将导致针对该 id 的任何 CSS 都不会被应用。

app/app.component.html

<div cdkDropList class="example-list" (cdkDropListDropped)="drop($event)">  @for (movie of movies; track movie) {    <div class="example-box" cdkDrag>      {{movie.title}}      <img *cdkDragPreview [src]="movie.poster" [alt]="movie.title">    </div>  }</div>

自定义拖动插入点

默认情况下，Angular 将 cdkDrag 预览插入到页面的 <body> 中，以避免定位和溢出问题。在某些情况下，这可能不理想，因为预览将不会应用其继承的样式。

你可以使用 cdkDrag 上的 cdkDragPreviewContainer 输入来更改 Angular 插入预览的位置。可能的值为：

值 描述 优点 缺点

禁用排序

在某些情况下，可拖动元素可以从一个 cdkDropList 拖动到另一个，但是用户不应该能够在源列表中对它们进行排序。在这些情况下，添加 cdkDropListSortingDisabled 属性以防止 cdkDropList 中的可拖动元素进行排序。如果被拖动的元素未被拖动到新的有效位置，这将保留其在源列表中的初始位置。

列表间复制项

默认情况下，当一个项从一个列表拖动到另一个列表时，它会从其原始列表中移出。但是，您可以配置指令来复制该项，将原始项保留在源列表中。

要启用复制，您可以设置 cdkDropListHasAnchor 输入。这会告诉 cdkDropList 创建一个“锚点”元素，该元素保留在原始容器中，并且不会随项移动。如果用户将项移回原始容器，锚点将自动移除。可以通过定位 .cdk-drag-anchor CSS 类来设置锚点元素的样式。

将 cdkDropListHasAnchor 与 cdkDropListSortingDisabled 结合使用，可以构建一个列表，用户可以从中复制项，而无需重新排序源列表（例如，产品列表和购物车）。

<div class="example-container">  <h2>Products</h2>  <div    cdkDropList    [cdkDropListData]="products"    [cdkDropListConnectedTo]="[cartList]"    cdkDropListSortingDisabled    cdkDropListHasAnchor    class="example-list"  >    @for (product of products; track $index) {      <div class="example-box" cdkDrag [cdkDragData]="product">{{product}}</div>    }  </div></div><div class="example-container">  <h2>Shopping cart</h2>  <div    cdkDropList    #cartList="cdkDropList"    [cdkDropListData]="cart"    class="example-list"    (cdkDropListDropped)="drop($event)"  >    @for (product of cart; track $index) {      <div class="example-box" cdkDrag>{{product}}</div>    }  </div></div>

自定义动画

拖放支持以下两者的动画：

  • 在列表内对可拖动元素进行排序
  • 将可拖动元素从用户放置它的位置移动到列表内的最终位置

要设置动画，请定义一个针对 transform 属性的 CSS 过渡。以下类可用于动画：

CSS 类名 添加过渡的结果
.cdk-drag 在拖动元素排序时为其添加动画。
.cdk-drag-animating 为可拖动元素从其放置位置到 cdkDropList 内最终位置的动画。

仅当拖动操作停止时，此 CSS 类才会应用于 cdkDrag 元素。

样式

cdkDrag 和 cdkDropList 指令仅应用功能所需的必要样式。应用程序可以通过定位这些指定的 CSS 类来定制其样式。

CSS 类名 描述
.cdk-drop-list cdkDropList 容器元素的选择器。
.cdk-drag cdkDrag 元素的选择器。
.cdk-drag-disabled 禁用 cdkDrag 元素的选择器。
.cdk-drag-handle cdkDragHandle 的宿主元素的选择器。
.cdk-drag-preview 拖动预览元素的选择器。这是当用户在可排序列表中拖动元素时出现在光标旁边的元素。

除非通过 *cdkDragPreview 自定义模板，否则该元素的外观与正在拖动的元素完全相同。
.cdk-drag-placeholder 拖动占位符元素的选择器。这是在拖动操作结束时，可拖动元素将被拖动到的位置显示的元素。

除非使用 cdkDragPlaceholder 指令进行自定义，否则此元素的外观与正在排序的元素完全相同。
.cdk-drop-list-dragging cdkDropList 容器元素的选择器，该元素当前正在拖动一个可拖动元素。
.cdk-drop-list-disabled 禁用的 cdkDropList 容器元素的选择器。
.cdk-drop-list-receiving cdkDropList 容器元素的选择器，该元素具有一个可拖动元素，它可以从当前正在拖动的已连接的放置列表中接收该元素。
.cdk-drag-anchor 启用 cdkDropListHasAnchor 时创建的锚点元素的选择器。此元素指示被拖动项的起始位置。

在可滚动容器中拖动

如果您的可拖动项位于可滚动容器中（例如，带有 overflow: auto 的 div），则自动滚动将不起作用，除非该可滚动容器具有 cdkScrollable 指令。没有它，CDK 无法在拖动操作期间检测或控制容器的滚动行为。

与其他组件的集成

CDK 的拖放功能可以与不同的组件集成。常见用例包括可排序的 MatTable 组件和可排序的 MatTabGroup 组件。
