Angular 动画简介

edit

重要提示：@angular/animations 包现已弃用。Angular 团队建议所有新代码都使用原生 CSS 配合 animate.enter 和 animate.leave 来实现动画。请访问新的 enter 和 leave 动画指南了解更多信息。另请参阅 从 Angular 的动画包迁移，了解如何开始将纯 CSS 动画迁移到您的应用中。

动画提供了运动的错觉：HTML 元素在一段时间内改变样式。 精心设计的动画可以让你的应用更有趣、更直接易用，但它们不仅仅是装饰性的。 动画可以通过多种方式改进你的应用和用户体验：

  • 如果没有动画，网页过渡可能会显得突兀和刺耳
  • 动效极大地增强了用户体验，因此动画让用户有机会察觉到应用对其操作的响应
  • 好的动画可以直观地将用户的注意力吸引到需要的地方

通常，动画涉及随时间推移的多种样式变换。 HTML 元素可以移动、改变颜色、放大或缩小、淡入淡出或滑出页面。 这些变化可以同时或按顺序发生。你可以控制每次变换的时序。

Angular 的动画体系构建于 CSS 功能之上，这意味着你可以为浏览器认为可动画化的任何 Property 制作动画。 这包括位置、尺寸、变换、颜色、边框等等。 W3C 在其 CSS Transitions 页面上维护着一个可动画化的 Property 列表。

关于本指南

本指南介绍了基本的 Angular 动画特性，帮助你开始向你的项目添加 Angular 动画。

开始使用

用于动画的主要 Angular 模块是 @angular/animations 和 @angular/platform-browser。

要开始向你的项目添加 Angular 动画，请导入特定于动画的模块以及标准的 Angular 功能。

  1. 启用动画模块

    从 @angular/platform-browser/animations/async 导入 provideAnimationsAsync 并将其添加到 bootstrapApplication 函数调用的 providers 列表中。

    启用动画

    1bootstrapApplication(AppComponent, {2  providers: [3    provideAnimationsAsync(),4  ]5});

    如果你需要在你的应用中立即执行动画

    如果您需要在应用程序加载时立即发生动画， 您将需要切换到急切加载的动画模块。请改从 @angular/platform-browser/animations 导入 provideAnimations， 并在 bootstrapApplication 函数调用中使用 provideAnimations代替 provideAnimationsAsync。

    对于基于 NgModule 的应用程序，导入 BrowserAnimationsModule，它将动画功能引入您的 Angular 根应用程序模块。

    src/app/app.module.ts

    import {NgModule} from '@angular/core';import {BrowserModule} from '@angular/platform-browser';import {BrowserAnimationsModule} from '@angular/platform-browser/animations';@NgModule({  imports: [BrowserModule, BrowserAnimationsModule],  declarations: [],  bootstrap: [],})export class AppModule {}

  2. 将动画函数导入到组件文件中

    如果你计划在组件文件中使用特定的动画函数，请从 @angular/animations 导入这些函数。

    src/app/app.component.ts

    import {Component, HostBinding, inject} from '@angular/core';import {  trigger,  state,  style,  animate,  transition,  // ...} from '@angular/animations';import {ChildrenOutletContexts, RouterLink, RouterOutlet} from '@angular/router';import {slideInAnimation} from './animations';@Component({  selector: 'app-root',  templateUrl: 'app.component.html',  styleUrls: ['app.component.css'],  imports: [RouterLink, RouterOutlet],  animations: [    slideInAnimation,    // animation triggers go here  ],})export class AppComponent {  @HostBinding('@.disabled')  public animationsDisabled = false;  private contexts = inject(ChildrenOutletContexts);  getRouteAnimationData() {    return this.contexts.getContext('primary')?.route?.snapshot?.data?.['animation'];  }  toggleAnimations() {    this.animationsDisabled = !this.animationsDisabled;  }}

    在本指南的末尾查看所有可用的动画函数。

  3. 添加动画元数据 Property

    在组件文件中，在 @Component() 装饰器内添加一个名为 animations: 的元数据属性。 您将定义动画的触发器放在 animations 元数据属性中。

    src/app/app.component.ts

    import {Component, HostBinding, inject} from '@angular/core';import {  trigger,  state,  style,  animate,  transition,  // ...} from '@angular/animations';import {ChildrenOutletContexts, RouterLink, RouterOutlet} from '@angular/router';import {slideInAnimation} from './animations';@Component({  selector: 'app-root',  templateUrl: 'app.component.html',  styleUrls: ['app.component.css'],  imports: [RouterLink, RouterOutlet],  animations: [    slideInAnimation,    // animation triggers go here  ],})export class AppComponent {  @HostBinding('@.disabled')  public animationsDisabled = false;  private contexts = inject(ChildrenOutletContexts);  getRouteAnimationData() {    return this.contexts.getContext('primary')?.route?.snapshot?.data?.['animation'];  }  toggleAnimations() {    this.animationsDisabled = !this.animationsDisabled;  }}

为过渡添加动画

让我们为一个过渡添加动画，该过渡将单个 HTML 元素从一种状态更改为另一种状态。 例如，你可以指定按钮根据用户的最后操作显示 Open 或 Closed。 当按钮处于 open 状态时，它是可见的并且是黄色的。 当它处于 closed 状态时，它是半透明的并且是蓝色的。

在 HTML 中，这些属性使用普通的 CSS 样式（如颜色和不透明度）来设置。 在 Angular 中，使用 style() 函数指定一组用于动画的 CSS 样式。 将一组样式收集到动画状态中，并为该状态命名，例如 open 或 closed。

提示： 让我们创建一个新的 open-close 组件，用简单的过渡来制作动画。

在终端中运行以下命令以生成组件：

ng g component open-close

这将在 src/app/open-close.component.ts 创建组件。

动画状态和样式

使用 Angular 的 state() 函数来定义在每次过渡结束时调用的不同状态。 此函数接受两个参数： 一个唯一的名称，如 open 或 closed，以及一个 style() 函数。

使用 style() 函数定义一组要与给定状态名称关联的样式。 对于包含连字符的样式属性，您必须使用驼峰式命名，例如 backgroundColor，或者将它们括在引号中，例如 'background-color'。

让我们看看 Angular 的 state() 函数如何与 style⁣­(⁠) 函数配合使用来设置 CSS 样式属性。 在此代码片段中，为状态同时设置了多个样式属性。 在 open 状态下，按钮的高度为 200 像素，不透明度为 1，背景色为黄色。

过渡和时序

在 Angular 中，你可以设置多种样式而无需任何动画。 但是，如果没有进一步的细化，按钮会立即转换，没有淡入淡出、没有收缩或其他可见的指示器表明正在发生更改。

为了使变化不那么突兀，您需要定义一个动画过渡来指定在一段时间内在一个状态和另一个状态之间发生的更改。 transition() 函数接受两个参数： 第一个参数接受一个定义两个过渡状态之间方向的表达式，第二个参数接受一个或一系列 animate() 步骤。

使用 animate() 函数来定义过渡的长度、延迟和缓动，并指定在过渡进行时定义样式的样式函数。 使用 animate() 函数来定义 keyframes() 函数以实现多步动画。 这些定义放置在 animate() 函数的第二个参数中。

动画元数据：duration、delay 和 easing

animate() 函数（transition 函数的第二个参数）接受 timings 和 styles 输入参数。

timings 参数接受数字或由三部分定义的字符串。

animate (duration)

or

animate ('duration delay easing')

第一部分 duration 是必要的。 duration 可以用毫秒表示为一个不带引号的数字，或者用秒表示为带引号和时间说明符的字符串。 例如，十分之一秒的 duration 可以表示为如下形式：

  • 以纯数字形式表示，单位为毫秒： 100

  • 以字符串形式表示，单位为毫秒： '100ms'

  • 以字符串形式表示，单位为秒： '0.1s'

第二个参数 delay 具有与 duration 相同的语法。 例如：

  • 等待 100 毫秒，然后运行 200 毫秒：'0.2s 100ms'

第三个参数 easing 控制动画在其运行时如何加速和减速。 例如，ease-in 使动画开始时缓慢，并随着进程的进行而加速。

  • 等待 100 毫秒，运行 200 毫秒。 使用减速曲线快速启动，然后缓慢减速至静止点： '0.2s 100ms ease-out'

  • 运行 200 毫秒，没有延迟。 使用标准曲线，开始时慢速，中间加速，然后在结尾处缓慢减速： '0.2s ease-in-out'

  • 立即开始，运行 200 毫秒。 使用加速曲线，开始时慢速，结束时达到全速： '0.2s ease-in'

提示： 有关缓动曲线的常规信息，请参阅 Material Design 网站上关于自然缓动曲线的主题。

本范例提供了一个从 open 到 closed 的状态过渡，状态之间的过渡时间为 1 秒。

src/app/open-close.component.ts

import {Component, input} from '@angular/core';import {trigger, transition, state, animate, style, AnimationEvent} from '@angular/animations';@Component({  selector: 'app-open-close',  animations: [    trigger('openClose', [      // ...      state(        'open',        style({          height: '200px',          opacity: 1,          backgroundColor: 'yellow',        }),      ),      state(        'closed',        style({          height: '100px',          opacity: 0.8,          backgroundColor: 'blue',        }),      ),      transition('open => closed', [animate('1s')]),      transition('closed => open', [animate('0.5s')]),      transition('* => closed', [animate('1s')]),      transition('* => open', [animate('0.5s')]),      transition('open <=> closed', [animate('0.5s')]),      transition('* => open', [animate('1s', style({opacity: '*'}))]),      transition('* => *', [animate('1s')]),    ]),  ],  templateUrl: 'open-close.component.html',  styleUrls: ['open-close.component.css'],})export class OpenCloseComponent {  logging = input(false);  isOpen = true;  toggle() {    this.isOpen = !this.isOpen;  }  onAnimationEvent(event: AnimationEvent) {    if (!this.logging) {      return;    }    // openClose is trigger name in this example    console.warn(`Animation Trigger: ${event.triggerName}`);    // phaseName is "start" or "done"    console.warn(`Phase: ${event.phaseName}`);    // in our example, totalTime is 1000 (number of milliseconds in a second)    console.warn(`Total time: ${event.totalTime}`);    // in our example, fromState is either "open" or "closed"    console.warn(`From: ${event.fromState}`);    // in our example, toState either "open" or "closed"    console.warn(`To: ${event.toState}`);    // the HTML element itself, the button in this case    console.warn(`Element: ${event.element}`);  }}

在前面的代码片段中，=> 操作符表示单向过渡，而 <=> 表示双向过渡。 在过渡中，animate() 指定过渡所需的时间。 在这种情况下，从 open 到 closed 的状态更改需要 1 秒，此处表示为 1s。

有帮助：关于在 state 和 transition 函数中使用样式的其他说明。

  • 使用 state() 定义应用于每个过渡结束时的样式，它们在动画完成后会持续存在
  • 使用 transition() 定义中间样式，这些样式在动画期间产生运动的错觉
  • 当动画被禁用时，可以跳过 transition() 样式，但不能跳过 state() 样式
  • 在同一个 transition() 参数中包含多个状态对：
    transition( 'on => off, off => void' )

触发动画

动画需要一个触发器，以便知道何时开始。 trigger() 函数收集状态和过渡，并为动画命名，以便您可以将其附加到 HTML 模板中的触发元素上。

trigger() 函数描述了要监视更改的属性名称。 当发生更改时，触发器会启动其定义中包含的操作。 这些操作可以是过渡或其他函数，我们稍后会看到。

在本范例中，我们将触发器命名为 openClose，并将其附着到 button 元素。 该触发器描述了 open 和 closed 状态，以及两个过渡的时序。

有帮助：在每次 trigger() 函数调用中，一个元素在任何给定时间只能处于一种状态。 但是，可以同时激活多个触发器。

定义动画并将它们附着到 HTML 模板

动画在控制要设置动画的 HTML 元素所在的组件的元数据中定义。 将定义动画的代码放在 @Component() 装饰器内的 animations: 属性下。

src/app/open-close.component.ts

为组件定义动画触发器后，通过将触发器名称括在方括号中并在其前面加上 @ 符号，将其附着到该组件模板中的元素。 然后，您可以使用标准的 Angular 属性（Property）绑定语法将触发器绑定到模板表达式，如下所示，其中 triggerName 是触发器的名称，expression 求值结果为已定义的动画状态。

<div [@triggerName]="expression">…</div>;

当表达式值更改为新状态时，动画将被执行或触发。

以下代码片段将触发器绑定到 isOpen 属性（Property）的值。

src/app/open-close.component.html

<nav>  <button type="button" (click)="toggle()">Toggle Open/Close</button></nav><div [@openClose]="isOpen ? 'open' : 'closed'" class="open-close-container">  <p>The box is now {{ isOpen ? 'Open' : 'Closed' }}!</p></div>

在本范例中，当 isOpen 表达式求值结果为 open 或 closed 的已定义状态时，它会通知触发器 openClose 状态已更改。 然后，由 openClose 代码来处理状态更改并启动状态更改动画。

对于进入或离开页面的元素（从 DOM 插入或删除），您可以使动画有条件地执行。 例如，将 *ngIf 与 HTML 模板中的动画触发器一起使用。

有帮助：在组件文件中，将定义动画的触发器设置为 @Component() 装饰器中 animations: 属性的值。

在 HTML 模板文件中，使用触发器名称将定义的动画附着到要动画化的 HTML 元素。

动画 API 概览

@angular/animations 模块提供的函数式 API 提供了一种特定于领域的语言 (DSL)，用于在 Angular 应用中创建和控制动画。 有关核心函数和相关数据结构的完整列表和语法详细信息，请参阅 API 参考。

函数名称 功能
trigger() 启动动画并充当所有其他动画函数调用的容器。HTML 模板绑定到 triggerName。使用第一个参数声明唯一的触发器名称。使用数组语法。
style() 定义一个或多个要在动画中使用的 CSS 样式。控制动画期间 HTML 元素的视觉外观。使用对象语法。
state() 创建一组命名的 CSS 样式，这些样式应在成功过渡到给定状态时应用。然后可以在其他动画函数中按名称引用该状态。
animate() 指定过渡的时序信息。delay 和 easing 的可选值。可以包含 style() 调用。
transition() 定义两个命名状态之间的动画序列。使用数组语法。
keyframes() 允许在指定的时间间隔内按顺序在样式之间进行更改。在 animate() 中使用。可以在每个 keyframe() 中包含多个 style() 调用。使用数组语法。
group() 指定一组动画步骤（内部动画）以并行运行。仅当所有内部动画步骤完成后，动画才会继续。在 sequence() 或 transition() 中使用。
query() 在当前元素中查找一个或多个内部 HTML 元素。
sequence() 指定按顺序逐个运行的动画步骤列表。
stagger() 错开多个元素的动画开始时间。
animation() 生成一个可重用的动画，可以从其他地方调用。与 useAnimation() 一起使用。
useAnimation() 激活一个可重用的动画。与 animation() 一起使用。
animateChild() 允许子组件上的动画与父组件在同一时间范围内运行。
