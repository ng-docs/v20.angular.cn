复杂动画序列

edit

重要：@angular/animations 包现已弃用。Angular 团队建议所有新代码的动画使用原生 CSS 的 animate.enter 和 animate.leave。请访问新的 enter 和 leave 动画指南了解更多信息。另请参阅 从 Angular 的 Animations 包迁移，了解如何在你的应用中开始迁移到纯 CSS 动画。

到目前为止，我们学习了单个 HTML 元素的简单动画。 Angular 还允许你为协调的序列制作动画，例如当元素进入和离开页面时，为整个网格或元素列表制作动画。 你可以选择并行运行多个动画，或者顺序运行离散动画，一个接一个地运行。

控制复杂动画序列的函数包括：

函数 详情
query() 查找一个或多个内部 HTML 元素。
stagger() 将级联延迟应用于多个元素的动画。
group() 并行运行多个动画步骤。
sequence() 一个接一个地运行动画步骤。

query() 函数

大多数复杂动画依赖于 query() 函数来查找子元素并为它们应用动画，基本示例包括：

范例 详情
query() 后跟 animate() 用于查询简单的 HTML 元素并直接对其应用动画。
query() 后跟 animateChild() 用于查询子元素，这些子元素本身已应用了动画元数据并触发此类动画（否则将被当前/父元素的动画阻止）。

query() 的第一个参数是 css 选择器 字符串，其中还可以包含以下 Angular 特定的令牌：

标记 详情
:enter
:leave		 用于进入/离开的元素。
:animating 用于当前正在制作动画的元素。
@*
@triggerName		 用于具有任何触发器或特定触发器的元素。
:self 正在制作动画的元素本身。

进入和离开元素

并非所有子元素实际上都被视为进入/离开；有时，这可能违反直觉并且令人困惑。请参阅 query api 文档以获取更多信息。

你还可以在动画示例（在动画 介绍部分 中介绍）的“查询”选项卡下看到此内容的说明。

使用 query() 和 stagger() 函数为多个元素制作动画

通过 query() 查询子元素后，stagger() 函数允许你定义动画的每个查询项之间的时序间隔，从而为元素设置动画并使其之间产生延迟。

以下示例演示了如何使用 query() 和 stagger() 函数为列表（英雄列表）设置动画，按顺序添加每个英雄，并带有轻微的延迟，从上到下。

  • 使用 query() 查找符合特定条件的进入页面的元素

  • 对于这些元素中的每一个，使用 style() 为元素设置相同的初始样式。 使其透明并使用 transform 将其移出位置，以便它可以滑入到位。

  • 使用 stagger() 将每个动画延迟 30 毫秒

  • 使用自定义定义的缓动曲线为屏幕上的每个元素制作 0.5 秒的动画，同时使其淡入并取消变换

src/app/hero-list-page.component.ts

import {Component, HostBinding, OnInit} from '@angular/core';import {trigger, transition, animate, style, query, stagger} from '@angular/animations';import {HEROES} from './mock-heroes';import {Hero} from './hero';@Component({  selector: 'app-hero-list-page',  templateUrl: 'hero-list-page.component.html',  styleUrls: ['hero-list-page.component.css'],  animations: [    trigger('pageAnimations', [      transition(':enter', [        query('.hero', [          style({opacity: 0, transform: 'translateY(-100px)'}),          stagger(30, [            animate('500ms cubic-bezier(0.35, 0, 0.25, 1)', style({opacity: 1, transform: 'none'})),          ]),        ]),      ]),    ]),    trigger('filterAnimation', [      transition(':enter, * => 0, * => -1', []),      transition(':increment', [        query(          ':enter',          [            style({opacity: 0, width: 0}),            stagger(50, [animate('300ms ease-out', style({opacity: 1, width: '*'}))]),          ],          {optional: true},        ),      ]),      transition(':decrement', [        query(':leave', [stagger(50, [animate('300ms ease-out', style({opacity: 0, width: 0}))])]),      ]),    ]),  ],})export class HeroListPageComponent implements OnInit {  @HostBinding('@pageAnimations')  public animatePage = true;  heroesTotal = -1;  get heroes() {    return this._heroes;  }  private _heroes: Hero[] = [];  ngOnInit() {    this._heroes = HEROES;  }  updateCriteria(criteria: string) {    criteria = criteria ? criteria.trim() : '';    this._heroes = HEROES.filter((hero) =>      hero.name.toLowerCase().includes(criteria.toLowerCase()),    );    const newTotal = this.heroes.length;    if (this.heroesTotal !== newTotal) {      this.heroesTotal = newTotal;    } else if (!criteria) {      this.heroesTotal = -1;    }  }}

使用 group() 函数的并行动画

你已经看到了如何在连续的动画之间添加延迟。 但你可能也想配置并行发生的动画。 例如，你可能想为同一元素的两个 CSS 属性设置动画，但为每个属性使用不同的 easing 函数。 为此，你可以使用动画 group() 函数。

有帮助： group() 函数用于对动画步骤进行分组，而不是对动画元素进行分组。

下面的示例在 :enter 和 :leave 上都使用了 group()，并使用了两种不同的时序配置，从而将两个独立的动画并行应用于同一元素。

src/app/hero-list-groups.component.ts（节选）

import {Component, output, input} from '@angular/core';import {trigger, state, style, animate, transition, group} from '@angular/animations';import {Hero} from './hero';@Component({  selector: 'app-hero-list-groups',  template: `    <ul class="heroes">      @for (hero of heroes(); track hero) {        <li [@flyInOut]="'in'">          <button class="inner" type="button" (click)="removeHero(hero.id)">            <span class="badge">{{ hero.id }}</span>            <span class="name">{{ hero.name }}</span>          </button>        </li>      }    </ul>  `,  styleUrls: ['./hero-list-page.component.css'],  animations: [    trigger('flyInOut', [      state(        'in',        style({          width: '*',          transform: 'translateX(0)',          opacity: 1,        }),      ),      transition(':enter', [        style({width: 10, transform: 'translateX(50px)', opacity: 0}),        group([          animate(            '0.3s 0.1s ease',            style({              transform: 'translateX(0)',              width: '*',            }),          ),          animate(            '0.3s ease',            style({              opacity: 1,            }),          ),        ]),      ]),      transition(':leave', [        group([          animate(            '0.3s ease',            style({              transform: 'translateX(50px)',              width: 10,            }),          ),          animate(            '0.3s 0.2s ease',            style({              opacity: 0,            }),          ),        ]),      ]),    ]),  ],})export class HeroListGroupsComponent {  readonly heroes = input<Hero[]>([]);  readonly remove = output<number>();  removeHero(id: number) {    this.remove.emit(id);  }}

顺序动画与并行动画

复杂的动画可以同时进行许多事情。 但如果你想创建一个涉及多个动画接连发生的动画呢？之前你使用了 group() 来同时并行运行多个动画。

第二个名为 sequence() 的函数可以让你将这些相同的动画一个接一个地运行。 在 sequence() 中，动画步骤由 style() 或 animate() 函数调用组成。

  • 使用 style() 立即应用提供的样式数据。
  • 使用 animate() 在给定的时间间隔内应用样式数据。

筛选器动画范例

看看范例页面上的另一个动画。 在“筛选器/交错”选项卡下，在搜索英雄文本框中输入一些文本，例如 Magnet 或 tornado。

筛选器在你键入时实时工作。 当你键入每个新字母时，元素会离开页面，并且筛选器会逐渐变得更严格。 当你删除筛选器框中的每个字母时，英雄列表会逐渐重新进入页面。

组件装饰器中的 filterAnimation 包含三个过渡。

src/app/hero-list-page.component.ts

import {Component, HostBinding, OnInit} from '@angular/core';import {trigger, transition, animate, style, query, stagger} from '@angular/animations';import {HEROES} from './mock-heroes';import {Hero} from './hero';@Component({  selector: 'app-hero-list-page',  templateUrl: 'hero-list-page.component.html',  styleUrls: ['hero-list-page.component.css'],  animations: [    trigger('pageAnimations', [      transition(':enter', [        query('.hero', [          style({opacity: 0, transform: 'translateY(-100px)'}),          stagger(30, [            animate('500ms cubic-bezier(0.35, 0, 0.25, 1)', style({opacity: 1, transform: 'none'})),          ]),        ]),      ]),    ]),    trigger('filterAnimation', [      transition(':enter, * => 0, * => -1', []),      transition(':increment', [        query(          ':enter',          [            style({opacity: 0, width: 0}),            stagger(50, [animate('300ms ease-out', style({opacity: 1, width: '*'}))]),          ],          {optional: true},        ),      ]),      transition(':decrement', [        query(':leave', [stagger(50, [animate('300ms ease-out', style({opacity: 0, width: 0}))])]),      ]),    ]),  ],})export class HeroListPageComponent implements OnInit {  @HostBinding('@pageAnimations')  public animatePage = true;  heroesTotal = -1;  get heroes() {    return this._heroes;  }  private _heroes: Hero[] = [];  ngOnInit() {    this._heroes = HEROES;  }  updateCriteria(criteria: string) {    criteria = criteria ? criteria.trim() : '';    this._heroes = HEROES.filter((hero) =>      hero.name.toLowerCase().includes(criteria.toLowerCase()),    );    const newTotal = this.heroes.length;    if (this.heroesTotal !== newTotal) {      this.heroesTotal = newTotal;    } else if (!criteria) {      this.heroesTotal = -1;    }  }}

此范例中的代码执行以下任务：

  • 当用户首次打开或导航到此页面时跳过动画（筛选器动画会窄化已存在的内容，因此它仅适用于 DOM 中已存在的元素）
  • 根据搜索输入的值筛选英雄

对于每次更改：

  • 通过将其不透明度和宽度设置为 0 来隐藏离开 DOM 的元素

  • 在 300 毫秒内，为一个进入 DOM 的元素添加动画效果。 在动画期间，该元素呈现其默认宽度和不透明度。

  • 如果有多个元素进入或离开 DOM，则会错开每个动画，从页面顶部开始，每个元素之间延迟 50 毫秒

为重新排序的列表的条目添加动画效果

尽管 Angular 开箱即用地正确地为 *ngFor 列表条目设置动画，但如果它们的顺序发生变化，它将无法做到这一点。 这是因为它会丢失对哪个元素是哪个元素的跟踪，导致动画中断。 帮助 Angular 跟踪此类元素的唯一方法是将 TrackByFunction 赋给 NgForOf 指令。 这确保了 Angular 始终知道哪个元素是哪个元素，从而使其能够始终将正确的动画应用于正确的元素。

重要： 如果你需要为 *ngFor 列表的条目设置动画，并且这些条目的顺序有可能在运行时发生变化，请始终使用 TrackByFunction。

动画和组件视图封装

Angular 动画基于组件的 DOM 结构，并且不直接考虑 视图封装，这意味着使用 ViewEncapsulation.Emulated 的组件表现得就像使用 ViewEncapsulation.None 一样（ViewEncapsulation.ShadowDom 的表现不同，我们稍后会讨论）。

例如，如果 query() 函数（你将在动画指南的其余部分中更多地看到它）应用于使用模拟视图封装的组件树的顶部，那么这样的查询将能够识别（并因此为）树的任何深度的 DOM 元素设置动画。

另一方面，ViewEncapsulation.ShadowDom 通过将 DOM 元素“隐藏”在 ShadowRoot 元素中来更改组件的 DOM 结构。这种 DOM 操作确实会阻止某些动画实现正常工作，因为它依赖于简单的 DOM 结构并且不考虑 ShadowRoot 元素。因此，建议避免将动画应用于包含使用 ShadowDom 视图封装的组件的视图。

动画序列摘要

Angular 用于为多个元素设置动画的函数以 query() 开始，用于查找内部元素；例如，收集 <div> 中的所有图像。 其余函数 stagger()、group() 和 sequence() 应用级联或允许你控制如何应用多个动画步骤。
