构建模板驱动型表单

edit

本教程向你展示如何创建一个模板驱动型表单。表单中的控件元素绑定到具有输入验证的数据属性（Property）。输入验证有助于维护数据完整性和样式，从而改善用户体验。

模板驱动型表单使用双向数据绑定，以便在模板中进行更改时更新组件中的数据模型，反之亦然。

模板驱动型表单 vs 响应式表单

Angular 支持交互式表单的两种设计方法。模板驱动型表单允许你在 Angular 模板中使用特定于表单的指令。响应式表单提供了一种模型驱动的方法来构建表单。

模板驱动型表单是小型或简单表单的绝佳选择，而响应式表单更具延展性，更适合复杂表单。有关这两种方法的比较，请参阅选择一种方法

你可以使用 Angular 模板构建几乎任何类型的表单 —— 登录表单、联系表单以及几乎任何业务表单。 你可以创造性地布置控件并将它们绑定到对象模型中的数据。 你可以指定验证规则并显示验证错误，有条件地允许来自特定控件的输入，触发内置的可视反馈等等。

目标

本教程教你如何执行以下操作：

  • 使用组件和模板构建 Angular 表单
  • 使用 ngModel 创建双向数据绑定，以读取和写入输入控件值
  • 使用特殊的 CSS 类提供视觉反馈，以跟踪控件的状态
  • 向用户显示验证错误，并根据表单状态有条件地允许来自表单控件的输入
  • 使用模板引用变量在 HTML 元素之间共享信息

构建模板驱动型表单

模板驱动表单依赖于在 FormsModule 中定义的指令。

指令 详情
NgModel 协调附着的表单元素中的值更改与数据模型中的更改，从而允许你使用输入验证和错误处理来响应用户输入。
NgForm 创建一个顶级的 FormGroup 实例并将其绑定到 <form> 元素以跟踪聚合表单值和验证状态。一旦导入 FormsModule，此指令默认会在所有 <form> 标签上激活。您无需添加特殊选择器。
NgModelGroup 创建并绑定一个 FormGroup 实例到 DOM 元素。

步骤概览

在本教程中，你将通过以下步骤将示例表单绑定到数据并处理用户输入。

  1. 构建基本表单。
    • 定义示例数据模型
    • 包含所需的底层架构，例如 FormsModule
  2. 使用 ngModel 指令和双向数据绑定语法将表单控件绑定到数据属性（Property）。
    • 了解 ngModel 如何使用 CSS 类报告控件状态
    • 命名控件，使它们可以被 ngModel 访问
  3. 使用 ngModel 跟踪输入的有效性和控件状态。
    • 添加自定义 CSS 以提供关于状态的视觉反馈
    • 显示和隐藏验证错误消息
  4. 通过添加到模型数据来响应原生 HTML 按钮点击事件。
  5. 使用表单的 ngSubmit 输出属性处理表单提交。
    • 禁用 Submit 按钮，直到表单有效
    • 提交后，将完成的表单换成页面上的不同内容

将输入控件绑定到数据属性（Property）

总结

本页讨论的 Angular 表单利用了以下框架特性来提供对数据修改、验证等的支持。

  • 一个 Angular HTML 表单模板
  • 一个带有 @Component 装饰器的表单组件类
  • 通过绑定到 NgForm.ngSubmit 事件属性来处理表单提交
  • 模板引用变量，例如 #actorForm 和 #name
  • 用于双向数据绑定的 [(ngModel)] 语法
  • 使用 name 属性进行验证和表单元素变更跟踪
  • 输入控件上引用变量的 valid 属性表明控件是否有效，或者是否应该显示错误消息
  • 通过绑定到 NgForm 有效性来控制 Submit 按钮的启用状态
  • 自定义 CSS 类，用于向用户提供关于无效控件的可视化反馈
