响应式表单

edit

响应式表单提供了一种模型驱动的方式来处理表单输入，这些输入的值会随时间而变化。 本指南将向你展示如何创建和更新基本的表单控件，逐步学习如何在组中使用多个控件、验证表单值，以及创建动态表单，在动态表单中，你可以在运行时添加或删除控件。

响应式表单概览

响应式表单使用显式且不可变的方式来管理表单在给定时间点的状态。 表单状态的每次更改都会返回一个新状态，从而在更改之间保持模型的完整性。 响应式表单围绕可观察者流构建，其中表单输入和值作为输入值流提供，可以同步访问。

响应式表单还为测试提供了一条直接的途径，因为你可以确信你的数据在被请求时是一致且可预测的。 流的任何使用者都可以安全地操纵这些数据。

响应式表单与模板驱动表单的区别在于其独特的方式。 响应式表单通过可观察者操作符提供对数据模型的同步访问和不变性，并通过可观察者流进行变更跟踪。

模板驱动表单允许直接访问来修改模板中的数据，但不如响应式表单那样显式，因为它们依赖于嵌入在模板中的指令以及可变数据来异步跟踪更改。 有关这两种范例之间详细的比较，请参阅表单概览。

添加基本的表单控件

使用表单控件有三个步骤。

  1. 生成一个新的组件并注册响应式表单模块。此模块声明了使用响应式表单所需的响应式表单指令。
  2. 实例化一个新的FormControl。
  3. 在模板中注册FormControl。

然后，你可以通过将组件添加到模板来显示表单。

以下示例展示了如何添加单个表单控件。 在该示例中，用户在输入字段中输入他们的姓名，捕获该输入值，并显示表单控件元素的当前值。

  1. 生成一个新的组件并导入 ReactiveFormsModule

    使用 CLI 命令 ng generate component 在您的项目中生成一个组件，并从 @angular/forms 包导入ReactiveFormsModule，然后将其添加到您组件的 imports 数组中。

    src/app/name-editor/name-editor.component.ts (节选)

    import {Component} from '@angular/core';import {FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-name-editor',  templateUrl: './name-editor.component.html',  styleUrls: ['./name-editor.component.css'],  imports: [ReactiveFormsModule],})export class NameEditorComponent {  name = new FormControl('');  updateName() {    this.name.setValue('Nancy');  }}

  2. 声明一个 FormControl 实例

    使用FormControl的构造函数来设置其初始值，在本例中为空字符串。通过在组件类中创建这些控件，您可以立即访问表单输入的 istate 以进行监听、更新和验证。

    src/app/name-editor/name-editor.component.ts

    import {Component} from '@angular/core';import {FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-name-editor',  templateUrl: './name-editor.component.html',  styleUrls: ['./name-editor.component.css'],  imports: [ReactiveFormsModule],})export class NameEditorComponent {  name = new FormControl('');  updateName() {    this.name.setValue('Nancy');  }}

  3. 在模板中注册控件

    在组件类中创建控件后，您必须将其与模板中的表单控件元素关联起来。使用FormControlDirective提供的formControl绑定来更新模板中的表单控件，该绑定也包含在ReactiveFormsModule中。

    src/app/name-editor/name-editor.component.html

    <label for="name">Name: </label><input id="name" type="text" [formControl]="name"><p>Value: {{ name.value }}</p><button type="button" (click)="updateName()">Update Name</button>

    使用模板绑定语法，表单控件现在已注册到模板中的 name 输入元素。表单控件和 DOM 元素相互通信：视图反映模型中的更改，模型反映视图中的更改。

  4. 显示组件

    当 <app-name-editor> 组件添加到模板时，分配给 name 属性的FormControl会被显示出来。

    src/app/app.component.html (名称编辑器)

    <h1>Reactive Forms</h1><app-name-editor /><app-profile-editor />

显示表单控件值

你可以通过以下方式显示该值。

  • 通过 valueChanges 可观察者，您可以在模板中使用AsyncPipe或在组件类中使用 subscribe() 方法来监听表单值中的更改
  • 使用 value 属性，它可以为你提供当前值的快照

以下示例向你展示如何在模板中使用插值来显示当前值。

src/app/name-editor/name-editor.component.html (控件值)

<label for="name">Name: </label><input id="name" type="text" [formControl]="name"><p>Value: {{ name.value }}</p><button type="button" (click)="updateName()">Update Name</button>

当你更新表单控件元素时，显示的值也会随之更改。

响应式表单通过每个实例提供的属性和方法来访问有关给定控件的信息。 这些底层AbstractControl类的属性和方法用于控制表单状态，并在处理输入验证时确定何时显示消息。

在API 参考中阅读有关其他FormControl属性和方法的信息。

替换表单控件值

响应式表单具有以编程方式更改控件值的方法，这使你可以灵活地在没有用户交互的情况下更新值。 表单控件实例提供了一个 setValue() 方法，该方法更新表单控件的值，并根据控件的结构验证所提供值的结构。 例如，当从后端 API 或服务检索表单数据时，使用 setValue() 方法将控件更新为其新值，从而完全替换旧值。

以下示例向组件类添加了一个方法，以使用 setValue() 方法将控件的值更新为 Nancy。

src/app/name-editor/name-editor.component.ts (更新值)

import {Component} from '@angular/core';import {FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-name-editor',  templateUrl: './name-editor.component.html',  styleUrls: ['./name-editor.component.css'],  imports: [ReactiveFormsModule],})export class NameEditorComponent {  name = new FormControl('');  updateName() {    this.name.setValue('Nancy');  }}

使用按钮更新模板以模拟名称更新。 当你单击 Update Name 按钮时，在表单控件元素中输入的值将反映为其当前值。

src/app/name-editor/name-editor.component.html (更新值)

<label for="name">Name: </label><input id="name" type="text" [formControl]="name"><p>Value: {{ name.value }}</p><button type="button" (click)="updateName()">Update Name</button>

表单模型是控件的事实之源，因此当你单击按钮时，输入的的值将在组件类中更改，从而覆盖其当前值。

提示： 在此示例中，你正在使用单个控件。 当将 setValue() 方法与表单组或表单数组实例一起使用时，该值需要与组或数组的结构匹配。

对表单控件进行分组

表单通常包含多个相关的控件。 响应式表单提供了两种将多个相关控件分组到单个输入表单中的方法。

表单组 详情
表单组 定义一个具有一组固定控件的表单，您可以将它们一起管理。本节讨论表单组基础知识。您还可以 嵌套表单组 来创建更复杂的表单。
表单数组 定义了一个动态表单，你可以在运行时添加和删除控件。你还可以嵌套表单数组以创建更复杂的表单。有关此选项的更多信息，请参阅创建动态表单。

正如表单控件实例使你可以控制单个输入字段一样，表单组实例跟踪一组表单控件实例（例如，表单）的表单状态。 在创建表单组时，表单组实例中的每个控件都按名称进行跟踪。 以下示例展示了如何在单个组中管理多个表单控件实例。

生成一个 ProfileEditor 组件，并从 @angular/forms 包导入 FormGroup 和 FormControl 类。

ng generate component ProfileEditor

    创建嵌套表单组

    表单组可以接受单个表单控件实例和其他表单组实例作为子项。 这使得组合复杂的表单模型更易于维护和逻辑分组。

    在构建复杂表单时，在较小的部分中管理信息的不同区域更容易。 使用嵌套的表单组实例可让你将大型表单组分解为更小、更易于管理的表单组。

    要创建更复杂的表单，请使用以下步骤。

    1. 创建一个嵌套组。
    2. 在模板中对嵌套表单进行分组。

    某些类型的信息自然会归入同一组。 姓名和地址是此类嵌套组的典型示例，并在以下范例中使用。

    要在 `profileForm` 中创建嵌套组，请将嵌套的 `address` 元素添加到表单组实例。

    src/app/profile-editor/profile-editor.component.ts (嵌套表单组)

    import {Component} from '@angular/core';import {FormGroup, FormControl, ReactiveFormsModule} from '@angular/forms';@Component({  selector: 'app-profile-editor',  templateUrl: './profile-editor.component.html',  styleUrls: ['./profile-editor.component.css'],  imports: [ReactiveFormsModule],})export class ProfileEditorComponent {  profileForm = new FormGroup({    firstName: new FormControl(''),    lastName: new FormControl(''),    address: new FormGroup({      street: new FormControl(''),      city: new FormControl(''),      state: new FormControl(''),      zip: new FormControl(''),    }),  });  updateProfile() {    this.profileForm.patchValue({      firstName: 'Nancy',      address: {        street: '123 Drew Street',      },    });  }}

更新数据模型的各个部分

当更新包含多个控件的表单组实例的值时，你可能只想更新模型的各个部分。 本节介绍如何更新表单控件数据模型的特定部分。

有两种更新模型值的方法：

方法 详情
setValue() 为单个控件设置新值。setValue() 方法严格遵守表单组的结构，并替换控件的整个值。
patchValue() 替换对象中定义的在表单模型中已更改的任何属性。

setValue() 方法的严格检查有助于捕获复杂表单中的嵌套错误，而 patchValue() 在这些错误上会静默失败。

在 ProfileEditorComponent 中，使用 updateProfile 方法和以下范例来更新用户的名字和街道地址。

使用 FormBuilder 服务生成控件

手动创建表单控件实例在处理多个表单时可能会变得重复。 FormBuilder 服务提供了方便的方法来生成控件。

使用以下步骤来利用此服务。

  1. 导入 FormBuilder 类。
  2. 注入 FormBuilder 服务。
  3. 生成表单内容。

以下范例展示了如何重构 ProfileEditor 组件以使用表单构建器服务来创建表单控件和表单组实例。

  1. 导入 FormBuilder 类

    从 @angular/forms 包导入 FormBuilder 类。

创建动态表单

FormArray 是 FormGroup 的一种替代方案，用于管理任意数量的未命名控件。 与表单组实例一样，您可以动态地从表单数组实例中插入和删除控件，并且表单数组实例的值和验证状态是从其子控件计算得出的。 但是，您不需要按名称为每个控件定义键，因此如果您事先不知道子值的数量，这是一个很好的选择。

要定义动态表单，请执行以下步骤。

  1. 导入 FormArray 类。
  2. 定义一个 FormArray 控件。
  3. 使用 getter 方法访问 FormArray 控件。
  4. 在模板中显示表单数组。

以下范例向你展示了如何在 ProfileEditor 中管理 aliases 数组。

  1. 导入 FormArray 类

    从 @angular/forms 导入 FormArray 类以用于类型信息。 FormBuilder 服务已准备好创建 FormArray 实例。

响应式表单 API 概览

下表列出了用于创建和管理响应式表单控件的基类和服务。 有关完整的语法详细信息，请参阅 Forms 包的 API 参考文档。

类

类 详情
AbstractControl 具体表单控件类 FormControl、FormGroup 和 FormArray 的抽象基类。它提供了它们的通用行为和属性。
FormControl 管理单个表单控件的值和有效性状态。它对应于 HTML 表单控件，例如 <input> 或 <select>。
FormGroup 管理一组 AbstractControl 实例的值和有效性状态。组的属性包括其子控件。组件中的顶层表单是 FormGroup。
FormArray 管理一组数字索引的 AbstractControl 实例的值和有效性状态。
FormBuilder 一个可注入的服务，提供用于创建控件实例的工厂方法。
FormRecord 跟踪一组 FormControl 实例的值和有效性状态，每个实例都具有相同的值类型。

指令

指令 详情
FormControlDirective 将独立的 FormControl 实例同步到表单控件元素。
FormControlName 通过名称将现有 FormGroup 实例中的 FormControl 同步到表单控件元素。
FormGroupDirective 将现有的 FormGroup 实例同步到 DOM 元素。
FormGroupName 将嵌套的 FormGroup 实例同步到 DOM 元素。
FormArrayName 将嵌套的 FormArray 实例同步到 DOM 元素。
