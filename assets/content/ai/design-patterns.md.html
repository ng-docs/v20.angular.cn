AI SDK 和信号 API 的设计模式

edit

与 AI 和大型语言模型 (LLM) API 交互会带来独特的挑战，例如管理异步操作、处理流式数据以及为可能缓慢或不可靠的网络请求设计响应式用户体验。Angular 的 信号和 resource API 提供了强大的工具来优雅地解决这些问题。

使用信号触发请求

处理用户提供的提示时，一种常见模式是将用户实时输入与其触发 API 调用的提交值分开。

  1. 在用户键入时，将用户的原始输入存储在一个信号中
  2. 当用户提交时（例如，通过单击按钮），使用第一个信号的内容更新第二个信号。
  3. 在你的 resource 的 params 字段中使用第二个信号。

此设置可确保资源的 loader 函数仅在用户显式提交提示时运行，而不是在每次击键时运行。你可以在 loader 字段中使用其他信号参数，例如 sessionId 或 userId（这对于创建持久的 LLM 会话很有用）。这样，请求始终使用这些参数的当前值，而不会重新触发在 loader 字段中定义的异步函数。

许多 AI SDK 都提供用于发出 API 调用的辅助方法。例如，Genkit 客户端库公开了一个用于调用 Genkit 流的 runFlow 方法，你可以从资源的 loader 调用它。对于其他 API，你可以使用 httpResource。

下面的示例展示了一个 resource，它获取 AI 生成故事的部分内容。仅当 storyInput 信号更改时，才会触发 loader。

// A resource that fetches three parts of an AI generated storystoryResource = resource({  // The default value to use before the first request or on error  defaultValue: DEFAULT_STORY,  // The loader is re-triggered when this signal changes  params: () => this.storyInput(),  // The async function to fetch data  loader: ({params}): Promise<StoryData> => {    // The params value is the current value of the storyInput signal    const url = this.endpoint();    return runFlow({ url, input: {      userInput: params,      sessionId: this.storyService.sessionId() // Read from another signal    }});  }});

为模板准备 LLM 数据

你可以配置 LLM API 以返回结构化数据。将你的 resource 进行强类型化以匹配 LLM 的预期输出，可以提供更好的类型安全性和编辑器自动补全。

要管理从资源派生的状态，请使用 computed 信号或 linkedSignal。因为 linkedSignal 提供对先前值的访问，所以它可以满足各种与 AI 相关的用例，包括

  • 构建聊天历史记录
  • 在 LLM 生成内容时保留或自定义模板显示的数据

在下面的示例中，storyParts 是一个 linkedSignal，它将从 storyResource 返回的最新故事部分追加到现有故事部分数组中。

storyParts = linkedSignal<string[], string[]>({  // The source signal that triggers the computation  source: () => this.storyResource.value().storyParts,  // The computation function  computation: (newStoryParts, previous) => {    // Get the previous value of this linkedSignal, or an empty array    const existingStoryParts = previous?.value || [];    // Return a new array with the old and new parts    return [...existingStoryParts, ...newStoryParts];  }});

性能和用户体验

LLM API 可能比传统的、更确定的 API 慢且更容易出错。你可以使用多种 Angular 功能来构建高性能且用户友好的界面。

  • 作用域加载：将 resource 放在直接使用数据的组件中。这有助于限制变更检测周期（尤其是在无区域应用程序中）并防止阻塞应用程序的其他部分。如果数据需要在多个组件之间共享，请从服务中提供 resource。
  • SSR 和水合：使用服务器端渲染 (SSR) 和增量水合来快速渲染初始页面内容。你可以为 AI 生成的内容显示一个占位符，并推迟获取数据，直到组件在客户端水合。
  • 加载状态：使用 resource 的 LOADING 状态在请求进行中时显示一个指示器，例如加载动画。此状态涵盖初始加载和重新加载。
  • 错误处理和重试：使用 resource 的 reload() 方法作为用户重试失败请求的简单方式，在依赖 AI 生成内容时可能更为普遍。

下面的示例演示了如何创建一个响应式 UI，以动态显示具有加载和重试功能的 AI 生成图像。

<!-- Display a loading spinner while the LLM generates the image -->@if (imgResource.isLoading()) {  <div class="img-placeholder">    <mat-spinner [diameter]="50" />  </div><!-- Dynamically populates the src attribute with the generated image URL -->} @else if (imgResource.hasValue()) {  <img [src]="imgResource.value()" /><!-- Provides a retry option if the request fails  -->} @else {  <div class="img-placeholder" (click)="imgResource.reload()">    <mat-icon fontIcon="refresh" />      <p>Failed to load image. Click to retry.</p>  </div>}

AI 模式实践：流式聊天响应

接口通常在响应数据到达时逐步显示基于 LLM 的 API 的部分结果。Angular 的资源 API 提供了流式响应的能力来支持此类模式。 resource 的 stream 属性接受一个异步函数，你可以使用它随时间推移将更新应用于信号值。正在更新的信号代表正在流式传输的数据。

characters = resource({  stream: async () => {    const data = signal<ResourceStreamItem<string>>({value: ''});    // Calls a Genkit streaming flow using the streamFlow method    // expose by the Genkit client SDK    const response = streamFlow({      url: '/streamCharacters',      input: 10    });    (async () => {      for await (const chunk of response.stream) {        data.update((prev) => {          if ('value' in prev) {            return { value: `${prev.value} ${chunk}` };          } else {            return { error: chunk as unknown as Error };          }        });      }    })();    return data;  }});

characters 成员是异步更新的，可以在模板中显示。

@if (characters.isLoading()) {  <p>Loading...</p>} @else if (characters.hasValue()) {  <p>{{characters.value()}}</p>} @else {  <p>{{characters.error()}}</p>}

在服务器端，例如在 server.ts 中，定义的端点将要流式传输的数据发送到客户端。以下代码将 Gemini 与 Genkit 框架结合使用，但此技术也适用于支持从 LLM 流式传输响应的其他 API：

import { startFlowServer } from '@genkit-ai/express';import { genkit } from "genkit/beta";import { googleAI, gemini20Flash } from "@genkit-ai/googleai";const ai = genkit({ plugins: [googleAI()] });export const streamCharacters = ai.defineFlow({    name: 'streamCharacters',    inputSchema: z.number(),    outputSchema: z.string(),    streamSchema: z.string(),  },  async (count, { sendChunk }) => {    const { response, stream } = ai.generateStream({    model: gemini20Flash,    config: {      temperature: 1,    },    prompt: `Generate ${count} different RPG game characters.`,  });  (async () => {    for await (const chunk of stream) {      sendChunk(chunk.content[0].text!);    }  })();  return (await response).text;});startFlowServer({  flows: [streamCharacters],});
