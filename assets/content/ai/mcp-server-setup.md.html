Angular CLI MCP 服务器设置

edit

Angular CLI 包含一个实验性的 模型上下文协议 (MCP) 服务器，使开发环境中的 AI 助手能够与 Angular CLI 进行交互。我们已支持由 CLI 驱动的代码生成、添加包等功能。

可用工具

Angular CLI MCP 服务器提供多种工具来协助你的开发工作流。默认情况下，以下工具已启用：

名称 描述 local-only read-only
get_best_practices 检索 Angular 最佳实践指南。本指南对于确保所有代码都遵循现代标准至关重要，包括独立组件、类型化表单和现代控制流。 ✅ ✅
list_projects 列出 Angular 工作区中定义的所有应用程序和库的名称。它会读取 angular.json 配置文件来识别项目。 ✅ ✅
search_documentation 搜索官方 Angular 文档，网址为 https://angular.dev。此工具应用于回答有关 Angular 的任何问题，例如 API、教程和最佳实践。 ❌ ✅

开始使用

要开始使用，请在终端中运行以下命令：

ng mcp

当从交互式终端运行时，此命令会显示有关如何配置主机环境以使用 MCP 服务器的说明。以下各节提供了几种常用编辑器和工具的示例配置。

Cursor

在项目根目录中创建一个名为 .cursor/mcp.json 的文件，并添加以下配置。你也可以在全局的 ~/.cursor/mcp.json 中进行配置。

{  "mcpServers": {    "angular-cli": {      "command": "npx",      "args": ["-y", "@angular/cli", "mcp"]    }  }}

Firebase Studio

在项目根目录中创建一个名为 .idx/mcp.json 的文件，并添加以下配置：

{  "mcpServers": {    "angular-cli": {      "command": "npx",      "args": ["-y", "@angular/cli", "mcp"]    }  }}

Gemini CLI

在项目根目录中创建一个名为 .gemini/settings.json 的文件，并添加以下配置：

{  "mcpServers": {    "angular-cli": {      "command": "npx",      "args": ["-y", "@angular/cli", "mcp"]    }  }}

JetBrains IDE

在 JetBrains IDE（如 IntelliJ IDEA 或 WebStorm）中，安装 JetBrains AI Assistant 插件后，转到 Settings | Tools | AI Assistant | Model Context Protocol (MCP)。添加一个新服务器并选择 As JSON。粘贴以下配置，该配置不使用顶级属性来表示服务器列表。

{  "name": "Angular CLI",  "command": "npx",  "args": [    "-y",    "@angular/cli",    "mcp"  ]}

VS Code

在项目根目录中，创建一个名为 .vscode/mcp.json 的文件并添加以下配置。请注意 servers 属性的使用。

{  "servers": {    "angular-cli": {      "command": "npx",      "args": ["-y", "@angular/cli", "mcp"]    }  }}

其他 IDE

对于其他 IDE，请查阅你的 IDE 文档以了解 MCP 配置文件（通常是 mcp.json）的正确位置。配置应包含以下代码片段。

{  "mcpServers": {    "angular-cli": {      "command": "npx",      "args": ["-y", "@angular/cli", "mcp"]    }  }}

命令选项

mcp 命令可以通过以下选项进行配置，这些选项作为参数传递在你的 IDE 的 MCP 配置中：

选项 类型 描述 默认值
--read-only boolean 仅注册不更改项目的工具。你的编辑器或代码代理仍可能执行编辑。 false
--local-only boolean 仅注册不需要互联网连接的工具。你的编辑器或代码代理仍可能通过网络发送数据。 false

例如，要在 VS Code 中以只读模式运行服务器，你需要像这样更新你的 mcp.json：

{  "servers": {    "angular-cli": {      "command": "npx",      "args": ["-y", "@angular/cli", "mcp", "--read-only"]    }  }}
