@loading、@error 和 @placeholder 块

edit

可延迟视图允许你定义在不同加载状态下显示的内容。

@placeholder 默认情况下，@defer 控制块在触发之前不渲染任何内容。@placeholder 是一个可选的控制块，用于声明在延迟内容加载之前显示的内容。加载完成后，Angular 会用延迟内容替换占位符。虽然此控制块是可选的，但 Angular 团队建议始终包含占位符。 在完整的可延迟视图文档中了解更多信息
@loading 这个可选的控制块允许你声明在任何延迟依赖项加载期间显示的内容。
@error 此控制块允许你声明在延迟加载失败时显示的内容。

上述所有子控制块的内容都是急性加载的。此外，某些特性需要一个 @placeholder 控制块。

在本活动中，你将学习如何使用 @loading、@error 和 @placeholder 控制块来管理可延迟视图的状态。

  1. 添加 @placeholder 控制块

    在你的 app.ts 中，向 @defer 控制块添加一个 @placeholder 控制块。

    @defer {  <article-comments />} @placeholder {  <p>Placeholder for comments</p>}

  2. 配置 @placeholder 控制块

    @placeholder 控制块接受一个可选参数，用于指定此占位符应显示的 minimum 最短时间。此 minimum 参数以毫秒 (ms) 或秒 (s) 的时间增量指定。此参数的存在是为了防止在快速获取延迟依赖项的情况下，占位符内容快速闪烁。

    @defer {  <article-comments />} @placeholder (minimum 1s) {  <p>Placeholder for comments</p>}

  3. 添加 @loading 控制块

    接下来，将一个 @loading 控制块添加到组件模板中。

    @loading 控制块接受两个可选参数：

    • minimum：此控制块应显示的最短时间
    • after：加载开始后，在显示加载模板之前等待的时间量

    这两个参数都以毫秒 (ms) 或秒 (s) 的时间增量指定。

    更新 app.ts，向 @loading 控制块添加一个 minimum 参数为 1s 以及 after 参数值为 500ms 的 @loading 控制块。

    @defer {  <article-comments />} @placeholder (minimum 1s) {  <p>Placeholder for comments</p>} @loading (minimum 1s; after 500ms) {  <p>Loading comments...</p>}

    注意：此范例使用两个参数，以 ; 字符分隔。

  4. 添加 @error 控制块

    最后，将一个 @error 控制块添加到 @defer 控制块。

    @defer {  <article-comments />} @placeholder (minimum 1s) {  <p>Placeholder for comments</p>} @loading (minimum 1s; after 500ms) {  <p>Loading comments...</p>} @error {  <p>Failed to load comments</p>}
