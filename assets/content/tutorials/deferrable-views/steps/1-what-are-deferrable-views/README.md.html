什么是可延迟视图？

edit

一个完全渲染的 Angular 页面可能包含许多不同的组件、指令和管道。虽然页面的某些部分应该立即显示给用户，但可能有些部分可以等待稍后显示。 Angular 的 可延迟视图 ，使用 @defer 语法，可以帮助你加速你的应用，它会告诉 Angular 延迟下载页面中不需要立即显示的部分的 JavaScript。

在本次实践中，你将学习如何使用可延迟视图来延迟加载组件模板的某个部分。

  1. 向模板的某个部分添加一个 @defer 控制块

    在您的 app.ts 中，将 article-comments 组件包装在 @defer 控制块中以延迟加载它。

    @defer {  <article-comments />}

    默认情况下，当浏览器空闲时，@defer 会加载 article-comments 组件。

    在浏览器的开发者控制台中，你可以看到 article-comments-component 惰性加载块文件是单独加载的（具体的文件名可能因运行而异）：

    Initial chunk files | Names                      |  Raw sizechunk-NNSQHFIE.js   | -                          | 769.00 kB | main.js             | main                       | 229.25 kB | Lazy chunk files    | Names                      |  Raw sizechunk-T5UYXUSI.js   | article-comments-component |   1.84 kB |

做得好！你已经学习了可延迟视图的基础知识。
