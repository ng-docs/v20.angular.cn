路由概览

edit

对于大多数应用来说，都会出现应用需要多个页面的情况。当这种情况不可避免地发生时，路由就会成为关乎用户体验性能的重要部分。

注意：在深入指南中了解有关路由的更多信息。

在本活动中，你将学习如何建立和配置你的应用以使用 Angular 路由器。

  1. 创建一个 app.routes.ts 文件

    在 app.routes.ts 文件中，进行如下更改：

    1. 从 @angular/router 包导入 Routes。
    2. 导出一个名为 routes 的常量，类型为 Routes，并将其值赋为 []。
    import {Routes} from '@angular/router';export const routes: Routes = [];

  2. 将路由添加到提供者

    在 app.config.ts 中，通过以下步骤将应用配置为使用 Angular 路由器：

    1. 从 @angular/router 导入 provideRouter 函数。
    2. 从 ./app.routes.ts 中导入 routes。
    3. 在 providers 数组中调用 provideRouter 函数，并将 routes 作为参数传入。
    import {ApplicationConfig} from '@angular/core';import {provideRouter} from '@angular/router';import {routes} from './app.routes';export const appConfig: ApplicationConfig = {  providers: [provideRouter(routes)],};

  3. 在组件中导入 RouterOutlet

    最后，为了确保您的应用已准备好使用 Angular 路由，您需要告诉应用您期望路由在哪里显示所需内容。通过使用来自 @angular/router 的 RouterOutlet 指令来完成此操作。

    通过添加 <router-outlet /> 来更新 App 的模板

    import {RouterOutlet} from '@angular/router';@Component({  ...  template: `    <nav>      <a href="/">Home</a>      |      <a href="/user">User</a>    </nav>    <router-outlet />  `,  imports: [RouterOutlet],})export class App {}

你的应用现在已设置为使用 Angular 路由器。干得漂亮！🙌

保持势头，继续学习下一步，为我们的应用定义路由。
