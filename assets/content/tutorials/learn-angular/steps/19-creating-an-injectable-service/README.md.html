创建可注入的服务

edit

Angular 中的依赖注入 (DI) 是该框架最强大的特性之一。可以将依赖注入视为 Angular 在运行时提供你的应用所需的资源的能力。依赖项可以是服务或其他一些资源。

注意：在核心指南中了解更多关于依赖注入的信息。

在此活动中，您将学习如何创建可注入服务。

使用服务的一种方式是充当与数据和 API 交互的方式。为了使服务可复用，你应该将逻辑保留在服务中，并在需要时在整个应用中共享它。

要使服务有资格被 DI 系统注入，请使用 @Injectable 装饰器。例如：

@Injectable({  providedIn: 'root'})class UserService {  // methods to retrieve and return data}

@Injectable 装饰器通知 DI 系统 UserService 可在类中请求。providedIn 设置此资源可用的范围。目前，足以理解 providedIn: 'root' 意味着 UserService 可供整个应用程序使用。
