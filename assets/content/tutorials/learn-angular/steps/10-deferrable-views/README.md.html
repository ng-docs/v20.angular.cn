可延迟视图

edit

有时在应用开发中，你最终会得到许多需要在你的应用中引用的组件，但由于各种原因，其中一些组件不需要立即加载。

也许它们位于可见区域下方，或者是直到稍后才交互的重型组件。 在这种情况下，我们可以使用可延迟视图稍后加载其中一些资源。

注意：在深入指南中了解有关使用 @defer 进行延迟加载的更多信息。

在此活动中，你将学习如何使用可延迟视图来延迟加载组件模板的某个部分。

  1. 在 comments 组件周围添加一个 @defer 控制块

    在你的应用中，博客文章页面在文章详情之后有一个评论组件。

    用 @defer 控制块包裹评论组件以延迟加载它。

    @defer {  <comments />}

    上面的代码是如何使用基本的 @defer 控制块的示例。 默认情况下，当浏览器空闲时，@defer 将加载 comments 组件。

  2. 添加一个占位符

    向 @defer 控制块添加一个 @placeholder 控制块。 @placeholder 控制块是你放置 html 的位置，它将在延迟加载开始之前显示。 @placeholder 控制块中的内容是急性加载的。

    @defer {  <comments />} @placeholder {  <p>Future comments</p>}

  3. 添加一个加载块

    向 @defer 控制块添加一个 @loading 控制块。 @loading 控制块是你放置 html 的位置，它将在当正在主动获取延迟内容但尚未完成时显示。 @loading 控制块中的内容是急性加载的。

    @defer {  <comments />} @placeholder {  <p>Future comments</p>} @loading {  <p>Loading comments...</p>}

  4. 添加最短持续时间

    @placeholder 和 @loading 部分都有可选参数，以防止在快速加载时发生闪烁。 @placeholder 具有 minimum，而 @loading 具有 minimum 和 after。 向 @loading 控制块添加 minimum 持续时间，以便它至少渲染 2 秒。

    @defer {  <comments />} @placeholder {  <p>Future comments</p>} @loading (minimum 2s) {  <p>Loading comments...</p>}

  5. 添加一个视口触发器

    可延迟视图有许多触发器选项。 添加一个视口触发器，以便内容在进入视口后延迟加载。

    @defer (on viewport) {  <comments />}

  6. 添加内容
