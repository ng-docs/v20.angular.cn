创建和更新你的第一个信号

edit

欢迎来到 Angular 信号教程！信号是 Angular 的响应式原始类型，它提供了一种管理状态并在状态更改时自动更新 UI 的方法。

在此活动中，你将学习如何：

  • 使用 signal() 函数创建你的第一个信号
  • 在模板中显示其值
  • 使用 set() 和 update() 方法更新信号值

让我们用信号构建一个交互式用户状态系统！

  1. 导入信号函数

    在组件文件的顶部从 @angular/core 导入 signal 函数。

    import {Component, signal, ChangeDetectionStrategy} from '@angular/core';

  2. 在你的组件中创建一个信号

    向你的组件类添加一个 userStatus 信号，并将其初始化为 'offline' 值。

    @Component({  /* Config omitted */})export class App {  userStatus = signal<'online' | 'offline'>('offline');}

  3. 在模板中显示信号值

    通过以下方式更新状态指示器以显示当前用户状态：

    1. 使用 [class]="userStatus()" 将信号绑定到类属性
    2. 通过将 ??? 替换为 {{ userStatus() }} 来显示状态文本
    <!-- Update from: --><div class="status-indicator offline">  <span class="status-dot"></span>  Status: ???</div><!-- To: --><div class="status-indicator" [class]="userStatus()">  <span class="status-dot"></span>  Status: {{ userStatus() }}</div>

    请注意，我们如何使用括号调用信号 userStatus() 来读取其值。

  4. 添加方法来更新信号

    向你的组件添加使用 set() 方法更改用户状态的方法。

    goOnline() {  this.userStatus.set('online');}goOffline() {  this.userStatus.set('offline');}

    set() 方法会用一个新值完全替换信号的值。

  5. 连接控件按钮

    按钮已在模板中。现在通过添加以下内容将它们连接到你的方法：

    1. 带有 (click) 的点击处理程序
    2. 当已处于该状态时，使用 [disabled] 设置禁用状态
    <!-- Add bindings to the existing buttons: --><button (click)="goOnline()" [disabled]="userStatus() === 'online'">  Go Online</button><button (click)="goOffline()" [disabled]="userStatus() === 'offline'">  Go Offline</button>

  6. 使用 update() 添加一个切换方法

    添加一个 toggleStatus() 方法，该方法使用 update() 方法在在线和离线之间切换。

    toggleStatus() {  this.userStatus.update(current => current === 'online' ? 'offline' : 'online');}

    update() 方法接受一个函数，该函数接收当前值并返回新值。当你需要根据现有值当前的状态来修改它时，这会很有用。

  7. 添加切换按钮处理程序

    切换按钮已在模板中。将其连接到你的 toggleStatus() 方法：

    <button (click)="toggleStatus()" class="toggle-btn">  Toggle Status</button>

恭喜！你已创建了你的第一个信号，并学会了如何使用 set() 和 update() 方法来更新它。signal() 函数创建了一个 Angular 跟踪的响应式值，当你更新它时，你的 UI 会自动反映这些更改。

接下来，你将学习如何使用 computed 从信号派生状态！

关于 ChangeDetectionStrategy.OnPush

你可能会在整个教程的组件装饰器中注意到 ChangeDetectionStrategy.OnPush。这是 Angular 组件使用信号的性能优化。目前，你可以安全地忽略它——只需知道它有助于你的应用在使用信号时运行得更快！你可以在变更检测策略 API 文档中了解更多信息。
