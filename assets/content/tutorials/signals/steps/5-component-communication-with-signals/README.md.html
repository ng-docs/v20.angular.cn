使用输入信号将数据传递给组件

edit

既然您已经学习了使用信号管理异步数据，那么让我们来探索 Angular 基于信号的 input() API，用于将数据从父组件传递到子组件，使组件数据流更具响应性和效率。如果您熟悉其他框架中的组件 props，那么 inputs 的概念是相同的。

在此活动中，您将向产品卡片组件添加信号输入，并了解父组件数据如何响应式地向下流动。

  1. 向 ProductCard 添加信号输入

    添加信号 input() 函数以在 product-card 组件中接收数据。

    // Add imports for signal inputsimport {Component, input} from '@angular/core';// Add these signal inputsname = input.required<string>();price = input.required<number>();available = input<boolean>(true);

    请注意 input.required() 如何创建一个必须提供的输入，而带有默认值的 input() 是可选的。

  2. 将输入连接到模板

    更新 product-card 中的模板以显示信号输入值。

    <div class="product-card">  <h3>{{ name() }}</h3>  <p class="price">\${{ price() }}</p>  <p class="status">Status:    @if (available()) {      Available    } @else {      Out of Stock    }  </p></div>

    输入信号在模板中的工作方式与常规信号一样——像调用函数一样调用它们来访问其值。

  3. 将父信号连接到子输入

    更新 app.ts 中 product-card 的用法，以传递动态信号值而不是静态值。

    <!-- Change from static values: --><product-card  name="Static Product"  price="99"  available="true"/><!-- To dynamic signals: --><product-card  [name]="productName()"  [price]="productPrice()"  [available]="productAvailable()"/>

    方括号 [] 创建属性绑定，将当前信号值传递给子组件。

  4. 测试响应式更新

    在 app.ts 中添加方法来更新父信号，看看子组件如何自动响应。

    updateProduct() {  this.productName.set('Updated Product');  this.productPrice.set(149);}toggleAvailability() {  this.productAvailable.set(!this.productAvailable());}
    <!-- Add controls to test reactivity --><div class="controls">  <button (click)="updateProduct()">Update Product Info</button>  <button (click)="toggleAvailability()">Toggle Availability</button></div>

    当父信号发生变化时，子组件会自动接收并显示新值！

太棒了！您已经了解了信号输入的工作原理：

  • 信号输入 - 使用 input() 和 input.required() 从父组件接收数据
  • 响应式更新 - 当父信号值更改时，子组件会自动更新
  • 类型安全 - 信号输入提供完整的 TypeScript 类型检查
  • 默认值 - 可选输入可以具有默认值，而必需输入必须提供

信号输入使组件通信更具响应性，并在许多情况下消除了对 OnChanges 生命周期钩子的需求。
