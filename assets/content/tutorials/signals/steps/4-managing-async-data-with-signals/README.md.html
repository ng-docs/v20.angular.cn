使用 Resources API 和信号管理异步数据

edit

既然您已经学会了如何使用链接信号派生状态，那么让我们来探索如何使用 Resource API 处理异步数据。Resource API 提供了一种使用信号管理异步操作的强大方法，内置了加载状态、错误处理和请求管理。

在此活动中，您将学习如何使用 resource() 函数异步加载数据，以及如何通过构建一个演示 Resource API 实际应用的用户配置文件加载器来处理异步操作的不同状态。

  1. 导入 resource 函数和 API

    将 resource 添加到现有导入中，并导入假的 API 函数。

    // Add resource to existing importsimport {Component, signal, computed, resource, ChangeDetectionStrategy} from '@angular/core';// Import mock API functionimport {loadUser} from './user-api';

  2. 为用户数据创建资源

    在组件类中添加一个属性，该属性创建一个资源，用于根据用户 ID 信号加载用户数据。

    userId = signal(1);userResource = resource({  params: () => ({ id: this.userId() }),  loader: (params) => loadUser(params.params.id)});

  3. 添加方法与资源进行交互

    添加方法来更改用户 ID 和重新加载资源。

    loadUser(id: number) {  this.userId.set(id);}reloadUser() {  this.userResource.reload();}

    更改 params 信号会自动触发重新加载，或者您可以使用 reload() 手动重新加载。

  4. 为资源状态创建计算信号

    添加计算信号以访问资源的各种状态。

    isLoading = computed(() => this.userResource.status() === 'loading');hasError = computed(() => this.userResource.status() === 'error');

    资源提供一个可以为 'loading'、'success' 或 'error' 的 status() 信号，一个用于加载数据的 value() 信号，以及一个安全检查数据是否可用的 hasValue() 方法。

  5. 连接按钮并显示资源状态

    模板结构已提供。现在连接所有内容：

    第一部分。为按钮添加点击处理程序：

    <button (click)="loadUser(1)">Load User 1</button><button (click)="loadUser(2)">Load User 2</button><button (click)="loadUser(999)">Load Invalid User</button><button (click)="reloadUser()">Reload</button>

    第二部分。用资源状态处理替换占位符：

    @if (isLoading()) {  <p>Loading user...</p>} @else if (hasError()) {  <p class="error">Error: {{ userResource.error()?.message }}</p>} @else if (userResource.hasValue()) {  <div class="user-info">    <h3>{{ userResource.value().name }}</h3>    <p>{{ userResource.value().email }}</p>  </div>}

    资源提供了不同的方法来检查其状态：

    • isLoading() - 获取数据时为 true
    • hasError() - 发生错误时为 true
    • userResource.hasValue() - 数据可用时为 true
    • userResource.value() - 访问加载的数据
    • userResource.error() - 访问错误信息

太棒了！您现在已经学会了如何将 Resource API 与信号一起使用。需要记住的关键概念：

  • 资源是响应式的：当参数更改时，它们会自动重新加载
  • 内置状态管理：资源提供 status()、value() 和 error() 信号
  • 自动清理：资源会自动处理请求取消和清理
  • 手动控制：您可以根据需要手动重新加载或中止请求
