Angular 服务

edit

本教程课程演示了如何创建一个 Angular 服务，并使用依赖注入将其包含在你的应用中。

你将学到什么

你的应用有一个服务来为你的应用提供数据。 在本课程结束时，该服务将从本地静态数据中读取数据。 在后面的课程中，你将更新该服务以从 Web 服务获取数据。

服务的概念预览

本教程介绍了 Angular 服务和依赖注入。

Angular 服务

Angular 服务 为你提供了一种方法来分离 Angular 应用数据和函数，这些数据和函数可以被你的应用中的多个组件使用。 为了被多个组件使用，服务必须是可注入的。 可注入的且被组件使用的服务将成为该组件的依赖项。 组件依赖于这些服务，没有它们就无法运行。

依赖注入

依赖注入 是一种机制，用于管理应用的组件的依赖项以及其他组件可以使用的服务。

  1. 为你的应用创建一个新服务

    此步骤为你的应用创建一个可注入的服务。

    在 IDE 的 终端 窗格中：

    1. 在你的项目目录中，导航到 first-app 目录。

    2. 在 first-app 目录中，运行此命令以创建新服务。

      ng generate service housing --skip-tests

    3. 运行 ng serve 以构建应用并将其服务于 http://localhost:4200。

    4. 确认应用构建时没有错误。 在继续下一步之前，更正任何错误。

  3. 将新服务注入 Home

    此步骤将新服务注入你的应用的 Home，以便它可以从服务读取应用数据。 在后续课程中，你将用实时数据源替换静态数据，以获取你在真实应用中可能获取的数据。

    在 IDE 的编辑窗格中，在 src/app/home/home.ts 中：

    1. 在 src/app/home/home.ts 的顶部，将 inject 添加到从 @angular/core 导入的项中。这将把 inject 函数导入到 Home 类中。

      更新 src/app/home/home.ts

摘要：在本课中，你向应用添加了一个 Angular 服务，并将其注入到了 Home 类中。 这使得应用获取数据的方式更加模块化。 目前，新服务从静态数据数组获取数据。 在后续课程中，你将重构该服务，使其从 API 端点获取数据。
