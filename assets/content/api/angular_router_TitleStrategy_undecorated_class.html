说明

提供在路由器导航后设置页面标题的策略。

内置实现会遍历路由器状态快照，并查找具有 title 属性的最深层主出口。给定下面的 Routes，导航到 /base/child(popup:aux) 会导致文档标题被设置为“child”。

[  {path: 'base', title: 'base', children: [    {path: 'child', title: 'child'},  ],  {path: 'aux', outlet: 'popup', title: 'popupTitle'}]

此类可用作自定义标题策略的基类。也就是说，您可以创建自己的扩展了 TitleStrategy 的类。请注意，在上面的示例中，来自命名出口的 title 从未被使用。但是，可以实现自定义策略来合并命名出口中的标题。
