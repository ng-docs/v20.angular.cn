@for 控制块为集合中的每个条目重复渲染一个控制块的内容。

语法 

@for (item of items; track item.name) {<li>{{ item.name }}</li>} @empty {<li>There are no items.</li>}

描述

@for 控制块会响应集合中的更改来渲染其内容。集合可以是任何 JavaScript 可迭代者，但使用常规的 Array 会有性能优势。

你可以选择在 @for 控制块内容之后立即包含一个 @empty 区块。当没有条目时，将显示 @empty 控制块的内容。

Angular 的 @for 控制块不支持像 JavaScript 的 continue 或 break 这样的流程修改语句。

track 和对象标识

track 表达式的值决定了一个键，该键用于将数组条目与 DOM 中的视图相关联。清晰地指示条目标识，使 Angular 可以在集合中添加、删除或移动条目时执行最少的 DOM 操作集。

为了优化性能，尤其是在遍历不可变数据的循环中，请确保有效地使用 track 表达式来唯一标识每个条目。由于可能导致性能不佳，因此 @for 循环需要 track 表达式。

对于保持静态的集合，track $index 提供了一种直接的跟踪机制。对于经历添加、删除或重新排序的动态集合，请选择每个条目的唯一属性作为跟踪键。

Track 表达式只能引用 $index、条目以及组件类的字段。如果 @for 控制块的 let 部分为 $index 引入了别名，也可以引用该别名。

$index 和其他上下文变量

在 @for 内容内部，始终有几个隐式变量可用：

变量 含义
$count 迭代集合中的条目数
$index 当前行的索引
$first 当前行是否为第一行
$last 当前行是否为最后一行
$even 当前行索引是否为偶数
$odd 当前行索引是否为奇数

这些变量始终以这些名称提供，但可以通过 let 段进行别名：

@for (item of items; track item.id; let idx = $index, e = $even) {Item #{{ idx }}: {{ item.name }}}

在使用嵌套的 @for 控制块（其中上下文变量名称可能冲突）的情况下，别名特别有用。
