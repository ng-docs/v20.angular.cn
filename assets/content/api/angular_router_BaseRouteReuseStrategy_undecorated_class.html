此基本路由复用策略仅在匹配的路由器配置相同时才复用路由。这可以防止仅当路由参数、查询参数或片段更改时组件被销毁和重新创建（也就是说，现有组件被复用）。

描述

此基本路由复用策略仅在匹配的路由器配置相同时才复用路由。这可以防止仅当路由参数、查询参数或片段更改时组件被销毁和重新创建（也就是说，现有组件被复用）。

此策略不存储任何路由以供稍后复用。

Angular 默认使用此策略。

它可以作为自定义路由重用策略的基类，即您可以创建自己的类来扩展 BaseRouteReuseStrategy 类。
