@angular/ssr

ServerRoutePrerenderWithParams

interface

使用静态站点生成 (SSG) 模式的服务器路由配置，包括对带参数路由的支持。

RenderModeServerRoutePrerenderPrerenderFallback

API

    
      interface ServerRoutePrerenderWithParams extends Omit<ServerRoutePrerender, 'fallback'> {}

fallback

PrerenderFallback | undefined

如果 SSG 路径未预渲染，则使用的可选策略。这与带参数的 URL 路由尤其相关，在这些路由中，某些路径可能不会在构建时预渲染。

此属性决定了如何处理对未预渲染路径的请求：

  • PrerenderFallback.Server：在请求时使用服务器端渲染（SSR）动态生成页面。
  • PrerenderFallback.Client：使用客户端渲染（CSR）在客户端获取和渲染页面。
  • PrerenderFallback.None：无回退；如果路径未预先渲染，服务器将不处理该请求。

getPrerenderParams

() => Promise<Record<string, string>[]>

一个返回 Promise 的函数，该 Promise 求解为一个对象数组，每个对象都表示一个带有 URL 参数的路由路径。此函数在注入器上下文中运行，允许访问 Angular 服务和依赖项。

它也适用于捕获所有路由（例如 /**），其中参数名称将是 **，返回值将是路径的片段，例如 /foo/bar。这些路由也可以组合使用，例如 /product/:id/**，其中参数化片段（:id）和捕获所有片段（**）可以一起使用来处理更复杂的路径。

path

string

与此路由关联的路径。

headers

Record<string, string> | undefined

要包含在此路由响应中的可选附加标头。

renderMode

RenderMode.Prerender

指定路由使用静态站点生成 (SSG) 模式。
