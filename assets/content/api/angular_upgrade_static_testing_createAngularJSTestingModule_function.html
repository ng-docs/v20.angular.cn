createAngularJSTestingModule

string

一个辅助函数，用于单元测试依赖于降级 Angular 服务的 AngularJS 服务。

此函数返回一个 AngularJS 模块，该模块配置为连接 AngularJS 和 Angular 注入器，而无需实际引导混合应用。这使得单元测试服务更简单、更快速。

在调用 angular.mocks.module 时使用返回的 AngularJS 模块，将此模块包含在单元测试注入器中。

在以下代码片段中，我们正在使用两个模块配置 $injector：AngularJS 的 ng1AppModule，它是我们混合应用的 AngularJS 部分；以及 Ng2AppModule，它是 Angular 部分。

{@example upgrade/static/ts/full/module.spec.ts region='angularjs-setup'}

完成此操作后，我们可以像平常一样通过 AngularJS $injector 获取服务。 已降级（或依赖于）降级 Angular 服务的服务将被按需实例化，由 Angular 的根 Injector 进行实例化。

在以下代码片段中，heroesService 是一个降级的 Angular 服务，我们正在从 AngularJS 访问它。

{@example upgrade/static/ts/full/module.spec.ts region='angularjs-spec'}

此辅助函数用于测试服务，而不是组件。对于组件测试，你仍然必须引导混合应用。有关更多信息，请参阅 UpgradeModule 或 downgradeModule。

生成的配置不会将 AngularJS 摘要连接到 Zone 钩子。测试编写者有责任根据需要调用 $rootScope.$apply，以触发来自 Angular 的异步事件的 AngularJS 处理程序。

此辅助函数会设置全局变量以保存共享的 Angular 和 AngularJS 注入器。

  • 每个 spec 只调用此辅助函数一次。
  • 请勿在与 createAngularTestingModule 相同的测试用例中使用 createAngularJSTestingModule。

这是示例应用程序及其单元测试，它们使用了 createAngularTestingModule 和 createAngularJSTestingModule。
