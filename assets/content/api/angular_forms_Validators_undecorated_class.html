email

ValidationErrors | null

验证器，要求控件的值通过电子邮件验证测试。

使用适用于常见用例的正则表达式模式测试值。该模式基于WHATWG HTML 规范中有效电子邮件地址的定义，并进行了一些增强以包含更多 RFC 规则（例如与域名和地址各部分长度相关的规则）。

与 WHATWG 版本的差异包括：

  • 不允许 local-part（@ 符号之前的部分）以句点 (.) 开头或结尾。
  • 不允许 local-part 超过 64 个字符。
  • 不允许整个地址超过 254 个字符。

如果此模式不满足您的业务需求，您可以使用 Validators.pattern() 来根据不同模式验证值。

@paramcontrolAbstractControl<any, any, any>
@returnsValidationErrors | null
用法说明

验证该字段是否符合有效的电子邮件模式 

const control = new FormControl('bad@', Validators.email);console.log(control.errors); // {email: true}

minLength

ValidatorFn

验证器，它要求控件值的条目数大于或等于所提供的最小长度。如果你使用 HTML5 minlength 属性，默认也会提供此验证器。请注意，minLength 验证器旨在仅用于具有数字 length 或 size 属性的类型，例如字符串、数组或集合。当值的 length 或 size 属性为 0 时（例如，在空字符串或空数组的情况下），也不会调用 minLength 验证器逻辑，以支持可选控件。如果空值不应被视为有效值，则可以使用标准 required 验证器。

@paramminLengthnumber
@returnsValidatorFn
用法说明

验证该字段至少包含 3 个字符 

const control = new FormControl('ng', Validators.minLength(3));console.log(control.errors); // {minlength: {requiredLength: 3, actualLength: 2}}
<input minlength="5">

pattern

ValidatorFn

验证器，它要求控件的值与正则表达式模式匹配。如果你使用 HTML5 pattern 属性，默认也会提供此验证器。

@parampatternstring | RegExp

一个正则表达式，将按原样用于测试值，或者一个字符串。如果传递的是字符串，则在提供的字符串前面加上 ^ 字符，并在后面加上 $ 字符（如果尚不存在），并且生成的正则表达式用于测试值。

@returnsValidatorFn
用法说明

验证该字段是否仅包含字母或空格 

const control = new FormControl('1', Validators.pattern('[a-zA-Z ]*'));console.log(control.errors); // {pattern: {requiredPattern: '^[a-zA-Z ]*$', actualValue: '1'}}
<input pattern="[a-zA-Z ]*">

使用 global 或 sticky 标志进行模式匹配

使用 g 或 y 标志创建并传递给 Validators.pattern 的 RegExp 对象，在连续运行时可能会对同一输入产生不同的结果。这是因为 RegExp.prototype.test 的行为在 ECMA-262 中有规定（当使用全局或粘性标志时，RegExp 会保留最后一个匹配项的索引）。 因此，建议在使用 Validators.pattern 时，不要传递启用全局或粘性标志的 RegExp 对象。

// Not recommended (since the `g` flag is used)const controlOne = new FormControl('1', Validators.pattern(/foo/g));// Goodconst controlTwo = new FormControl('1', Validators.pattern(/foo/));

描述

提供一组内置的验证器，这些验证器可用于表单控件。

验证器是一个函数，它处理一个 FormControl 或一组控件，并返回一个错误映射或 null。null 映射表示验证已通过。
