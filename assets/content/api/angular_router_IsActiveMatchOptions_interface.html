IsActiveMatchOptions

interface
stable
代码

一组选项，用于指定在给定当前路由状态的 UrlTree 的情况下，如何确定一个 UrlTree 是否处于活动状态。

Router#isActive

API

    
      interface IsActiveMatchOptions {}

matrixParams

"exact" | "subset" | "ignored"

定义比较两个 UrlTree 的矩阵参数的策略。

矩阵参数匹配取决于匹配段的策略。也就是说，如果 paths 选项设置为 'subset'，则只会比较匹配段的矩阵参数。

  • 'exact'：要求匹配的段也必须具有精确的矩阵参数匹配。
  • 'subset'：路由器活动 UrlTree 中的匹配片段可能包含额外的矩阵参数，但问题中的 UrlTree 中存在的参数必须匹配。
  • 'ignored'：在比较 UrlTree 时，将忽略矩阵参数。

queryParams

"exact" | "subset" | "ignored"

定义比较两个 UrlTree 的查询参数的策略。

  • 'exact'：查询参数必须精确匹配。
  • 'subset'：活动的 UrlTree 可能包含额外的参数，但必须匹配问题中的 UrlTree 中存在的任何参数的键和值。
  • 'ignored'：在比较 UrlTree 时，将忽略查询参数。

paths

"exact" | "subset"

定义比较 UrlTree 的 UrlSegment 的策略。

  • 'exact'：每个 UrlTree 中的所有片段必须匹配。
  • 'subset'：如果一个 UrlTree 是活动路由的子树，则确定其为活动状态。也就是说，活动路由可能包含额外的片段，但必须至少包含所讨论的 UrlTree 的所有片段。

fragment

"exact" | "ignored"
  • 'exact'：表示 UrlTree 的片段必须相等。
  • 'ignored'：在确定 UrlTree 是否处于活动状态时，将不比较片段。
