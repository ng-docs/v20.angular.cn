setValue

void

设置 FormGroup 的值。它接受一个与组结构匹配的对象，其中控件名称作为键。

@paramvalueɵIsAny<TControl, { [key: string]: any; }, { [K in keyof TControl]: ɵRawValue<TControl[K]>; }>

与组结构匹配的控件的新值。

@paramoptions{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; }

配置选项，用于确定控件在值更改后如何传播更改和发出事件。 配置选项会传递给 * updateValueAndValidity 方法。

  • onlySelf：为 true 时，每次更改仅影响此控件，而不影响其父控件。默认为 false。
  • emitEvent：当为 true 或未提供（默认值）时，当控件值更新时，statusChanges 和 valueChanges 可观察对象都会发出包含最新状态和值的事件。当为 false 时，不发出任何事件。
@returnsvoid
用例说明

为表单组设置完整的值 

const form = new FormGroup({  first: new FormControl(),  last: new FormControl()});console.log(form.value);   // {first: null, last: null}form.setValue({first: 'Nancy', last: 'Drew'});console.log(form.value);   // {first: 'Nancy', last: 'Drew'}

reset

void

重置 FormGroup，将所有后代标记为 pristine 和 untouched，并将所有后代的值设置为其默认值，或者如果未提供默认值则设置为 null。

你可以通过传入一个状态映射表来重置为特定的表单状态，该映射表与你的表单结构匹配，其中控件名称作为键。 该状态可以是独立的值，也可以是同时包含值和禁用状态的表单状态对象。

@paramvalueɵTypedOrUntyped<TControl, ɵIsAny<TControl, { [key: string]: any; }, Partial<{ [K in keyof TControl]: ɵValue<TControl[K]> | FormControlState<ɵValue<TControl[K]>>; }>>, any>

使用初始值或定义初始值和禁用状态的对象来重置控件。

@paramoptions{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; }

配置选项，用于确定当组重置时，控件如何传播更改和发出事件。

  • onlySelf：为 true 时，每次更改仅影响此控件，而不影响其父控件。默认为 false。
  • emitEvent：当为 true 或未提供（默认值）时，当控件重置时，statusChanges 和 valueChanges 可观察对象都会发出包含最新状态和值的事件。当为 false 时，不发出任何事件。配置选项会传递给 * updateValueAndValidity 方法。
@returnsvoid
用例说明

重置表单组的值 

const form = new FormGroup({  first: new FormControl('first name'),  last: new FormControl('last name')});console.log(form.value);  // {first: 'first name', last: 'last name'}form.reset({ first: 'name', last: 'last name' });console.log(form.value);  // {first: 'name', last: 'last name'}

重置表单组的值和禁用状态 

const form = new FormGroup({  first: new FormControl('first name'),  last: new FormControl('last name')});form.reset({  first: {value: 'name', disabled: true},  last: 'last'});console.log(form.value);  // {last: 'last'}console.log(form.get('first').status);  // 'DISABLED'

enabled

errors

ValidationErrors | null

一个对象，其中包含验证失败产生的任何错误；如果没有错误，则为 null。

pristine

boolean

如果用户尚未更改 UI 中的值，则控件是 pristine 的。

dirty

boolean

如果用户已更改 UI 中的值，则控件是 dirty 的。

touched

boolean

如果控件被标记为 touched，则为 True。

一旦用户在其上触发了 blur 事件，控件就会被标记为 touched。

untouched

boolean

如果控件尚未被标记为已触碰，则为 True

如果用户尚未在其上触发 blur 事件，则控件为 untouched。

events

any

一个多播可观察对象，每当控件的状态更改时都会发出事件。 它会在值、状态、pristine 或 touched 更改时发出。

注意：在值更改时，发出操作在此控件的值更新后立即发生。父控件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍后更新，因此从此事件的回调中访问父控件的值（使用 value 属性）可能会导致获取尚未更新的值。请改为订阅父控件的 events。 对于其他事件类型，事件在父控件更新后发出。

valueChanges

Observable<TValue>

一个多播可观察对象，每当控件的值在 UI 中或以编程方式更改时，都会发出一个事件。每次调用 enable() 或 disable() 而不传递 {emitEvent: false} 作为函数参数时，它也会发出一个事件。

注意：发出操作在此控件的值更新后立即发生。父控件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍后更新，因此从此事件的回调中访问父控件的值（使用 value 属性）可能会导致获取尚未更新的值。请改为订阅父控件的 valueChanges 事件。

statusChanges

Observable<FormControlStatus>

一个多播可观察对象，每当控件的验证 status 重新计算时，都会发出一个事件。

updateOn

FormHooks

报告 AbstractControl 的更新策略（即控件自行更新的事件）。 可能的值：'change' | 'blur' | 'submit' 默认值：'change'

描述

跟踪一组 FormControl 实例的值和有效性状态。

一个 FormGroup 将每个子 FormControl 的值聚合到一个对象中， 其中每个控件名称作为键。它通过归约其子控件的状态值来计算其状态。 例如，如果组中的一个控件无效，则整个组将变为无效。

FormGroup 是在 Angular 中定义表单的四个基本构建块之一， 另外三个是 FormControl、FormArray 和 FormRecord。

实例化 FormGroup 时，将子控件的集合作为第一个参数传入。 每个子控件的键会注册该控件的名称。

FormGroup 适用于预先知道键的用例。 如果您需要动态添加和删除控件，请改用 FormRecord。

FormGroup 接受一个可选的类型参数 TControl，它是一个对象类型，其值是内部控件的类型。

用法说明

创建一个包含 2 个控件的表单组 

const form = new FormGroup({  first: new FormControl('Nancy', Validators.minLength(2)),  last: new FormControl('Drew'),});console.log(form.value);   // {first: 'Nancy', last; 'Drew'}console.log(form.status);  // 'VALID'

类型参数和可选控件

FormGroup 接受一个泛型参数，该参数是一个包含其内部控件的对象。 此类型通常会自动推断，但如果您愿意，可以随时显式指定它。

如果您有可选的控件（即，可以删除它们，您可以在类型中使用 ?）：

const form = new FormGroup<{  first: FormControl<string|null>,  middle?: FormControl<string|null>, // Middle name is optional.  last: FormControl<string|null>,}>({  first: new FormControl('Nancy'),  last: new FormControl('Drew'),});

创建一个带有组级别验证器的表单组

您可以将组级别验证器作为第二个参数包含在内，或将组级别异步验证器作为第三个参数包含在内。当您要执行考虑多个子控件的值的验证时，这些会派上用场。

const form = new FormGroup({  password: new FormControl('', Validators.minLength(2)),  passwordConfirm: new FormControl('', Validators.minLength(2)),}, passwordMatchValidator);function passwordMatchValidator(g: FormGroup) {   return g.get('password').value === g.get('passwordConfirm').value      ? null : {'mismatch': true};}

与 FormControl 实例一样，您可以选择将验证器和异步验证器作为选项对象的一部分传入。

const form = new FormGroup({  password: new FormControl('')  passwordConfirm: new FormControl('')}, { validators: passwordMatchValidator, asyncValidators: otherValidator });

为表单组中的所有控件设置 updateOn 属性（Property）

选项对象用于为每个子控件的 updateOn 属性（Property）设置默认值。如果在组级别将 updateOn 设置为 'blur'，则所有子控件都默认为 'blur'，除非子控件已明确指定了不同的 updateOn 值。

const c = new FormGroup({  one: new FormControl()}, { updateOn: 'blur' });

将 FormGroup 与可选控件一起使用

在 FormGroup 中可以包含可选控件。可以使用 removeControl 稍后删除可选控件，并且在调用 reset 时可以省略可选控件。必须在组的类型中将可选控件声明为可选。

const c = new FormGroup<{one?: FormControl<string>}>({  one: new FormControl('')});

请注意，c.value.one 的类型为 string|null|undefined。这是因为调用 c.reset({}) 而不提供可选键 one 将导致它变为 null。
