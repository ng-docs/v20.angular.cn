versionUpdates

Observable<VersionEvent>

每当在服务器上检测到新版本时，都会发出一个 VersionDetectedEvent 事件。

每当检查或下载新版本失败时，都会发出一个 VersionInstallationFailedEvent 事件。

每当下载了新版本并准备好激活时，都会发出一个 VersionReadyEvent 事件。

unrecoverable

Observable<UnrecoverableStateEvent>

每当 Service Worker 用于为该客户端提供服务的应用版本处于无法恢复的损坏状态（需要完全重新加载页面才能恢复）时，都会发出一个 UnrecoverableStateEvent 事件。

isEnabled

boolean

如果 Service Worker 已启用（浏览器支持并通过 ServiceWorkerModule 启用），则为 true。

checkForUpdate

Promise<boolean>

检查更新，并等待直到从服务器下载新版本并准备好激活。

@returnsPromise<boolean>

activateUpdate

Promise<boolean>

将当前客户端（即浏览器选项卡）更新到准备好激活的最新版本。

在大多数情况下，你不应该使用此方法，而应该通过重新加载页面来更新客户端。

在不重新加载的情况下更新客户端很容易导致应用崩溃，这是因为应用外壳程序与其他页面资源（例如惰性加载的代码块）之间存在版本不匹配，这些代码块的文件名可能在不同版本之间发生更改。

仅当您确定此方法对您的特定用例是安全时，才使用此方法。

@returnsPromise<boolean>
