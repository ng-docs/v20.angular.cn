@angular/common/http

HttpResourceRequest

interface
实验性v19.2 起
代码

将发送到后端的 httpResource 请求的结构。

API

    
      interface HttpResourceRequest {}

url

string

请求的 URL。

此 URL 不应包含查询参数。而是通过 params 字段指定查询参数。

method

string | undefined

请求的 HTTP 方法，如果未指定，则默认为 GET。

body

unknown

要随请求发送的正文（如果有）。

如果用户未指定 Content-Type 标头，Angular 将尝试根据 body 的类型设置一个。

参数

HttpParams | Record<string, string | number | boolean | readonly (string | number | boolean)[]> | undefined

将附加到请求 URL 的查询参数字典。

headers

HttpHeaders | Record<string, string | readonly string[]> | undefined

要包含在传出请求中的标头字典。

context

HttpContext | undefined

存储在键值对字典中的请求上下文。

reportProgress

boolean | undefined

如果为 true，则将为请求启用进度事件，并通过 HttpResource.progress Signal 传递。

withCredentials

boolean | undefined

指定是否应在传出请求上设置 withCredentials 标志。

此标志使浏览器随请求一起发送 Cookie 和其他身份验证信息。

keepalive

boolean | undefined

当使用 fetch 实现且设置为 true 时，如果发起请求的页面在请求完成之前被卸载，浏览器将不会中止关联的请求。

cache

RequestCache | (string & {}) | undefined

控制请求如何与浏览器的 HTTP 缓存进行交互。 这会影响响应是从缓存中检索，如何存储，还是完全绕过缓存。

credentials

(string & {}) | RequestCredentials | undefined

请求的凭据模式，它决定了如何处理 cookie 和其他身份验证信息。 这会影响凭据是否与跨域请求一起发送。

priority

(string & {}) | RequestPriority | undefined

指示请求的相对优先级。浏览器可能会使用它来决定分派请求和获取资源的顺序。

mode

(string & {}) | RequestMode | undefined

请求的模式，它决定了请求如何与浏览器的安全模型进行交互。 这会影响 CORS（跨域资源共享）和同源策略等事项。

redirect

(string & {}) | RequestRedirect | undefined

请求的重定向模式，它决定了如何处理重定向。 这会影响请求是否自动跟随重定向，或者在发生重定向时是否失败。

referrer

string | undefined

请求的引用者，可用于指示请求的来源。 这对于安全和分析目的很有用。 值为同源 URL、"about:client" 或空字符串，用于设置请求的引用者。

integrity

string | undefined

请求的完整性元数据，可用于确保请求的内容符合预期。 由请求获取的资源的加密哈希值

transferCache

boolean | { includeHeaders?: string[] | undefined; } | undefined

为此请求配置服务器端渲染传输缓存。

有关更多信息，请参阅有关传输缓存的文档。

timeout

number | undefined

后端 HTTP 请求的超时时间（以毫秒为单位）。
