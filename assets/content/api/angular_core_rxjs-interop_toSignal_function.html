说明

将 Observable 的当前值作为响应式 Signal 获取。

toSignal 返回一个 Signal，该信号通过订阅给定的 Observable 来提供对该 Observable 所产生值的同步响应式访问。返回的 Signal 将始终具有订阅产生的最新值，如果 Observable 出错，则会抛出错误。

当 requireSync 设置为 true 时，toSignal 将断言 Observable 在订阅后会立即产生一个值。这种情况下不需要 initialValue，并且返回的信号不包含 undefined 类型。

默认情况下，当当前的 注入上下文被销毁时，订阅将自动清理。例如，当在组件构建期间调用 toSignal 时，订阅将在组件被销毁时被清理。如果注入上下文不可用，则可以改用显式传递的 Injector。

如果订阅应持续到 Observable 本身完成，则可以改为指定 manualCleanup 选项，这将禁用自动订阅拆除。在这种配置中，也不需要注入上下文。
