@angular/cdk/drag-drop

CdkDropList

类
代码

包裹一组可拖拽条目的容器。

API

    
      class CdkDropList<T = any> implements OnDestroy {}

element

any

connectedTo

string | CdkDropList<any> | (string | CdkDropList<any>)[]

此容器连接到的其他可拖拽容器，条目可以转移到这些容器中。可以是其他 drop 容器的引用，或是它们的唯一 ID。

data

T

附加到此容器的任意数据。

orientation

DropListOrientation

列表的方向。

id

string

此拖放区的唯一 ID。可在另一个 CdkDropList 的 connectedTo 中用作引用。

lockAxis

DragAxis | null

沿指定轴锁定容器内可拖拽元素的位置。

disabled

boolean

是否禁用从此容器开始拖拽序列。

disabled

boolean

sortingDisabled

boolean

是否禁用此拖放列表内的排序。

enterPredicate

(drag: CdkDrag<any>, drop: CdkDropList<any>) => boolean

用于确定是否允许将条目移动到拖放容器的函数。

sortPredicate

(index: number, drag: CdkDrag<any>, drop: CdkDropList<any>) => boolean

用于确定条目是否可以排序到特定索引的函数。

autoScrollDisabled

boolean

当用户将指针移近边缘时是否自动滚动视图。

autoScrollStep

NumberInput

自动滚动元素时，每帧要滚动的像素数。

elementContainerSelector

string | null

将用于为拖放列表解析备用元素容器的选择器。 传递备用容器对于那些可能无法控制列表内可拖拽项的父节点（例如，由于内容投影）的情况很有用。 这允许如下用法：

<div cdkDropList cdkDropListElementContainer=".inner">  <div class="inner">    <div cdkDrag></div>  </div></div>

hasAnchor

boolean

默认情况下，当条目离开其初始容器时，其占位符将转移到新容器。 如果这不符合你的用例，你可以启用此选项，它将克隆占位符并将其保留在原始容器内。 如果条目返回到初始容器，锚点元素将自动移除。

可以通过定位 cdk-drag-anchor 类来设置克隆的占位符的样式。

此选项与 cdkDropListSortingDisabled 结合使用，可在拖放列表中实现复制行为，非常有用。

dropped

EventEmitter<CdkDragDrop<T, any, any>>

当用户将条目放入容器中时发出事件。

entered

EventEmitter<CdkDragEnter<T, T>>

当用户将新的拖拽条目移入此容器时发出事件。

exited

EventEmitter<CdkDragExit<T, T>>

当用户通过将条目拖入另一个容器而将其从容器中移除时发出事件。

sorted

EventEmitter<CdkDragSortEvent<T, T>>

当用户在主动拖拽时交换条目时发出事件。
