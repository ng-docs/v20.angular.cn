eventCoalescing

boolean | undefined

（可选）指定是否合并事件变更检测。 考虑以下情况。

<div (click)="doSomething()">  <button (click)="doSomethingElse()"></button></div>

当按钮被点击时，由于事件冒泡，两个事件处理程序都将被调用，并且会触发 2 次变更检测。我们可以合并此类事件，以便仅触发一次变更检测。

默认情况下，此选项设置为 false，这意味着事件不会被合并，并且变更检测将被多次触发。如果此选项设置为 true，则在上述场景中，变更检测将只触发一次。

runCoalescing

boolean | undefined

可选地指定是否将 NgZone#run() 方法调用合并为一次变更检测。

考虑以下情况。

for (let i = 0; i < 10; i ++) {  ngZone.run(() => {    // do something  });}

这种情况会多次触发变更检测。使用 ngZoneRunCoalescing 选项，事件循环中的所有变更检测都只触发一次。此外，变更检测在 requestAnimation 中执行。

ignoreChangesOutsideZone

boolean | undefined
@deprecated

引入此选项是出于谨慎，旨在让开发人员选择退出 v18 中的新行为，即当上述事件发生在 Zone 外部时，会调度变更检测。在发布后监控结果后，我们已确定此特性运行正常，并且认为不应通过将此选项设置为 true 来禁用它。

当为 false 时，当 Angular 收到模板需要刷新的明确指示时，会调度变更检测。这包括：

  • 调用 ChangeDetectorRef.markForCheck
  • 调用 ComponentRef.setInput
  • 更新在模板中读取的 Signal
  • 附着一个标记为脏视图
  • 移除视图
  • 注册渲染钩子（仅当渲染钩子执行上述操作之一时，模板才会刷新）
