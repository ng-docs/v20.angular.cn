描述

当应用于模板中的元素时，使该元素成为一个链接，该链接会启动到某个路由的导航。导航会在页面上的一个或多个 <router-outlet> 位置打开一个或多个路由组件。

给定路由配置 [{ path: 'user/:name', component: UserCmp }]， 以下代码创建到该路由的静态链接： <a routerLink="/user/bob">链接到用户组件</a>

你可以使用动态值来生成链接。 对于动态链接，请传递一个路径段数组， 后跟每个段的参数。 例如，['/team', teamId, 'user', userName, {details: true}] 会生成一个指向 /team/11/user/bob;details=true 的链接。

多个静态段可以合并为一个术语，并与动态段组合。 例如，['/team/11/user', userName, {details: true}]

你提供给链接的输入被视为当前 URL 的增量。 例如，假设当前 URL 是 /user/(box//aux:team)。 链接 <a [routerLink]="['/user/jim']">Jim</a> 会创建 URL /user/(jim//aux:team)。 更多信息请参阅 Router#createUrlTree。

用法说明

你可以在链接中使用绝对路径或相对路径，设置查询参数， 控制参数的处理方式，并保留导航状态的历史记录。

相对链接路径

第一个段名称可以加上前缀 /、./ 或 ../。

  • 如果第一个段以 / 开头，则路由器会从应用的根路径查找路由。
  • 如果第一个段以 ./ 开头，或者不以斜杠开头，则路由器会在当前激活路由的子路由中查找。
  • 如果第一个段以 ../ 开头，则路由器会在路由树中向上移动一级。

设置和处理查询参数与片段

以下链接会将查询参数和片段添加到生成的 URL：

<a [routerLink]="['/user/bob']" [queryParams]="{debug: true}" fragment="education">  link to user component</a>

默认情况下，该指令使用给定的查询参数构造新的 URL。此范例会生成链接：/user/bob?debug=true#education。

你可以指示该指令通过在链接中指定 queryParamsHandling 选项来以不同的方式处理查询参数。 允许的值为：

  • 'merge'：将给定的 queryParams 合并到当前的查询参数中。
  • 'preserve'：保留当前的查询参数。

例如：

<a [routerLink]="['/user/bob']" [queryParams]="{debug: true}" queryParamsHandling="merge">  link to user component</a>

当 routerLink 输入是 UrlTree 时，不能使用 queryParams、fragment、queryParamsHandling、preserveFragment 和 relativeTo。

请参阅 UrlCreationOptions#queryParamsHandling。

保留导航历史记录

您可以提供一个 state 值，该值将被持久化到浏览器的 History.state 属性。 例如：

<a [routerLink]="['/user/bob']" [state]="{tracingId: 123}">  link to user component</a>

使用 Router#getCurrentNavigation 来检索已保存的 导航状态值。例如，在 NavigationStart 事件期间捕获 tracingId：

// Get NavigationStart eventsrouter.events.pipe(filter(e => e instanceof NavigationStart)).subscribe(e => {  const navigation = router.getCurrentNavigation();  tracingService.trace({id: navigation.extras.state.tracingId});});

与 RouterLink 兼容的自定义元素

为了使自定义元素能够与 routerLink 一起工作，相应的自定义 元素必须实现 href 属性，并且必须在静态属性/getter observedAttributes 的数组中列出 href。
