HTTP_TRANSFER_CACHE_ORIGIN_MAP

常量
stable
代码

如果你的应用在服务器和客户端上使用不同的 HTTP 源（通过 HttpClient）来发起 API 调用，HTTP_TRANSFER_CACHE_ORIGIN_MAP 令牌允许你在这些源之间建立映射，以便 HttpTransferCache 特性能够识别这些请求为相同的请求，并在客户端的水合过程中复用服务器上缓存的数据。

API

    
      const HTTP_TRANSFER_CACHE_ORIGIN_MAP: InjectionToken<Record<string, string>>;

描述

如果你的应用在服务器和客户端上使用不同的 HTTP 源（通过 HttpClient）来发起 API 调用，HTTP_TRANSFER_CACHE_ORIGIN_MAP 令牌允许你在这些源之间建立映射，以便 HttpTransferCache 特性能够识别这些请求为相同的请求，并在客户端的水合过程中复用服务器上缓存的数据。

重要提示：HTTP_TRANSFER_CACHE_ORIGIN_MAP 令牌仅应在你应用的服务器代码中提供（通常是在 app.server.config.ts 脚本中）。如果 Angular 检测到该令牌在客户端运行时被定义，它将抛出错误。

用法说明

当通过服务器上的 http://internal-domain.com:8080 和客户端上的 https://external-domain.com 访问同一 API 端点时，你可以使用以下配置：

// in app.server.config.ts{    provide: HTTP_TRANSFER_CACHE_ORIGIN_MAP,    useValue: {        'http://internal-domain.com:8080': 'https://external-domain.com'    }}
