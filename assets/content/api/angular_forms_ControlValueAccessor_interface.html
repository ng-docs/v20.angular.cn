registerOnChange

void

注册一个回调函数，当控件的值在 UI 中更改时，将调用该函数。

当值从视图传播到模型时，表单 API 会在初始化时调用此方法以更新表单模型。

在你自己的值访问器中实现 registerOnChange 方法时，请保存给定的函数，以便你的类在适当的时候调用它。

@paramfnany

要注册的回调函数

@returnsvoid
用法说明

存储更改函数

以下范例将提供的函数存储为内部方法。

registerOnChange(fn: (_: any) => void): void {  this._onChange = fn;}

当 UI 中的值更改时，调用已注册的函数，以允许表单 API 自行更新：

host: {   '(change)': '_onChange($event.target.value)'}

registerOnTouched

void

注册一个回调函数，当在 blur 时更新表单模型时，表单 API 会在初始化时调用该函数。

在你自己的值访问器中实现 registerOnTouched 时，请保存给定的函数，以便在应将控件视为已失焦或“已触碰”时，你的类可以调用它。

@paramfnany

要注册的回调函数

@returnsvoid
用法说明

存储回调函数

以下范例将提供的函数存储为内部方法。

registerOnTouched(fn: any): void {  this._onTouched = fn;}

在 blur（或等效操作）时，你的类应调用已注册的函数，以允许表单 API 自行更新：

host: {   '(blur)': '_onTouched()'}

setDisabledState

void

当控件状态更改为“DISABLED”或从“DISABLED”更改时，表单 API 调用的函数。根据状态，它会启用或禁用相应的 DOM 元素。

@paramisDisabledboolean

要在元素上设置的禁用状态

@returnsvoid
用法说明

以下是如何将 disabled 属性写入原生 DOM 元素的范例：

setDisabledState(isDisabled: boolean): void {  this._renderer.setProperty(this._elementRef.nativeElement, 'disabled', isDisabled);}

描述

定义了一个接口，它充当 Angular 表单 API 和 DOM 中的原生元素之间的桥梁。

实现此接口以创建一个与 Angular 表单集成的自定义表单控件指令。
