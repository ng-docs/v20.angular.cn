代码

当指令的任何数据绑定属性发生更改时，将调用此生命周期钩子。 定义 ngOnChanges() 方法来处理这些更改。

DoCheckOnInit生命周期钩子指南

API

    
      interface OnChanges {}

ngOnChanges

void

每当默认变更检测器检查完数据绑定属性之后，如果检测到至少一个属性已更改，并且在检查视图和内容子项之前，就会立即调用此回调方法。

@paramchangesSimpleChanges

已更改的属性。

@returnsvoid

使用须知

以下代码片段展示了一个组件如何实现此接口，以便为输入属性定义一个 on-changes 处理程序。

@Component({        selector: 'my-cmp',        template: `...`,        standalone: false,      })      class MyComponent implements OnChanges {        @Input() prop: number = 0;        ngOnChanges(changes: SimpleChanges) {          // changes.prop contains the old and the new value...        }      }
