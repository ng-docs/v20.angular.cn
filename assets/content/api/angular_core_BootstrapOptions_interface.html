为引导启动过程提供额外的选项。

弃用警告

API

ngZone

NgZone | "zone.js" | "noop" | undefined
@deprecated

BootstrapOptions 已弃用。请改而在模块的 providers 数组中提供 NgZone。

可选地指定在提供者中未配置时应使用哪个 NgZone。

  • 提供你自己的 NgZone 实例。
  • zone.js - 使用需要 Zone.js 的默认 NgZone。
  • noop - 使用 NoopNgZone，它什么也不做。

ngZoneEventCoalescing

boolean | undefined
@deprecated

BootstrapOptions 已弃用。请改用 provideZoneChangeDetection 来配置合并。

可以选择指定是否合并事件变更检测。 考虑以下情况。

<div (click)="doSomething()">  <button (click)="doSomethingElse()"></button></div>

当按钮被点击时，由于事件冒泡，两个事件处理程序都将被调用，并且会触发 2 次变更检测。我们可以合并这类事件，以便只触发一次变更检测。

默认情况下，此选项将为 false。因此，事件将不会被合并，并且变更检测将被多次触发。如果此选项设置为 true，则将通过调度动画帧异步触发变更检测。因此，在上述情况下，变更检测将仅触发一次。

ngZoneRunCoalescing

boolean | undefined
@deprecated

BootstrapOptions 已弃用。请改用 provideZoneChangeDetection 来配置合并。

可选地指定是否应将 NgZone#run() 方法调用合并为一次变更检测。

考虑以下情况。

for (let i = 0; i < 10; i ++) {  ngZone.run(() => {    // do something  });}

这种情况会多次触发变更检测。 使用 ngZoneRunCoalescing 选项，事件循环中的所有变更检测仅触发一次。 此外，变更检测在 requestAnimation 中执行。

ignoreChangesOutsideZone

boolean | undefined
@deprecated

引入此选项是出于谨慎，旨在让开发人员选择退出 v18 中的新行为，即当上述事件发生在 Zone 之外时，会调度变更检测。在监控发布后的结果后，我们已确定此特性按预期工作，并且不认为应该通过将此选项设置为 true 来禁用它。

当为 false 时，当 Angular 收到需要刷新模板的明确指示时，会调度变更检测。这包括：

  • 调用 ChangeDetectorRef.markForCheck
  • 调用 ComponentRef.setInput
  • 更新在模板中读取的 Signal
  • 附着一个标记为脏的视图
  • 移除视图
  • 注册渲染钩子（仅当渲染钩子执行上述操作之一时，才会刷新模板）
