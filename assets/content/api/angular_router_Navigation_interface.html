控制此导航所用策略的选项。 请参阅 NavigationExtras。

Navigation | null

上一个成功的 Navigation 对象。只有一个之前的导航可用，因此这个之前的 Navigation 对象自身的 previousNavigation 值为 null。

abort

() => void

如果导航尚未完成或尚未到达路由激活点，则中止导航。 如果导航已过可以中止的点，则此函数无效。

说明

关于导航操作的信息。 使用 Router.getCurrentNavigation() 方法检索最新的导航对象。

  • id ：当前导航的唯一标识符。
  • initialUrl : 在导航之前传递给 Router#navigateByUrl() 的目标 URL。 这是路由器解析或应用重定向之前的 URL。
  • extractedUrl : 使用 UrlSerializer.extract() 解析后的初始目标 URL。
  • finalUrl : 应用重定向后的提取 URL。 此 URL 可能无法立即获得，因此此属性可能为 undefined。 在 RoutesRecognized 事件触发后，此属性将得到保证设置。
  • trigger ：标识此导航是如何触发的。 ——“imperative”——由 router.navigateByUrl 或 router.navigate 触发。 ——“popstate”——由 popstate 事件触发。 ——“hashchange”——由 hashchange 事件触发。
  • extras : 一个 NavigationExtras 选项对象，它控制了此导航所使用的策略。
  • previousNavigation : 上一个成功的 Navigation 对象。只有一个之前的导航可用，因此这个之前的 Navigation 对象自身的 previousNavigation 值为 null。
