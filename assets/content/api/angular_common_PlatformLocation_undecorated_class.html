描述

应用开发者不应直接使用此类。请使用 Location。

PlatformLocation 封装了所有对 DOM API 的调用，这使得 Router 可以 与平台无关。 这意味着我们可以为 Angular 支持的不同平台提供 PlatformLocation 的不同实现。 例如，@angular/platform-browser 提供了特定于浏览器环境的实现，而 @angular/platform-server 则提供了适用于服务器端渲染的实现。

当需要与 pushState、popState 等 DOM API 交互时，所有 LocationStrategy 的实现都会直接使用 PlatformLocation 类。

LocationStrategy 又被 Location 服务使用，而该服务又被 Router 直接使用以在路由之间导航。 由于 Router / Location / LocationStrategy 与 DOM API 之间的所有交互都通过 PlatformLocation 类进行，因此它们都是平台无关的。
