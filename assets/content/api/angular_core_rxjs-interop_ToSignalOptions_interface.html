ToSignalOptions

interface
稳定v20.0 起
代码

toSignal 的选项。

API

    
      interface ToSignalOptions<T> {}

initialValue

unknown

由 toSignal 生成的信号的初始值。

在可观察者发出它的第一个值之前，这将是信号的值。

requireSync

boolean | undefined

是否要求在 toSignal 订阅时，可观察者同步发出值。

如果此值为 true，toSignal 将断言可观察者在订阅时立即产生一个值。 设置此选项无需处理信号类型中的 undefined 或提供 initialValue，但如果此要求未满足，则会产生运行时错误。

injector

Injector | undefined

用于提供用于清理可观察者订阅的 DestroyRef 的 Injector。

如果未提供此项，则会从当前的 注入上下文 中检索 DestroyRef，除非请求了手动清理。

manualCleanup

boolean | undefined

当 toSignal 的创建上下文被销毁时，订阅是否应自动清理（通过 DestroyRef）。

如果启用了手动清理，则不使用 DestroyRef，并且订阅将一直存在，直到 Observable 本身完成。

equal

ValueEqualityFn<T> | undefined

一个比较函数，它定义了可观察者发出的值的相等性。

如果提供了初始值，则针对该初始值执行相等性比较。
