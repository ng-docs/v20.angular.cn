API

forRoot

ModuleWithProviders<RouterModule>

创建并配置一个模块，其中包含所有路由器提供者和指令。 （可选）建立一个应用监听器以执行初始导航。

当在根模块注册 NgModule 时，按如下方式导入：

@NgModule({  imports: [RouterModule.forRoot(ROUTES)]})class MyNgModule {}
@paramroutesRoutes

一个 Route 对象数组，用于定义应用的导航路径。

@paramconfigExtraOptions | undefined

一个 ExtraOptions 配置对象，用于控制导航的执行方式。

@returnsModuleWithProviders<RouterModule>

forChild

ModuleWithProviders<RouterModule>

创建一个模块，其中包含所有路由器指令和一个注册路由的提供者， 而无需创建新的 Router 服务。 当为子模块和惰性加载的子模块注册时，按如下方式创建 NgModule：

@NgModule({  imports: [RouterModule.forChild(ROUTES)]})class MyNgModule {}
@paramroutesRoutes

一个 Route 对象数组，用于定义子模块的导航路径。

@returnsModuleWithProviders<RouterModule>

描述

添加用于应用内视图导航的指令和提供者。 使用 Angular 的 Router 服务来声明式地指定应用状态和管理状态转换。

您可以多次导入此 NgModule，为每个惰性加载的捆绑包导入一次。 但是，只有一个 Router 服务可以处于活动状态。 为确保这一点，在导入此模块时，有两种注册路由的方式：

  • forRoot() 方法创建一个 NgModule，其中包含所有指令、给定的路由以及 Router 服务本身。
  • forChild() 方法创建一个 NgModule，其中包含所有指令和给定的路由，但不包含 Router 服务。
