API

    
      interface NavigationExtras extends UrlCreationOptions {}

relativeTo

ActivatedRoute | null | undefined

指定用于相对导航的根 URI。

例如，考虑以下路由配置，其中父路由有两个子路由。

[{  path: 'parent',  component: ParentComponent,  children: [{    path: 'list',    component: ListComponent  },{    path: 'child',    component: ChildComponent  }]}]

以下 go() 函数通过将目标 URI 解释为相对于已激活的 child 路由来导航到 list 路由

 @Component({...}) class ChildComponent {   constructor(private router: Router, private route: ActivatedRoute) {}   go() {     router.navigate(['../list'], { relativeTo: this.route });   } }

null 或 undefined 值表示导航命令应相对于根路径应用。

queryParams

Params | null | undefined

设置 URL 的查询参数。

// Navigate to /results?page=1router.navigate(['/results'], { queryParams: { page: 1 } });

fragment

string | undefined

设置 URL 的哈希片段。

// Navigate to /results#toprouter.navigate(['/results'], { fragment: 'top' });

queryParamsHandling

QueryParamsHandling | null | undefined

如何在路由器链接中处理查询参数以进行下一次导航。 选项之一：

  • preserve：保留当前参数。
  • merge：将新参数与当前参数合并。

“preserve”选项会丢弃任何新的查询参数：

// from /view1?page=1 to/view2?page=1router.navigate(['/view2'], { queryParams: { page: 2 },  queryParamsHandling: "preserve"});

“merge”选项将新的查询参数附加到当前 URL 中的参数：

// from /view1?page=1 to/view2?page=1&otherKey=2router.navigate(['/view2'], { queryParams: { otherKey: 2 },  queryParamsHandling: "merge"});

如果当前参数与 queryParams 对象中的参数之间发生键冲突，则使用新值。

preserveFragment

boolean | undefined

为 true 时，为下一次导航保留 URL 片段

// Preserve fragment from /results#top to /view#toprouter.navigate(['/view'], { preserveFragment: true });

onSameUrlNavigation

OnSameUrlNavigation | undefined

如何处理对当前 URL 的导航请求。

此值是 OnSameUrlNavigation 中可用选项的一个子集， 并且将优先于为 Router 设置的默认值。

skipLocationChange

boolean | undefined

为 true 时，导航时不将新状态推入历史记录。

// Navigate silently to /viewthis.router.navigate(['/view'], { skipLocationChange: true });

replaceUrl

boolean | undefined

为 true 时，导航时替换历史记录中的当前状态。

// Navigate to /viewthis.router.navigate(['/view'], { replaceUrl: true });

state

{ [k: string]: any; } | undefined

开发者定义的、可传递给任何导航的状态。 在导航执行期间，通过 Router.getCurrentNavigation() 方法返回的 Navigation.extras 对象访问此值。

导航完成后，路由器会将一个包含此值的对象以及 navigationId 写入 history.state。 在激活此路由之前调用 location.go() 或 location.replaceState() 时，会写入该值。

请注意，history.state 未通过对象相等性测试，因为路由器在每次导航时都会添加 navigationId。

info

unknown

使用此项来传达关于此特定导航的瞬时信息，例如它是如何发生的。 这样，它就与将设置为 history.state 的持久值 state 不同。 此对象直接分配给路由器的当前 Navigation （不会被复制或克隆），因此应谨慎修改。

一个关于如何使用它的例子是，根据到达特定路由的方式，触发不同的单页导航动画。 例如，考虑一个照片库应用，你可以通过各种路由访问相同的照片 URL 和状态：

  • 在画廊视图中点击它
  • 点击
  • 在相册中查看另一张照片时点击“下一张”或“上一张”
  • 等等。

它们中的每一个都希望在导航时有不同的动画。 此信息不适合存储在持久 URL 或历史记录条目状态中，但从应用的其余部分传递到路由器仍然很重要。

此信息可以与视图过渡特性和 onViewTransitionCreated 回调结合使用。 该信息可以在回调中使用，以便在文档上设置类，从而控制过渡动画，并在过渡动画完成后删除这些类。

browserUrl

string | UrlTree | undefined

设置此项后，Router 会将浏览器的地址栏更新为与给定的 UrlTree 匹配，而不是用于路由匹配的那个。
