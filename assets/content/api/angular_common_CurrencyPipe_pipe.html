@paramvaluestring | number

要格式化为货币的数字。

@paramcurrencyCodestring | undefined

货币代码（符合 ISO 4217 标准），例如美元的 USD 和欧元的 EUR。默认货币代码可以通过 DEFAULT_CURRENCY_CODE 注入令牌进行配置。

@paramdisplaystring | boolean | undefined

货币指示符的格式。以下之一：

  • code：显示代码（例如 USD）。
  • symbol（默认）：显示符号（例如 $）。
  • symbol-narrow：对于货币具有两个符号的语言环境，请使用窄符号。 例如，加拿大元 CAD 的符号为 CA$，窄符号为 $。如果语言环境没有窄符号，则使用该语言环境的标准符号。
  • String：使用给定的字符串值，而不是代码或符号。 例如，空字符串将禁止显示货币和符号。
  • Boolean（在 v5 版本中标记为已弃用）：true 表示符号，false 表示代码。
@paramdigitsInfostring | undefined

十进制表示法选项，由以下格式的字符串指定：
{minIntegerDigits}.{minFractionDigits}-{maxFractionDigits}。

  • minIntegerDigits：小数点前的最少整数位数。 默认为 1。
  • minFractionDigits：小数点后的最少位数。 默认为 2。
  • maxFractionDigits：小数点后的最大小数位数。 默认为 2。 如果未提供，数字将根据 ISO 4217 的规定进行格式化，保留适当位数的小数。 例如，加拿大元有 2 位小数，而智利比索则没有。
@paramlocalestring | undefined

用于区域设置格式规则的区域设置代码。 未提供时，将使用 LOCALE_ID 的值，默认值为 en-US。 请参阅 设置您的应用区域设置。

用法说明

以下代码显示了管道如何根据各种格式规范将数字转换为文本字符串，其中调用者的默认语言环境为 en-US。

@Component({  selector: 'currency-pipe',  template: `<div>    <!--output '$0.26'-->    <p>A: {{ a | currency }}</p>    <!--output 'CA$0.26'-->    <p>A: {{ a | currency: 'CAD' }}</p>    <!--output 'CAD0.26'-->    <p>A: {{ a | currency: 'CAD' : 'code' }}</p>    <!--output 'CA$0,001.35'-->    <p>B: {{ b | currency: 'CAD' : 'symbol' : '4.2-2' }}</p>    <!--output '$0,001.35'-->    <p>B: {{ b | currency: 'CAD' : 'symbol-narrow' : '4.2-2' }}</p>    <!--output '0 001,35 CA$'-->    <p>B: {{ b | currency: 'CAD' : 'symbol' : '4.2-2' : 'fr' }}</p>    <!--output 'CLP1' because CLP has no cents-->    <p>B: {{ b | currency: 'CLP' }}</p>  </div>`,  standalone: false,})export class CurrencyPipeComponent {  a: number = 0.259;  b: number = 1.3495;}
