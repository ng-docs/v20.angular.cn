bootstrapApplication

Promise<ApplicationRef>

引导启动 Angular 应用的一个实例，并将一个独立组件渲染为该应用的根组件。有关独立组件的更多信息，请参见本指南。

@paramrootComponentType<unknown>

对要渲染的独立组件的引用。

@paramoptionsApplicationConfig | undefined

引导操作的额外配置，有关更多信息，请参阅ApplicationConfig。

@paramcontextBootstrapContext | undefined

可选的上下文对象，可用于提供预先存在的平台注入器。这对于高级用例很有用，例如服务器端渲染，其中为每个请求创建平台。

@returnsPromise<ApplicationRef>

用法说明

传入此函数的根组件必须是独立的（应在 @Component 装饰器配置中具有 standalone: true 标志）。

@Component({  standalone: true,  template: 'Hello world!'})class RootComponent {}const appRef: ApplicationRef = await bootstrapApplication(RootComponent);

你可以通过在作为第二个参数传递的对象中指定 providers 字段来添加应在应用注入器中可用的提供者列表：

await bootstrapApplication(RootComponent, {  providers: [    {provide: BACKEND_URL, useValue: 'https://yourdomain.com/api'}  ]});

importProvidersFrom 辅助方法可用于从任何现有的 NgModule（以及它导入的所有 NgModule 的传递依赖）中收集所有提供者：

await bootstrapApplication(RootComponent, {  providers: [    importProvidersFrom(SomeNgModule)  ]});

注意：bootstrapApplication方法默认不包含Testability。您可以通过使用provideProtractorTestingSupport()函数获取必要的提供者列表，并将它们添加到providers数组中来添加Testability，例如：

import {provideProtractorTestingSupport} from '@angular/platform-browser';await bootstrapApplication(RootComponent, {providers: [provideProtractorTestingSupport()]});
