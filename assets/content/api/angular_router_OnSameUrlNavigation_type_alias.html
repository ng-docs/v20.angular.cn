描述

如何处理对当前 URL 的导航请求。选项之一：

  • 'ignore'：如果请求与当前状态相同，路由器将忽略该请求。
  • 'reload'：即使 URL 与当前状态不同，路由器也会处理该 URL。 您可能想要使用此选项的一个示例是，如果 canMatch 守卫依赖于 应用状态，并且最初拒绝导航到某个路由。修复状态后，您希望 重新导航到同一 URL，以便具有 canMatch 守卫的路由可以激活。

请注意，这仅配置路由是否重新处理 URL 并触发相关的重定向、守卫和解析器等操作和事件。默认情况下，当路由器在不先访问不同组件的情况下重新导航到同一组件类型时，它会复用组件实例。此行为由 RouteReuseStrategy 配置。为了在导航到同一 URL 时重新加载路由组件，您需要将 onSameUrlNavigation 设置为 'reload'并提供一个为 shouldReuseRoute 返回 false 的 RouteReuseStrategy。此外，除非路径或路径参数已更改（由 runGuardsAndResolvers 配置），否则大多数路由的解析器和守卫不会运行。
