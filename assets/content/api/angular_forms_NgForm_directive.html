描述

创建一个顶级的 FormGroup 实例并将其绑定到表单以跟踪聚合表单值和验证状态。

导入 FormsModule 后，此指令将默认在所有 <form> 标签上激活。您无需添加特殊选择器。

您可以使用 ngForm 作为键（例如 #myForm="ngForm"）将该指令导出到本地模板变量。这是可选的，但很有用。底层 FormGroup 实例的许多属性都会在指令本身上重复，因此对它的引用使您可以访问表单的聚合值和有效性状态，以及 dirty 和 touched 等用户交互属性。

要将子控件注册到表单，请使用带有 name 属性的 NgModel。您可以使用 NgModelGroup 在表单内创建子组。

如有必要，请监听该指令的 ngSubmit 事件，以便在用户触发表单提交时收到通知。ngSubmit 事件会发出原始的表单提交事件。

在模板驱动表单中，所有 <form> 标签都会自动标记为 NgForm。要导入 FormsModule 但在某些表单中跳过其使用，例如使用原生 HTML5 验证，请添加 ngNoForm，这样 <form> 标签就不会创建 NgForm 指令。在响应式表单中，使用 ngNoForm 是不必要的，因为 <form> 标签是惰性的。在这种情况下，您将避免使用 formGroup 指令。

用法说明

监听表单提交

以下示例展示了如何从 “ngSubmit” 事件中捕获表单值。

import {Component} from '@angular/core';import {NgForm} from '@angular/forms';@Component({  selector: 'example-app',  template: `    <form #f="ngForm" (ngSubmit)="onSubmit(f)" novalidate>      <input name="first" ngModel required #first="ngModel" />      <input name="last" ngModel />      <button>Submit</button>    </form>    <p>First name value: {{ first.value }}</p>    <p>First name valid: {{ first.valid }}</p>    <p>Form value: {{ f.value | json }}</p>    <p>Form valid: {{ f.valid }}</p>  `,  standalone: false,})export class SimpleFormComp {  onSubmit(f: NgForm) {    console.log(f.value); // { first: '', last: '' }    console.log(f.valid); // false  }}

设置更新选项

以下示例向你展示了如何使用 ngFormOptions 从默认值更改 “updateOn” 选项。

<form [ngFormOptions]="{updateOn: 'blur'}">   <input name="one" ngModel>  <!-- this ngModel will update on blur --></form>

原生 DOM 验证 UI

为了防止原生 DOM 表单验证 UI 干扰 Angular 的表单验证，每当 FormModule 或 ReactiveFormModule 导入到应用中时，Angular 都会自动在任何 <form> 上添加 novalidate 属性。如果你想使用 Angular 表单显式启用原生 DOM 验证 UI，你可以将 ngNativeValidate 属性添加到 <form> 元素：

<form ngNativeValidate>  ...</form>
