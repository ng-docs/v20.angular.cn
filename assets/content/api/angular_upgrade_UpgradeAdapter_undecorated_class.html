downgradeNg2Component

Function

允许从 AngularJS 中使用 Angular 组件。

使用 downgradeNg2Component 从 Angular 组件创建一个 AngularJS 指令定义工厂。适配器将从 AngularJS 模板中引导 Angular 组件。

@paramcomponentType<any>
@returnsFunction
用法说明

心智模型

  1. 该组件通过在 AngularJS 模板中列出而被实例化。这意味着宿主元素由 AngularJS 控制，但组件的视图将由 Angular 控制。
  2. 即使组件是在 AngularJS 中实例化的，它也将使用 Angular 语法。 必须这样做，因为我们必须遵循 Angular 组件不声明应如何解释属性（Attribute）。
  3. ng-model 由 AngularJS 控制，并通过来自 ControlValueAccessor 接口的方式与降级的 Angular 组件进行通信

upgradeNg1Component

Type<any>

允许从 Angular 中使用 AngularJS 组件。

使用 upgradeNg1Component 从 AngularJS 组件指令创建一个 Angular 组件。适配器将从 Angular 模板中引导 AngularJS 组件。

@paramnamestring
@returnsType<any>
用法说明

心智模型

  1. 该组件通过在 Angular 模板中列出而被实例化。这意味着宿主元素由 Angular 控制，但组件的视图将由 AngularJS 控制。

支持的特性

  • 绑定（Bindings）：
    • 属性（Attribute）：<comp name="World">
    • 插值： <comp greeting="Hello {{name}}!">
    • 表达式： <comp [name]="username">
    • 事件： <comp (close)="doSomething()">
  • 内容嵌入：是
  • 仅支持 指令定义对象 的部分功能：
    • compile：不受支持，因为宿主元素归 Angular 所有，Angular 不允许在编译期间修改 DOM 结构。
    • controller：受支持。（注意：不支持注入 $attrs 和 $transclude。）
    • controllerAs：受支持。
    • bindToController：受支持。
    • link：受支持。（注意：仅支持 pre-link 函数。）
    • name：受支持。
    • priority：已忽略。
    • replace：不受支持。
    • require：受支持。
    • restrict：必须设置为 'E'。
    • scope：受支持。
    • template：受支持。
    • templateUrl：受支持。
    • terminal：已忽略。
    • transclude：受支持。

说明

使用 UpgradeAdapter 来允许 AngularJS 和 Angular 在单个应用中共存。

UpgradeAdapter 允许：

  1. 从 AngularJS 组件指令创建 Angular 组件 （参见 UpgradeAdapter#upgradeNg1Component）
  2. 从 Angular 组件创建 AngularJS 指令。 （参见 UpgradeAdapter#downgradeNg2Component）
  3. 引导启动一个混合的 Angular 应用，其中包含在单个应用中共存的两个框架。

用法说明

心智模型

当思考混合应用如何工作时，拥有一个心智模型会很有用，它可以描述正在发生的事情，并解释在最低级别上发生的事情。

  1. 在单个应用中运行着两个独立的框架，每个框架都将另一个框架视为黑盒。
  2. 页面上的每个 DOM 元素都恰好归一个框架所有。无论哪个框架实例化了该元素，该框架就是所有者。每个框架仅更新/与其自己的 DOM 元素交互，而忽略其他元素。
  3. 无论 AngularJS 指令在何处实例化，它们始终在 AngularJS 框架代码库内部执行。
  4. 无论 Angular 组件在何处实例化，它们始终在 Angular 框架代码库内部执行。
  5. AngularJS 组件可以升级为 Angular 组件。这将创建一个 Angular 指令，该指令在该位置引导启动 AngularJS 组件指令。
  6. Angular 组件可以降级为 AngularJS 组件指令。这将创建一个 AngularJS 指令，该指令在该位置引导启动 Angular 组件。
  7. 每当实例化适配器组件时，宿主元素都归执行实例化的框架所有。然后，另一个框架实例化并拥有该组件的视图。这意味着组件绑定将始终遵循实例化框架的语义。语法始终是 Angular 语法。
  8. AngularJS 始终首先被引导启动，并拥有最底层的视图。
  9. 新应用正在 Angular Zone 中运行，因此不再需要调用 $apply()。
