downgradeComponent

any

一个辅助函数，允许从 AngularJS 中使用 Angular 组件。

属于支持预先编译的混合升级应用的 upgrade/static 库的一部分

此辅助函数返回一个工厂函数，用于注册 AngularJS 包装器指令，以“降级” Angular 组件。

@paraminfo{ component: Type<any>; downgradedModule?: string | undefined; propagateDigest?: boolean | undefined; inputs?: string[] | undefined; outputs?: string[] | undefined; selectors?: string[] | undefined; }

包含有关正在降级的组件的信息：

  • component: Type<any>：要降级的组件的类型
  • downgradedModule?: string：组件所属的已降级模块的名称（如果有），由调用 downgradeModule() 返回。当需要实例化该组件时，它将是引导（bootstrap）相应的 Angular 模块的模块。

（仅当使用 downgradeModule() 降级多个 Angular 模块时，此选项才是必要的。） - `propagateDigest?: boolean`：是否在每次 $digest 时，对组件执行 detectChanges。 如果设置为 `false`，则当任何组件的输入发生更改时，仍将执行变更检测。 （默认值：true）
@returnsany

描述

一个辅助函数，允许从 AngularJS 中使用 Angular 组件。

属于支持预先编译的混合升级应用的 upgrade/static 库的一部分

此辅助函数返回一个工厂函数，用于注册 AngularJS 包装器指令，以“降级” Angular 组件。

用法说明

范例

假设你有一个名为 ng2Heroes 的 Angular 组件，需要在 AngularJS 模板中可用。

// This Angular component will be "downgraded" to be used in AngularJS@Component({  selector: 'ng2-heroes',  // This template uses the upgraded `ng1-hero` component  // Note that because its element is compiled by Angular we must use camelCased attribute names  template: `<header><ng-content selector="h1"></ng-content></header>    <ng-content selector=".extra"></ng-content>    <div *ngFor="let hero of heroes()">      <ng1-hero [hero]="hero" (onRemove)="removeHero.emit(hero)"        ><strong>Super Hero</strong></ng1-hero      >      </div>    <button (click)="addHero.emit()">Add Hero</button>`,  standalone: false,})export class Ng2HeroesComponent {  heroes = input<Hero[]>([]);  addHero = output<void>();  removeHero = output<Hero>();}

我们必须创建一个 AngularJS 指令，使此 Angular 组件在 AngularJS 模板中可用。 downgradeComponent() 函数返回一个工厂函数，我们可以使用它来定义包装“已降级”组件的 AngularJS 指令。

// This directive will act as the interface to the "downgraded" Angular componentng1AppModule.directive('ng2Heroes', downgradeComponent({component: Ng2HeroesComponent}));
