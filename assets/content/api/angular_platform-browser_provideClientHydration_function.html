provideClientHydration

EnvironmentProviders

建立必要的提供者，以启用应用的 hydration 功能。

默认情况下，此函数为大多数应用启用推荐的一组特性，以获得最佳性能。它包括以下特性：

  • 协调 DOM 水合。要了解更多信息，请点击这里。
  • 在服务器上运行时对 HttpClient 响应进行缓存，并将此缓存传输到客户端，以避免额外的 HTTP 请求。要了解关于数据缓存的更多信息，请点击这里。

这些函数允许你禁用一些默认特性或启用新的特性：

  • withNoHttpTransferCache 以禁用 HTTP 传输缓存
  • withHttpTransferCacheOptions 以配置一些 HTTP 传输缓存选项
  • withI18nSupport 以启用对 i18n 控制块的水合支持
  • withEventReplay 以启用对重放用户事件的支持

用法说明

当使用 bootstrapApplication 函数时，在应用中启用水合的基本示例：

bootstrapApplication(AppComponent, {  providers: [provideClientHydration()]});

或者，如果您正在使用 NgModules，您可以在根应用模块的提供者列表中添加 provideClientHydration。

@NgModule({  declarations: [RootCmp],  bootstrap: [RootCmp],  providers: [provideClientHydration()],})export class AppModule {}
