属性装饰器，用于配置内容查询。

API

@ContentChild(
  selector: ProviderToken<unknown> | Function | string,
  opts?: { descendants?: boolean; read?: any; static?: boolean },
);

描述

属性装饰器，用于配置内容查询。

用于从内容 DOM 中获取与选择器匹配的首个元素或指令。 如果内容 DOM 发生变化，并且新的子条目与选择器匹配，则该属性（Property）将会更新。

不会检索其他组件模板中的元素或指令，因为对于组件的祖先组件来说，组件的模板始终是一个黑盒。

元数据属性（Property）：

  • selector - 用于查询的指令类型或名称。
  • descendants - 如果为 true (默认值)，则包括该元素的所有后代。如果为 false，则仅查询该元素的直接子条目。
  • read - 用于从查询到的元素中读取不同的令牌。
  • static - 如果为 true，则在变更检测运行之前求解查询结果；如果为 false，则在变更检测之后求解。默认为 false。

支持以下选择器。

  • 带有 @Component 或 @Directive 装饰器的任何类
  • 模板引用变量（字符串形式），例如，使用 @ContentChild('cmp') 查询 <my-component #cmp></my-component>
  • 当前组件的子组件树中定义的任何提供者 (例如 @ContentChild(SomeService) someService: SomeService)
  • 通过字符串令牌定义的任何提供者 (例如 @ContentChild('someToken') someTokenVal: any)
  • 一个 TemplateRef（例如，使用 @ContentChild(TemplateRef) template; 查询 <ng-template></ng-template>）

read 支持以下值：

  • 带有 @Component 或 @Directive 装饰器的任何类
  • 此查询的 selector 所匹配的组件的注入器上定义的任何提供者
  • 通过字符串令牌定义的任何提供者 (例如 {provide: 'token', useValue: 'val'})
  • TemplateRef、ElementRef 和 ViewContainerRef

动态查询和静态查询之间的区别：

查询 详情
动态查询 (static: false) 查询在 ngAfterContentInit() 之前求解
回调被调用。结果会随着视图的变化而更新，例如
ngIf 和 ngFor 控制块的变化。
视图已创建之后，但在变更检测运行之前（在 ngOnInit() 回调之前）
被调用）。但是，结果永远不会更新以反映视图的变化，例如
ngIf 和 ngFor 控制块的变化。
