onChange

void

为 URL 更改注册侦听器。此 API 用于捕获 AngularJS 框架执行的更新。这些更改是 $locationChangeStart 和 $locationChangeSuccess 事件的子集，当 AngularJS 更新其内部引用的浏览器 URL 版本时，会触发这些事件。

$locationChange 事件可能发生，但浏览器 URL (window.location) 保持不变。只有当 AngularJS 实际更新浏览器 URL (window.location) 时，才会触发此 onChange 回调。

@paramfn(url: string, state: unknown, oldUrl: string, oldState: unknown) => void

@paramerr(e: Error) => void

在当前 $digest 期间对 $location 的更改将替换当前历史记录，而不是添加新的记录。

@returnsthis

state

2 个重载

当在没有参数的情况下调用时，获取历史状态对象。

当使用一个参数调用时，更改历史状态对象并返回 $location。状态对象稍后会传递给 pushState 或 replaceState。

此方法仅在 HTML5 模式下受支持，且仅在支持 HTML5 History API 方法（如 pushState 和 replaceState）的浏览器中受支持。如果您需要支持旧版浏览器（如 Android < 4.0），请勿使用此方法。
