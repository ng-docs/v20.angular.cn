使用说明

启动过程

当启动组件时，Angular 会将其挂载到目标 DOM 元素上，并启动自动变更检测。可以使用 rootSelectorOrNode 参数提供目标 DOM 元素。

如果未提供目标 DOM 元素，Angular 将尝试在页面上使用正在启动的组件的 selector 查找一个（使用第一个匹配的元素）。

tick

void

调用此方法以显式处理变更检测及其副作用。

在开发模式下，tick() 还会执行第二个变更检测周期，以确保未检测到更多更改。如果在第二个周期中检测到其他更改，则应用中的绑定具有无法在单个变更检测过程中求解的副作用。 在这种情况下，Angular 会抛出一个错误，因为 Angular 应用在其中只能有一个变更检测过程，所有变更检测都必须在此过程中完成。

使用须知

isStable 范例和注意事项

请注意关于 isStable 的两个要点，如下面的范例所示：

  • 如果在应用启动时启动任何类型的周期性异步任务（例如，轮询过程，使用 setInterval、setTimeout 或使用 RxJS 操作符（如 interval）启动），则应用将永远不会稳定；
  • isStable 可观察者在 Angular Zone 之外运行。

假设你启动了一个周期性任务（此处使用 RxJS interval 递增计数器），同时订阅了 isStable。

constructor(appRef: ApplicationRef) {  appRef.isStable.pipe(     filter(stable => stable)  ).subscribe(() => console.log('App is stable now');  interval(1000).subscribe(counter => console.log(counter));}

在此范例中，isStable 将永远不会发出 true，并且永远不会记录跟踪信息“App is stable now”。

如果要在应用稳定时执行某些操作，则必须等待应用稳定后再启动轮询过程。

constructor(appRef: ApplicationRef) {  appRef.isStable.pipe(    first(stable => stable),    tap(stable => console.log('App is stable now')),    switchMap(() => interval(1000))  ).subscribe(counter => console.log(counter));}

在此范例中，将记录跟踪信息“App is stable now”，然后计数器开始每秒递增。

另请注意，此可观察者在 Angular Zone 之外运行，这意味着订阅此可观察者的代码不会触发变更检测。

假设你不是记录计数器值，而是更新组件的字段并在其模板中显示它。

constructor(appRef: ApplicationRef) {  appRef.isStable.pipe(    first(stable => stable),    switchMap(() => interval(1000))  ).subscribe(counter => this.value = counter);}

由于 isStable 可观察者在 Zone 之外运行，因此 value 字段将正确更新，但模板不会刷新！

你必须手动触发变更检测以更新模板。

constructor(appRef: ApplicationRef, cd: ChangeDetectorRef) {  appRef.isStable.pipe(    first(stable => stable),    switchMap(() => interval(1000))  ).subscribe(counter => {    this.value = counter;    cd.detectChanges();  });}

或者使订阅回调在 Zone 内部运行。

constructor(appRef: ApplicationRef, zone: NgZone) {  appRef.isStable.pipe(    first(stable => stable),    switchMap(() => interval(1000))  ).subscribe(counter => zone.run(() => this.value = counter));}
