valid

enabled

errors

ValidationErrors | null

一个对象，其中包含因验证失败而产生的任何错误；如果没有错误，则为 null。

pristine

boolean

如果用户尚未更改 UI 中的值，则控件是 pristine 的。

dirty

boolean

如果用户已更改 UI 中的值，则控件是 dirty 的。

touched

boolean

如果控件被标记为 touched，则为 True。

一旦用户在其上触发了 blur 事件，控件就会被标记为 touched。

untouched

boolean

如果控件未被标记为 touched，则为 True

如果用户尚未在其上触发 blur 事件，则控件是 untouched 的。

events

any

一个多播的可观察对象，每当控件状态更改时都会发出事件。 它会在值、状态、pristine 或 touched 更改时发出。

注意：在值更改时，发出事件的操作会在此控件的值更新之后立即发生。父控件的值（例如，如果此 FormControl 是 FormGroup 的一部分）会稍后更新，因此从此事件的回调中访问父控件的值（使用 value 属性）可能会导致获得尚未更新的值。请改为订阅父控件的 events。 对于其他事件类型，事件会在父控件更新后发出。

valueChanges

Observable<TValue>

一个多播的可观察对象，每当控件的值在 UI 中或以编程方式更改时，都会发出事件。 每次你调用 enable() 或 disable() 而没有传递 {emitEvent: false} 作为函数参数时，它也会发出一个事件。

注意：发出事件的操作会在此控件的值更新之后立即发生。父控件的值（例如，如果此 FormControl 是 FormGroup 的一部分）会稍后更新，因此从此事件的回调中访问父控件的值（使用 value 属性）可能会导致获得尚未更新的值。请改为订阅父控件的 valueChanges 事件。

statusChanges

Observable<FormControlStatus>

一个多播的可观察对象，每当控件的验证 status 重新计算时，都会发出事件。

updateOn

FormHooks

用法说明

例如，对于以下 FormGroup：

form = new FormGroup({  address: new FormGroup({ street: new FormControl() })});

从根表单到 'street' 控件的路径是 'address' -> 'street'。

它可以采用以下两种格式之一提供给此方法：

  1. 字符串控件名称的数组，例如 ['address', 'street']
  2. 一个字符串中以句点分隔的控件名称列表，例如 'address.street'

描述

跟踪一组 FormControl 实例的值和有效性状态，这些实例具有相同的值类型。

FormRecord 与 FormGroup 非常相似，不同之处在于它可以用于动态键，并根据需要添加和删除控件。

FormRecord 接受一个泛型参数，该参数描述了它所包含的控件的类型。

用法说明

let numbers = new FormRecord({bill: new FormControl('415-123-456')});numbers.addControl('bob', new FormControl('415-234-567'));numbers.removeControl('bill');
