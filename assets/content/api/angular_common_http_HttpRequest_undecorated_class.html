body

T | null

请求体；如果未设置，则为 null。

请求体并非强制不可变的，因为它们可以包含对任何用户定义数据类型的引用。但是，拦截器应注意通过将它们视为不可变来保持幂等性。

headers

HttpHeaders

此请求的传出标头。

context

HttpContext

拦截器可以使用的共享且可变上下文

reportProgress

boolean

是否应以公开进度事件的方式发出此请求。

进度事件开销很大（变更检测会在每个事件上运行），因此只有当使用者打算监视它们时才应请求它们。

注意：FetchBackend 不支持上传进度报告。

withCredentials

boolean

是否应随传出凭据（Cookie）一起发送此请求。

credentials

RequestCredentials

请求的凭证模式，它决定了如何处理 cookie 和 HTTP 身份验证。 这会影响是否将 cookie 与请求一起发送，以及如何处理身份验证。

keepalive

boolean

当使用 fetch 实现并设置为 true 时，如果发起请求的页面在请求完成之前被卸载，浏览器将不会中止关联的请求。

cache

RequestCache

控制请求如何与浏览器的 HTTP 缓存进行交互。 这会影响是否从缓存中检索响应，如何存储响应，或者是否完全绕过缓存。

priority

RequestPriority

指示请求的相对优先级。浏览器可能会使用此信息来决定分派请求和获取资源的顺序。

mode

RequestMode

请求的模式，它决定了请求将如何与浏览器的安全模型进行交互。 这会影响 CORS（跨域资源共享）和同源策略等事项。

redirect

RequestRedirect

请求的重定向模式，它决定了如何处理重定向。 这会影响请求是自动跟随重定向，还是在发生重定向时失败。

referrer

string

请求的引用者，可用于指示请求的来源。 这对于安全和分析目的很有用。 值为同源 URL、"about:client" 或空字符串，用于设置请求的引用者。

integrity

string

请求的完整性元数据，可用于确保请求是以预期内容发出的。 要获取的资源的加密哈希值

responseType

"arraybuffer" | "blob" | "text" | "json"

服务器的预期响应类型。

这用于在将响应返回给请求者之前对其进行适当的解析。

method

string

传出的 HTTP 请求方法。

参数

HttpParams

发出的 URL 参数。

要以 URL 查询字符串格式传递 HTTP 参数的字符串表示形式，可以使用 HttpParamsOptions 的 fromString。例如：

new HttpParams({fromString: 'angular=awesome'})

urlWithParams

string

设置了所有 URL 参数的传出 URL。

transferCache

boolean | { includeHeaders?: string[] | undefined; } | undefined

请求的 HttpTransferCache 选项

timeout

number | undefined

后端 HTTP 请求的超时时间（以毫秒为单位）。

serializeBody

string | ArrayBuffer | Blob | FormData | URLSearchParams | null

将自由格式的主体转换为适合传输到服务器的序列化格式。

描述

一个发出的 HTTP 请求，带有可选的类型化主体。

HttpRequest 表示一个发出的请求，包括 URL、方法、标头、正文和其他请求配置选项。实例应假定为不可变的。要修改 HttpRequest，应使用 clone 方法。
