@angular/core

指令

decorator
代码

用来将某个类标记为 Angular 指令的装饰器。你可以定义自己的指令，以便将自定义行为附着到 DOM 中的元素。

API

@Directive({
  selector?: string | undefined;
  inputs?:
    | (
        | string
        | {
            name: string;
            alias?: string | undefined;
            required?: boolean | undefined;
            transform?: ((value: any) => any) | undefined;
          }
      )[]
    | undefined;
  outputs?: string[] | undefined;
  providers?: Provider[] | undefined;
  exportAs?: string | undefined;
  queries?: { [key: string]: any } | undefined;
  host?: { [key: string]: string } | undefined;
  jit?: true | undefined;
  standalone?: boolean | undefined;
  hostDirectives?:
    | (
        | Type<unknown>
        | {
            directive: Type<unknown>;
            inputs?: string[] | undefined;
            outputs?: string[] | undefined;
          }
      )[]
    | undefined;
})

描述

用来将某个类标记为 Angular 指令的装饰器。你可以定义自己的指令，以便将自定义行为附着到 DOM 中的元素。

这些选项提供了配置元数据，这些元数据决定了指令在运行时应如何处理、实例化和使用。

指令类（如组件类）可以实现生命周期钩子，以影响其配置和行为。

使用须知

要定义指令，请使用装饰器标记该类并提供元数据。

import {Directive} from '@angular/core';@Directive({  selector: 'my-directive',})export class MyDirective {...}

声明指令

默认情况下，指令被标记为独立，这使得它们可供应用程序中的其他组件使用。

@Directive({  selector: 'my-directive',})

请确保标记为独立的指令尚未在 NgModule 中声明。

** 在 NgModule 中声明指令 ** 如果你想在 ngModule 中声明一个指令，请在 Directive 装饰器元数据中添加 standalone: false 标志，并将该指令添加到你的 ngModule 的 declarations 和 exports 字段中。

@Directive({  standalone: false,  selector: 'my-directive',})class MyDirective {}@NgModule({  declarations: [MyDirective, SomeComponent],  exports: [MyDirective], // making it available outside of this module})class SomeNgModule {}

在 NgModule 中声明指令时，请确保：

  • 该指令恰好在一个 NgModule 中声明。
  • 该指令不是 standalone 的。
  • 你不要重新声明从另一个模块导入的指令。
  • 如果要使此指令可供 NgModule 外部的组件访问，则该指令也包含在 exports 字段中。
