@angular/core

RendererType2

interface
stable
代码

供 RendererFactory2 使用，将自定义渲染数据和样式与渲染实现关联起来。

API

    
      interface RendererType2 {}

id

string

新渲染器的唯一标识字符串，用于在为封装创建唯一样式时。

encapsulation

ViewEncapsulation

视图封装类型，它决定了样式如何应用于 DOM 元素。它是以下类型之一

  • Emulated（默认值）：模拟样式的原生作用域。
  • Native：使用渲染器的原生封装机制。
  • ShadowDom：使用现代的 Shadow DOM 并为组件的宿主元素创建 ShadowRoot。
  • None：不提供任何模板或样式封装。

styles

string[]

定义要存储在渲染器实例上的 CSS 样式。

data

{ [kind: string]: any; }

定义要存储在渲染器实例上的任意开发者定义的数据。 这对于委托给其他渲染器的渲染器很有用。

getExternalStyles

((encapsulationId?: string | undefined) => string[]) | null | undefined

框架使用此函数来创建外部运行时样式 URL 列表。
