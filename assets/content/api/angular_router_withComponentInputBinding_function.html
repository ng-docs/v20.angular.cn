withComponentInputBinding

ComponentInputBindingFeature

使路由（Router）状态中的信息能够直接绑定到路由（Route）配置中组件的输入（inputs）。

@returnsComponentInputBindingFeature

用法说明

如何启用此特性的基本范例：

const appRoutes: Routes = [];bootstrapApplication(AppComponent,  {    providers: [      provideRouter(appRoutes, withComponentInputBinding())    ]  });

来自以下任何来源的路由绑定信息：

  • 查询参数
  • 路径和矩阵参数
  • 静态路由数据
  • 来自求解器的数据

重复的键会按照从上到下的相同顺序求解，从最低到最高，这意味着求解器具有最高的优先级，并将覆盖来自路由的任何其他信息。

重要的是，当输入属性（Property）在路由数据中没有具有匹配键的条目时，此输入属性（Property）将被设置为 undefined。这可以防止在数据从路由中删除后（例如，如果删除了查询参数）保留先前的信息。 可以使用路由上的求解器来提供默认值，以确保该值始终存在，或者在组件中使用输入属性（Property）和输入转换。
