AbstractControl响应式表单指南用法说明

API

}

defaultValue

TValue

每当控件在没有显式值的情况下重置时，将使用此 FormControl 的默认值。有关配置默认值的更多信息，请参阅FormControlOptions#nonNullable。

setValue

void

为表单控件设置一个新值。

@paramvalueTValue

控件的新值。

@paramoptions{ onlySelf?: boolean | undefined; emitEvent?: boolean | undefined; emitModelToViewChange?: boolean | undefined; emitViewToModelChange?: boolean | undefined; } | undefined

配置选项，用于确定当值更改时控件如何传播更改和发出事件。 配置选项将传递给 * updateValueAndValidity 方法。

  • onlySelf：为 true 时，每次更改仅影响此控件，而不影响其父控件。默认为 false。
  • emitEvent：当为 true 或未提供（默认值）时，当控件值更新时，statusChanges 和 valueChanges 可观察对象都会发出包含最新状态和值的事件。 当为 false 时，不发出任何事件。
  • emitModelToViewChange：当为 true 或未提供（默认值）时，每次更改都会触发 onChange 事件以更新视图。
  • emitViewToModelChange：当为 true 或未提供（默认值）时，每次更改都会触发 ngModelChange 事件以更新模型。
@returnsvoid

@paramvalueTValue

reset

void

重置表单控件，将其标记为 pristine 和 untouched，并重置该值。新值将是所提供的值（如果已传递）、null 或初始值（如果在构造函数中通过 FormControlOptions 设置了 nonNullable）。

// By default, the control will reset to null.const dog = new FormControl('spot');dog.reset(); // dog.value is null// If this flag is set, the control will instead reset to the initial value.const cat = new FormControl('tabby', {nonNullable: true});cat.reset(); // cat.value is "tabby"// A value passed to reset always takes precedence.const fish = new FormControl('finn', {nonNullable: true});fish.reset('bubble'); // fish.value is "bubble"
@paramformStateTValue | FormControlState<TValue> | undefined

父控件。

status

FormControlStatus

控件的验证状态。

valid

boolean

当控件的 status 为 VALID 时，该控件为 valid。

invalid

boolean

当控件的 status 为 INVALID 时，该控件为 invalid。

pending

boolean

当控件的 status 为 PENDING 时，该控件为 pending。

disabled

boolean

当控件的 status 为 DISABLED 时，该控件为 disabled。

禁用的控件无需进行验证检查，并且不包含在其祖先控件的聚合值中。

enabled

boolean

只要控件的 status 不是 DISABLED，该控件就为 enabled。

errors

ValidationErrors | null

一个对象，其中包含因验证失败而生成的任何错误；如果没有错误，则为 null。

pristine

boolean

如果用户尚未更改 UI 中的值，则控件为 pristine。

dirty

boolean

如果用户已更改 UI 中的值，则控件为 dirty。

touched

boolean

如果控件标记为 touched，则为 True。

一旦用户在其上触发了 blur 事件，控件就会被标记为 touched。

untouched

boolean

如果控件尚未标记为 touched，则为 True

如果用户尚未在其上触发 blur 事件，则控件为 untouched。

events

any

一个多播可观察对象，每次控件状态更改时都会发出事件。 它会针对值、状态、pristine 或 touched 更改发出。

注意：在值更改时，发出操作在此控件的值更新后立即发生。父控件的值（例如，如果此 FormControl 是 FormGroup 的一部分）稍后更新，因此从此事件的回调中访问父控件的值（使用 value 属性）可能会导致获取尚未更新的值。请改为订阅父控件的 events。 对于其他事件类型，事件在父控件更新后发出。

valueChanges

Observable<TValue>

这是一个多播可观察对象，每当控件的值在 UI 中或以编程方式更改时，它都会发出一个事件。 每次你调用 enable() 或 disable() 而不传递 {emitEvent: false} 作为函数参数时，它也会发出一个事件。

注意：发出事件发生在控件的值更新之后。父控件的值（例如，如果此 FormControl 是 FormGroup 的一部分）会稍后更新，因此从此事件的回调中访问父控件的值（使用 value 属性）可能会导致获得尚未更新的值。请改为订阅父控件的 valueChanges 事件。

statusChanges

Observable<FormControlStatus>

这是一个多播可观察对象，每当控件的验证 status 重新计算时，它都会发出一个事件。

updateOn

FormHooks

报告 AbstractControl 的更新策略（即控件自行更新的事件）。 可能的值：'change' | 'blur' | 'submit' 默认值：'change'

控件名称列表，用于指定如何从当前控件移动到应查询错误的控件。

@returnsboolean
用法说明

例如，对于以下 FormGroup：

form = new FormGroup({  address: new FormGroup({ street: new FormControl() })});

从根表单到 'street' 控件的路径将是 'address' -> 'street'。

可以通过以下两种格式之一提供给此方法：

  1. 字符串控件名称的数组，例如 ['address', 'street']
  2. 一个字符串中以句点分隔的控件名称列表，例如 'address.street'

如果未提供路径，则此方法检查当前控件上的错误。

root

AbstractControl<any, any, any>

检索此控件的顶层祖先控件。

描述

跟踪单个表单控件的值和验证状态。

这是 Angular 表单的四个基本构建块之一，另外三个是FormGroup、FormArray 和 FormRecord。它扩展了 AbstractControl 类，该类实现了大多数用于访问值、验证状态、用户交互和事件的基础功能。

FormControl 接受一个泛型参数，该参数描述其值的类型。此参数始终隐式包含 null，因为控件可以被重置。要更改此行为，请设置 nonNullable 或参阅下面的使用说明。

请参阅下面的用法范例。

用法说明

初始化表单控件

实例化一个 FormControl，并带有一个初始值。

const control = new FormControl('some value');console.log(control.value);     // 'some value'

以下范例使用表单状态对象初始化控件。在这种情况下，value 和 disabled 键是必要的。

const control = new FormControl({ value: 'n/a', disabled: true });console.log(control.value);     // 'n/a'console.log(control.status);    // 'DISABLED'

以下范例使用同步验证器初始化控件。

const control = new FormControl('', Validators.required);console.log(control.value);      // ''console.log(control.status);     // 'INVALID'

以下范例使用选项对象初始化控件。

const control = new FormControl('', {   validators: Validators.required,   asyncValidators: myAsyncValidator});

单一类型参数

FormControl 接受一个泛型参数，该参数描述其值的类型。 在大多数情况下，此参数将被推断。

如果要将控件初始化为 null，或者希望提供更广泛的类型，则可以显式指定参数：

let fc = new FormControl<string|null>(null);fc.setValue('foo');

您可能会注意到 null 始终添加到控件的类型中。这是因为如果您调用 reset，控件将变为 null。您可以通过设置 {nonNullable: true} 来更改此行为。

配置控件以在失焦事件时更新

将 updateOn 选项设置为 'blur' 以在失焦 event 时更新。

const control = new FormControl('', { updateOn: 'blur' });

配置控件以在提交事件时更新

将 updateOn 选项设置为 'submit' 以在提交 event 时更新。

const control = new FormControl('', { updateOn: 'submit' });

将控件重置为特定值

您可以通过传递独立值或包含值和禁用状态的表单状态对象来重置为特定的表单状态（这些是仅有的两个无法计算的属性）。

const control = new FormControl('Nancy');console.log(control.value); // 'Nancy'control.reset('Drew');console.log(control.value); // 'Drew'

将控件重置为其初始值

如果您希望始终将控件重置为其初始值（而不是 null），则可以传递 nonNullable 选项：

const control = new FormControl('Nancy', {nonNullable: true});console.log(control.value); // 'Nancy'control.reset();console.log(control.value); // 'Nancy'

将控件重置为初始值和禁用状态 

const control = new FormControl('Nancy');console.log(control.value); // 'Nancy'console.log(control.status); // 'VALID'control.reset({ value: 'Drew', disabled: true });console.log(control.value); // 'Drew'console.log(control.status); // 'DISABLED'
