DomSanitizer 通过对值进行无害化处理，使其在不同的 DOM 上下文中安全使用，从而帮助防止跨站脚本安全漏洞 (XSS)。

API

sanitize

string | null

从已知安全的值或安全性未知的值中获取安全值。

如果给定值已经是 SafeValue，则此方法返回解包后的值。 如果安全上下文是 HTML 且给定值是普通字符串，则此方法会 sanitize 该字符串，移除任何潜在不安全的内容。 对于任何其他安全上下文，如果提供普通字符串，此方法将抛出错误。

@paramcontextSecurityContext
@paramvaluestring | SafeValue | null
@returnsstring | null

bypassSecurityTrustHtml

SafeHtml

绕过安全性并信任给定的值是安全的 HTML。仅当绑定的 HTML 不安全（例如，包含 <script> 标记）并且代码应该执行时才使用此方法。无害化处理程序将保持安全 HTML 完好无损，因此在大多数情况下不应使用此方法。

警告： 使用不受信任的用户数据调用此方法会将你的应用暴露于 XSS 安全风险！

@paramvaluestring
@returnsSafeHtml

说明

DomSanitizer 通过对值进行无害化处理，使其在不同的 DOM 上下文中安全使用，从而帮助防止跨站脚本安全漏洞 (XSS)。

例如，当在 <a [href]="someValue"> 超链接中绑定 URL 时，将对 someValue 进行无害化处理，以使攻击者无法注入例如 javascript: URL，从而在网站上执行代码。

在特定情况下，可能有必要禁用无害化处理，例如，如果应用确实需要生成带有动态值的 javascript: 样式链接。 用户可以通过使用 bypassSecurityTrust... 方法之一构造一个值来绕过安全性，然后从模板绑定到该值。

这些情况应该非常少见，并且必须格外小心，以避免创建跨站脚本 (XSS) 安全漏洞！

当使用 bypassSecurityTrust... 时，请确保尽早调用该方法，并尽可能靠近值的来源，以便轻松验证其使用不会产生安全漏洞。

如果值是安全的，则不需要（也不建议）绕过安全性，例如，不以可疑协议开头的 URL，或者不包含危险代码的 HTML 片段。无害化处理程序会保持安全值完好无损。
