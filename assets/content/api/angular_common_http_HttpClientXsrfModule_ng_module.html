弃用警告

如果希望禁用 XSRF 保护，请改用 withXsrfConfiguration({cookieName: 'XSRF-TOKEN', headerName: 'X-XSRF-TOKEN'}) 作为提供者，或使用 withNoXsrfProtection。

@paramoptions{ cookieName?: string | undefined; headerName?: string | undefined; }

一个可以指定 Cookie 名称和/或标头名称的对象。

  • Cookie 名称默认为 XSRF-TOKEN。
  • 标头名称默认为 X-XSRF-TOKEN。
@returnsModuleWithProviders<HttpClientXsrfModule>

说明

为发出的请求配置 XSRF 保护支持。

对于支持基于 Cookie 的 XSRF 保护系统的服务器，请直接使用它来配置具有正确 Cookie 和标头名称的 XSRF 保护。

如果未提供任何名称，则默认 Cookie 名称为 XSRF-TOKEN，默认标头名称为 X-XSRF-TOKEN。
