一个可注入的 HammerJS Manager，用于手势识别。配置特定的事件识别。

弃用警告

HammerJS 集成已被弃用。请替换为您自己的实现。

API

    
      class HammerGestureConfig {}

events

string[]

一组用于 Angular 的手势支持的事件名称。 Angular 支持所有内置的识别器，如 HammerJS 文档中所列。

overrides

{ [key: string]: Object; }

将手势事件名称映射到一组配置选项，这些选项指定对特定属性的默认值的重写。

键是要配置的受支持的事件名称，选项对象包含一组属性，其中包含要应用于命名识别器事件的重写值。例如，要禁用 rotate 事件的识别，请指定 {"rotate": {"enable": false}}。

不存在的属性将采用 HammerJS 的默认值。 有关哪些属性支持哪些事件以及它们的允许值和默认值的信息，请参阅 HammerJS 文档。

options

{ cssProps?: any; domEvents?: boolean | undefined; enable?: boolean | ((manager: any) => boolean) | undefined; preset?: any[] | undefined; touchAction?: string | undefined; recognizers?: any[] | undefined; inputClass?: any; inputTarget?: EventTarget | undefined; } | undefined

可以为给定事件重写其默认值的属性。 不同的属性集适用于不同的事件。 有关哪些属性支持哪些事件以及它们的允许值和默认值的信息，请参阅 HammerJS 文档。
