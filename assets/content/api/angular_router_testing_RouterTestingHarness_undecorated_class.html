触发 Router 导航并等待其完成。

带有为该工具创建的 RouterOutlet 的根组件用于渲染 Route 组件。在后续对 navigateForTest 的调用中，根组件将在同一测试中复用。

当测试带有拒绝导航的守卫的 Routes 时，RouterOutlet 可能未激活，并且 activatedComponent 可能为 null。

触发路由器导航并等待其完成。

带有为该工具创建的 RouterOutlet 的根组件用于渲染 Route 组件。

{@example router/testing/test/router_testing_harness_examples.spec.ts region='RoutedComponent'}

在同一测试中后续调用 navigateByUrl 时，根组件会被复用。

此函数还使测试依赖于 ActivatedRoute 数据的组件变得更容易。

{@example router/testing/test/router_testing_harness_examples.spec.ts region='ActivatedRoute'}

@paramurlstring

导航的目标。传递给 Router.navigateByUrl。

@paramrequiredRoutedComponentTypeType<T>

导航完成后，RouterOutlet 的已激活组件的必要类型。如果出口未激活或激活了不同的组件，此函数将抛出一个错误。

@returnsPromise<T>

create

Promise<RouterTestingHarness>

创建一个 RouterTestingHarness 实例。

该 RouterTestingHarness 还创建了自己的带有 RouterOutlet 的根组件，用于渲染路由组件。

如果已创建实例，则抛出错误。 使用此工具还需要在 ModuleTeardownOptions 中设置 destroyAfterEach: true
