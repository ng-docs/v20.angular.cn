用于在验证表达式在被检查后未发生更改时禁用详尽检查。

这意味着未被标记为检查的 OnPush 组件将不会被检查。 此行为是 Angular 当前的默认行为。在视图树上运行变更检测时， 被标记为检查的视图会被刷新，并且用于检查它的标志会被移除。 当 Angular 第二次检查视图以确保没有任何变化时，OnPush 组件将不再被标记，也不会被检查。

@paramoptions{ exhaustive: false; }
@returnsEnvironmentProviders
function provideCheckNoChangesConfig(options: { interval?: number | undefined; exhaustive: true; }): EnvironmentProviders;
  • interval 会定期在应用视图上运行 checkNoChanges。这在无区域（zoneless）的应用中可能很有用， 可以定期确保没有发生更改，而无需通知 Angular 模板需要刷新。
  • 详尽（exhaustive）选项在验证表达式未发生更改时，会将所有应用视图视为 ChangeDetectionStrategy.Default。 所有附加到 ApplicationRef 的视图以及这些视图的所有后代都将被检查更改（不包括通过 ChangeDetectorRef.detach() 分离的子树）。 这很有用，因为在常规变更检测之后运行的检查不适用于使用 ChangeDetectionStrategy.OnPush 的组件。 此检查将暴露由 OnPush 组件隐藏的任何现有错误。
