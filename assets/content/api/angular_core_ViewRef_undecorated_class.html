变更检测用法

API

    
      abstract class ViewRef extends ChangeDetectorRef {}

destroy

void

销毁此视图以及所有与之关联的数据结构。

@returnsvoid

destroyed

boolean

报告此视图是否已被销毁。

onDestroy

void

一个生命周期钩子，为视图提供额外的开发者定义清理功能。

@paramcallbackFunction

一个处理函数，用于清理与视图关联的开发者定义数据。当调用 destroy() 方法时调用。

@returnsvoid

markForCheck

void

当视图使用 ChangeDetectionStrategy#OnPush (checkOnce) 变更检测策略时，显式地将该视图标记为已更改，以便可以再次检查它。

当输入属性已更改或视图中已触发事件时，组件通常会被标记为脏（需要重新渲染）。调用此方法以确保即使这些触发器未发生，也会检查组件。

@returnsvoid

detach

void

将此视图从变更检测树中分离。 分离的视图在重新附着之前不会被检查。 与 detectChanges() 结合使用以实现本地变更检测检查。

分离的视图在变更检测运行期间不会被检查，直到它们被重新附着，即使它们被标记为脏也是如此。

@returnsvoid

detectChanges

void

检查此视图及其子视图。与 ChangeDetectorRef#detach 结合使用以实现本地变更检测检查。

@returnsvoid

checkNoChanges

void

检查变更检测器及其子检测器，如果检测到任何更改，则抛出错误。

在开发模式下使用，以验证运行变更检测不会引入其他更改。在生产模式下调用它不起作用。

@deprecated

这是一个仅用于测试的 API，在生产接口中没有位置。 当应用程序在开发模式下运行时，checkNoChanges 已是 ApplicationRef tick 的一部分。 有关更精细的 checkNoChanges 验证，请使用 ComponentFixture。

@returnsvoid

reattach

void

将先前分离的视图重新附着到变更检测树。默认情况下，视图会附着到树上。

@returnsvoid
