messages

发出收到的推送通知消息的有效负载。

notificationClicks

Observable<{ action: string; notification: NotificationOptions & { title: string; }; }>

发出收到的推送通知消息的有效负载，以及用户与之互动的操作。如果未使用任何操作，则 action 属性包含一个空字符串 ''。

请注意，notification 属性不包含 Notification 对象，而是包含一个 NotificationOptions 对象，该对象还包括 Notification 对象的 title。

notificationCloses

Observable<{ action: string; notification: NotificationOptions & { title: string; }; }>

发出已关闭通知的载荷，以及与关闭事件关联的操作（如果有）。如果没有使用操作，则 action 属性包含一个空字符串 ''。

请注意，notification 属性不包含 Notification 对象，而是包含一个 NotificationOptions 对象，该对象还包括 Notification 对象的 title。

pushSubscriptionChanges

Observable<{ oldSubscription: PushSubscription | null; newSubscription: PushSubscription | null; }>

发出推送订阅的更新，包括先前（oldSubscription）和当前（newSubscription）的值。根据上下文，任一订阅都可能为 null：

  • 如果不存在先前的订阅，则 oldSubscription 为 null。
  • 如果订阅无效且未被替换，则 newSubscription 为 null。

此流允许客户端对推送订阅的自动更改做出反应，例如由浏览器过期或密钥轮换触发的更改。

subscription

用法说明

您可以将 SwPush 实例作为依赖项注入到任何组件或服务中。

要订阅，请调用 SwPush.requestSubscription()，它会请求用户权限。 该调用返回一个带有新的 PushSubscription 实例的 Promise。

如果用户拒绝权限，或者浏览器阻止或不支持 Push API 或 ServiceWorkers，则请求将被拒绝。 检查 SwPush.isEnabled 以确认状态。

通过推送具有以下有效负载的消息来调用 Push Notifications。

{  "notification": {    "actions": NotificationAction[],    "badge": USVString,    "body": DOMString,    "data": any,    "dir": "auto"|"ltr"|"rtl",    "icon": USVString,    "image": USVString,    "lang": DOMString,    "renotify": boolean,    "requireInteraction": boolean,    "silent": boolean,    "tag": DOMString,    "timestamp": DOMTimeStamp,    "title": DOMString,    "vibrate": number[]  }}

只有 title 是必需的。请参阅 Notification 实例属性。

在订阅有效期间，Service Worker 会监听 PushEvent 事件的发生，并响应地创建 Notification 实例。

使用 SwPush.unsubscribe() 进行取消订阅。
