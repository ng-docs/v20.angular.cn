路由器事件，使你可以跟踪路由器的生命周期。

API

    
      type Event = | NavigationStart  | NavigationEnd  | NavigationCancel  | NavigationError  | RoutesRecognized  | GuardsCheckStart  | GuardsCheckEnd  | RouteConfigLoadStart  | RouteConfigLoadEnd  | ChildActivationStart  | ChildActivationEnd  | ActivationStart  | ActivationEnd  | Scroll  | ResolveStart  | ResolveEnd  | NavigationSkipped

说明

路由器事件，使你可以跟踪路由器的生命周期。

这些事件按以下顺序发生：

  • NavigationStart：导航开始。
  • RouteConfigLoadStart：在路由器惰性加载路由配置之前。
  • RouteConfigLoadEnd：在路由已被惰性加载之后。
  • RoutesRecognized：当路由器解析 URL 并且路由被识别时。
  • GuardsCheckStart：当路由器开始路由的守卫阶段时。
  • ChildActivationStart：当路由器开始激活路由的子路由时。
  • ActivationStart：当路由器开始激活路由时。
  • GuardsCheckEnd：当路由器成功完成路由的守卫阶段时。
  • ResolveStart：当路由器开始路由的 resolve 阶段时。
  • ResolveEnd：当路由器成功完成路由的 resolve 阶段时。
  • ChildActivationEnd：当路由器完成激活路由的子路由时。
  • ActivationEnd：当路由器完成激活路由时。
  • NavigationEnd：当导航成功结束时。
  • NavigationCancel：当导航被取消时。
  • NavigationError：当导航由于意外错误而失败时。
  • Scroll：当用户滚动时。
