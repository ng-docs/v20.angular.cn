说明

拦截并处理 HttpRequest 或 HttpResponse。

大多数拦截器都会先转换传出的请求，然后再通过调用 next.handle(transformedReq) 将其传递给链中的下一个拦截器。拦截器也可以转换响应事件流，方法是在 next.handle() 返回的流上应用额外的 RxJS 操作符。

极少数情况下，拦截器可能会完全处理该请求，并组合一个新的事件流，而不是调用 next.handle()。这是一种可接受的行为，但请记住，后续的拦截器将被完全跳过。

拦截器在单个请求的事件流上返回多个响应的情况也很少见，但也是有效的。

用法说明

要为整个应用使用同一实例的 HttpInterceptors，请仅在您的 AppModule 中导入 HttpClientModule，并将拦截器添加到根应用注入器。 如果您在不同模块（例如，在惰性加载模块中）多次导入 HttpClientModule，则每次导入都会创建 HttpClientModule 的新副本，这将覆盖在根模块中提供的拦截器。
