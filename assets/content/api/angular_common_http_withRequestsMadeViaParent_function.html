withRequestsMadeViaParent

HttpFeature<HttpFeatureKind.RequestsMadeViaParent>

配置当前的 HttpClient 实例，使其通过父注入器的 HttpClient 发起请求，而不是直接发起。

默认情况下，provideHttpClient 会将其注入器中的 HttpClient 配置为一个独立的实例。 例如，即使父注入器中的 HttpClient 配置了一个或多个拦截器，它们也不会拦截通过此实例发起的请求。

启用此选项后，一旦请求通过当前注入器的拦截器，它将被委托给父注入器的 HttpClient 链，而不是直接分派，并且父配置中的拦截器将应用于该请求。

如果注入器层级中有多个 HttpClient 实例，则可能在多个级别使用 withRequestsMadeViaParent，这会导致请求“冒泡”直到达到根级别或一个未配置此选项的 HttpClient。
