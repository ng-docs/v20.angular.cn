描述

Angular 的 <ng-template> 元素定义了一个默认情况下不会渲染的模板。

使用 <ng-template>，你可以定义模板内容，这些内容只有在你直接或间接明确指示 Angular 执行渲染时才会被渲染，从而使你可以完全控制内容的显示方式和时间。

请注意，如果你将内容包裹在 <ng-template> 中，而不指示 Angular 渲染它，则此类内容将不会显示在页面上。例如，请看下面的 HTML 代码，当处理它时，Angular 不会渲染短语 "Hip! Hip! Hooray!" 中的中间的 "Hip!"，因为周围有 <ng-template>。

  <p>Hip!</p>  <ng-template>    <p>Hip!</p>  </ng-template>  <p>Hooray!</p>

用法说明

结构型指令

<ng-template> 的主要用途之一是保存将由结构型指令使用的模板内容。这些指令可以根据它们自己的逻辑添加和删除模板内容的副本。

当使用结构型指令简写形式时，Angular 会在幕后创建一个 <ng-template> 元素。

TemplateRef

<ng-template> 元素表示为 TemplateRef 类的实例。

要将模板的副本添加到 DOM，请将此对象传递给 ViewContainerRef 的 createEmbeddedView() 方法。

模板变量

<ng-template> 元素可以在模板中被引用，可以使用标准模板变量。

这就是 <ng-template> 元素如何用作 ngIf 的 else 子句。

这样的模板变量可以与 ngTemplateOutlet 指令结合使用，以渲染 <ng-template> 标签内定义的内容。

查询

Query（例如 ViewChild）可以找到与 <ng-template> 元素关联的 TemplateRef，以便可以以编程方式使用它；例如，将其传递给 ViewContainerRef 的 createEmbeddedView() 方法。

上下文

在 <ng-template> 标签内，你可以引用周围外部模板中存在的变量。 此外，上下文对象可以与 <ng-template> 元素关联。 这样的对象包含可以通过模板（let 和 as）声明从模板内容中访问的变量。
