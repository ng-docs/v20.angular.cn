过渡

AnimationTransitionMetadata

声明一个动画过渡，当满足某个特定条件时播放。

用法说明

状态更改表达式

状态更改表达式指示 Angular 何时运行过渡动画，它可以是

  • 具有特定语法的字符串
  • 或者是一个函数，用于比较先前的状态和当前状态（绑定到元素触发器的表达式的值），如果应该发生过渡，则返回 true，否则返回 false

字符串格式可以是：

  • fromState => toState，这表示当绑定到触发器元素的表达式从 fromState 变为 toState 时，应发生过渡动画

    范例：

      transition('open => closed', animate('.5s ease-out', style({ height: 0 }) ))

  • fromState <=> toState，这表示当绑定到触发器元素的表达式从 fromState 变为 toState 或反之亦然时，应发生过渡动画

    范例：

      transition('enabled <=> disabled', animate('1s cubic-bezier(0.8,0.3,0,1)'))

  • :enter/:leave，这表示当元素进入或离开 DOM 时，应发生过渡动画

    范例：

      transition(':enter', [    style({ opacity: 0 }),    animate('500ms', style({ opacity: 1 }))  ])

  • :increment/:decrement，这表示当绑定到触发器元素的数值表达式的值增加或减少时，应发生过渡动画

    范例：

      transition(':increment', query('@counter', animateChild()))

  • 以上任何一项的序列（以逗号分隔），这表示每当状态更改表达式之一匹配时，都应发生过渡动画

    范例：

      transition(':increment, * => enabled, :enter', animate('1s ease', keyframes([    style({ transform: 'scale(1)', offset: 0}),    style({ transform: 'scale(1.1)', offset: 0.7}),    style({ transform: 'scale(1)', offset: 1})  ]))),

另请注意，在这种上下文中：

  • void 可用于指示元素不存在
  • 星号可以用作匹配任何状态的通配符
  • （作为上述结果，void => * 等价于 :enter，而 * => void 则等价于 :leave）
  • true 和 false 也分别匹配表达式值 1 和 0（但不匹配真值和假值）

请注意进入和离开的元素，因为它们的过渡效果会给开发人员带来常见的陷阱。

请注意，当带有触发器的元素进入 DOM 时，其 :enter 过渡效果始终会执行，但是如果该元素与其父元素一起被移除，则其 :leave 过渡效果将不会执行（因为它将在其过渡效果有机会执行之前“无警告”地被移除，这种过渡效果发生的唯一方式是元素自行退出 DOM）。

最终状态动画

如果过渡的最后一步是对 animate() 的调用，该调用使用了没有 style 数据的计时值，那么该步骤将被自动视为元素达到最终状态的最终动画弧，在这种情况下，Angular 会自动添加或移除 CSS 样式以确保元素处于正确的最终状态。

用例

  • 基于触发器表达式值应用的过渡动画
<div [@myAnimationTrigger]="myStatusExp"> ...</div>
trigger("myAnimationTrigger", [  ..., // states  transition("on => off, open => closed", animate(500)),  transition("* <=> error", query('.indicator', animateChild()))])
  • 基于自定义逻辑（取决于触发器的表达式值和提供的参数）应用的过渡动画
    <div [@myAnimationTrigger]="{ value: stepName, params: { target: currentTarget }}"> ...</div>
    trigger("myAnimationTrigger", [  ..., // states  transition(    (fromState, toState, _element, params) =>      ['firststep', 'laststep'].includes(fromState.toLowerCase())      && toState === params?.['target'],    animate('1s')  )])
