API

    
      type HttpInterceptorFn = (  req: HttpRequest<unknown>,  next: HttpHandlerFn,) => Observable<HttpEvent<unknown>>

描述

用于通过 HttpClient 发起的 HTTP 请求的拦截器。

HttpInterceptorFn 是中间件函数，当发出请求时，HttpClient 会调用它们。 这些函数有机会修改传出的请求或返回的任何响应，以及阻止、重定向或以其他方式更改请求或响应的语义。

提供了一个代表下一个拦截器（或将发出实际 HTTP 请求的后端）的 HttpHandlerFn。大多数拦截器会委托给此函数，但这并非必需（有关更多详细信息，请参阅 HttpHandlerFn）。

HttpInterceptorFn 在 注入上下文 中执行。 它们可以通过配置它们的 EnvironmentInjector 访问 inject()。

用法说明

这是一个 noop 拦截器，它在不修改请求的情况下传递请求：

export const noopInterceptor: HttpInterceptorFn = (req: HttpRequest<unknown>, next:HttpHandlerFn) => {  return next(modifiedReq);};

如果你想更改请求，请先克隆它，然后在将其传递给 next() 处理函数之前修改克隆。

这是一个基本拦截器，它将 bearer 令牌添加到标头

export const authenticationInterceptor: HttpInterceptorFn = (req: HttpRequest<unknown>, next:HttpHandlerFn) => {   const userToken = 'MY_TOKEN'; const modifiedReq = req.clone({     headers: req.headers.set('Authorization', `Bearer ${userToken}`),   });   return next(modifiedReq);};
