迁移到 inject 函数

edit

与基于构造函数的注入相比，Angular 的 inject 函数提供了更准确的类型以及与标准装饰器更好的兼容性。

这个原理图转换你的类中基于构造函数的注入，改为使用 inject 函数。

使用以下命令运行原理图：

ng generate @angular/core:inject

之前 

import { Component, Inject, Optional } from '@angular/core';import { MyService } from './service';import { DI_TOKEN } from './token';@Component()export class MyComp {  constructor(    private service: MyService,    @Inject(DI_TOKEN) @Optional() readonly token: string) {}}

之后 

import { Component, inject } from '@angular/core';import { MyService } from './service';import { DI_TOKEN } from './token';@Component()export class MyComp {  private service = inject(MyService);  readonly token = inject(DI_TOKEN, { optional: true });}

迁移选项

此迁移包含多个选项来自定义其输出。

path

确定应该迁移项目中的哪个子路径。传入 . 或留空以迁移整个目录。

migrateAbstractClasses

Angular 不验证抽象类的参数是否可注入。这意味着迁移无法可靠地将它们迁移到 inject 而不会有破坏的风险，这就是为什么默认情况下禁用它们的原因。 如果你想迁移抽象类，请启用此选项，但请注意你可能必须 手动修复一些破坏。

backwardsCompatibleConstructors

默认情况下，迁移会尝试尽可能清理代码，包括从构造函数中删除参数，甚至在构造函数不包含任何代码时删除整个构造函数。 在某些情况下，当带有 Angular 装饰器的类从带有 Angular 装饰器的其他类继承时，这可能会导致编译错误。 如果启用此选项，迁移将生成一个额外的构造函数签名以保持向后兼容，但会增加更多代码。

之前 

import { Component } from '@angular/core';import { MyService } from './service';@Component()export class MyComp {  constructor(private service: MyService) {}}

之后 

import { Component } from '@angular/core';import { MyService } from './service';@Component()export class MyComp {  private service = inject(MyService);  /** Inserted by Angular inject() migration for backwards compatibility */  constructor(...args: unknown[]);  constructor() {}}

nonNullableOptional

如果带有 @Optional 装饰器的参数注入失败，Angular 会返回 null，这意味着任何 @Optional 参数的实际类型将是 | null。但是，由于装饰器无法影响它们的类型，因此存在大量现有代码的类型不正确。类型在 inject() 中固定，这可能导致出现新的编译错误。如果您启用此选项，迁移将在 inject() 调用后生成一个非空断言以匹配旧类型，但可能会隐藏类型错误。

注意： 非空断言不会添加到已类型化为可为空的参数中，因为依赖于它们的代码可能已经考虑了它们的可空性。

之前 

import { Component, Inject, Optional } from '@angular/core';import { TOKEN_ONE, TOKEN_TWO } from './token';@Component()export class MyComp {  constructor(    @Inject(TOKEN_ONE) @Optional() private tokenOne: number,    @Inject(TOKEN_TWO) @Optional() private tokenTwo: string | null) {}}

之后 

import { Component, inject } from '@angular/core';import { TOKEN_ONE, TOKEN_TWO } from './token';@Component()export class MyComp {  // Note the `!` at the end.  private tokenOne = inject(TOKEN_ONE, { optional: true })!;  // Does not have `!` at the end, because the type was already nullable.  private tokenTwo = inject(TOKEN_TWO, { optional: true });}
