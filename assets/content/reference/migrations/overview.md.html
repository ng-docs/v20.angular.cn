迁移

edit

了解如何将现有的 Angular 项目逐步迁移到最新特性。

独立

独立组件提供了一种构建 Angular 应用的简化方式。独立组件直接指定它们的依赖，而不是通过 NgModule 获取。

立即迁移

控制流语法

内置的控制流语法允许你使用更符合工效学的语法，这种语法更接近 JavaScript，并且具有更好的类型检查。它取代了导入 CommonModule 以使用 *ngFor、*ngIf 和 *ngSwitch 等功能的必要性。

立即迁移

inject() 函数

与基于构造函数的注入相比，Angular 的 inject 函数提供了更准确的类型以及与标准装饰器更好的兼容性。

立即迁移

惰性加载路由

将急性加载的组件路由转换为惰性加载的路由。这允许构建过程将生产捆绑包拆分成更小的块，从而在初始页面加载时加载更少的 JavaScript。

立即迁移

新的 `input()` API

将现有的 @Input 字段转换为现在已可用于生产环境的新的信号输入 API。

立即迁移

新的 `output()` 函数

将现有的 @Output 自定义事件转换为现在已可用于生产环境的新的 output 函数。

立即迁移

作为信号的查询

将现有的装饰器查询字段转换为改进后的信号查询 API。该 API 现在已可用于生产环境。

立即迁移

清理未使用的导入

清理项目中未使用的导入。

立即尝试

自闭合标签

尽可能将组件模板转换为使用自闭合标签。

立即迁移
