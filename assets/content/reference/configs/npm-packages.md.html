工作区 npm 依赖

edit

Angular 框架、Angular CLI 以及 Angular 应用所使用的组件都打包为 npm 包，并通过 npm 注册表进行分发。

你可以使用 npm CLI 客户端 下载和安装这些 npm 软件包。 默认情况下，Angular CLI 使用 npm 客户端。

有帮助：有关所需的 Node.js 和 npm 版本及安装信息，请参阅 本地环境设置。

如果你的机器上已经有正在运行的项目使用了其他版本的 Node.js 和 npm，请考虑使用 nvm 来管理多个版本的 Node.js 和 npm。

package.json

npm 安装在 package.json 文件中标识的包。

当 CLI 命令 ng new 创建新的工作区时，会创建一个 package.json 文件。 工作区中的所有项目（包括 CLI 在创建工作区时创建的初始应用项目）都使用此 package.json。 使用 ng generate library 创建的库将包含它们自己的 package.json 文件。

最初，此 package.json 包含一组入门软件包，其中一些是 Angular 所必需的，另一些则支持常见的应用场景。 随着应用的演进，你可以向 package.json 中添加软件包。

默认依赖

以下 Angular 软件包作为依赖项包含在新的 Angular 工作区的默认 package.json 文件中。 有关 Angular 软件包的完整列表，请参阅 API 参考。

软件包名称 详情
@angular/animations Angular 的旧版动画库可以轻松定义和应用页面和列表过渡等动画效果。有关更多信息，请参阅 旧版动画指南。
@angular/common Angular 团队提供的常用服务、管道和指令。
@angular/compiler Angular 的模板编译器。它理解 Angular 模板，并可以将它们转换为使应用运行的代码。
@angular/compiler-cli Angular 的编译器，由 Angular CLI 的 ng build 和 ng serve 命令调用。它在标准的 TypeScript 编译中，使用 @angular/compiler 处理 Angular 模板。
@angular/core 框架的关键运行时部分，是每个应用程序都需要的部分。包括所有元数据装饰器，如 @Component、依赖注入和组件生命周期钩子。
@angular/forms 同时支持模板驱动表单和响应式表单。请参阅表单简介。
@angular/platform-browser 所有与 DOM 和浏览器相关的内容，尤其是那些有助于渲染到 DOM 中的部分。
@angular/platform-browser-dynamic 包括 提供者 以及使用 即时编译器 在客户端编译和运行应用的方法。
@angular/router 当浏览器 URL 更改时，路由器模块会在你的应用页面之间导航。有关更多信息，请参阅路由与导航。
@angular/cli 包含用于运行 ng 命令的 Angular CLI 二进制文件。
@angular-devkit/build-angular 包含用于捆绑、测试和为 Angular 应用和库启动服务器的默认 CLI 构建器。
rxjs 一个使用 Observables 进行响应式编程的库。
zone.js 当原生 JavaScript 操作引发事件时，Angular 依赖 zone.js 运行 Angular 的变更检测过程。
typescript TypeScript 编译器、语言服务器和内置类型定义。
