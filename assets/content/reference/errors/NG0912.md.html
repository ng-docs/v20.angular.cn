组件 ID 生成冲突

edit

当创建组件时，Angular 会为每个组件生成唯一的组件 ID。这是使用 Angular 组件元数据生成的，包括但不限于：选择器、宿主绑定的数量、类属性名、视图和内容查询。当两个组件的元数据相同时（通常共享相同的选择器），就会发生 ID 生成冲突。

组件 ID 在 Angular 内部使用于：

  • 用于 DOM 节点的额外注解，以实现样式封装
  • 在水合期间，用于在服务器端渲染后恢复应用状态。

为了避免组件 ID 冲突可能导致的问题，建议按照下述方法解决它们。

组件 ID 冲突的范例 

@Component({  selector: 'my-component',  template: 'complex-template',})class SomeComponent {}@Component({  selector: 'my-component',  template: 'empty-template',})class SomeMockedComponent {}

定义这两个组件将触发 ID 生成冲突，并且在开发期间会显示警告。

调试错误

警告消息包含 ID 冲突的两个组件的类名。

可以使用以下解决方案之一来解决此问题：

  1. 更改两个组件之一的选择器。例如，使用像 :not() 这样没有效果的伪选择器和一个不同的标签名。
@Component({  selector: 'my-component:not(p)',  template: 'empty-template',})class SomeMockedComponent {}
  1. 向其中一个组件添加额外的宿主属性（Attribute）。
@Component({  selector: 'my-component',  template: 'complex-template',  host: {'some-binding': 'some-value'},})class SomeComponent {}
