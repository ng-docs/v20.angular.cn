已引导的 NgModule 没有指定要初始化的组件

edit

此错误意味着用于引导应用的 NgModule 缺少关键信息，Angular 无法继续执行引导过程。

当 @NgModule 注解中缺少 NgModule 的 bootstrap 属性（或为空数组），并且该 NgModule 类上未定义 ngDoBootstrap 生命周期钩子时，就会发生此错误。

有关引导过程的更多信息，请参见本指南。

以下范例会触发此错误。

@NgModule({  declarations: [AppComponent],  imports: [BrowserModule, AppRoutingModule],  providers: [],})export class AppModule {}// The `AppModule` is used for bootstrapping, but the `@NgModule.bootstrap` field is missing.platformBrowser().bootstrapModule(AppModule);
@NgModule({  declarations: [AppComponent],  imports: [BrowserModule, AppRoutingModule],  providers: [],  bootstrap: [],})export class AppModule {}// The `AppModule` is used for bootstrapping, but the `@NgModule.bootstrap` field contains an empty array.platformBrowser().bootstrapModule(AppModule);

调试此错误

请确保用于引导的 NgModule 已正确建立：

  • 要么 @NgModule 注解中存在 bootstrap 属性（且包含非空数组）
  • 或者 ngDoBootstrap 方法存在于 NgModule 类上
