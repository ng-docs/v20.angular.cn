缺少结构型指令

edit

此诊断可确保独立组件在其模板中使用自定义结构型指令（例如 *select 或 *featureFlag）时，已导入这些指令所需的依赖项。

import {Component} from '@angular/core';@Component({  // Template uses `*select`, but no corresponding directive imported.  imports: [],  template: `<p *select="let data from source">{{data}}</p>`,})class MyComponent {}

这样做有什么问题？

未使用导入就使用结构型指令会在运行时失败，因为 Angular 会尝试绑定到 HTML 元素的 select 属性，而该属性并不存在。

我应该怎么做？

请确保将相应的结构型指令导入到组件中：

import {Component} from '@angular/core';import {SelectDirective} from 'my-directives';@Component({  // Add `SelectDirective` to the `imports` array to make it available in the template.  imports: [SelectDirective],  template: `<p *select="let data from source">{{data}}</p>`,})class MyComponent {}

配置要求

必须启用 strictTemplates 才能发出任何扩展诊断。 missingStructuralDirective 除了 strictTemplates 之外没有其他要求。

如果无法避免该怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "missingStructuralDirective": "suppress"      }    }  }}

有关更多信息，请参阅 扩展诊断配置。
