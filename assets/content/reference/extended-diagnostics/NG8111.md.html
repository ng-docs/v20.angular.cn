应该在事件绑定中调用函数。

edit

此诊断会检测事件绑定中未调用的函数。

import {Component, signal, Signal} from '@angular/core';@Component({  template: `<button (click)="onClick">Click me</button>`,})class MyComponent {  onClick() {    console.log('clicked');  }}

那有什么问题？

当事件被触发时，应该调用事件绑定中的函数。 如果函数未被调用，则在事件触发时它将不会执行。

我应该怎么做？

确保在事件绑定中使用函数时调用该函数，以便在事件触发时执行该函数。

import {Component} from '@angular/core';@Component({  template: `<button (click)="onClick()">Click me</button>`,})class MyComponent {  onClick() {    console.log('clicked');  }}

配置要求

必须启用strictTemplates 才能发出任何扩展诊断。 除了 strictTemplates 之外，uninvokedFunctionInEventBinding 没有额外的要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "uninvokedFunctionInEventBinding": "suppress"      }    }  }}

有关更多信息，请参阅扩展诊断配置。
