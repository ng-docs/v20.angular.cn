未加括号的空值合并

edit

此诊断可检测空值合并运算符 (??) 与逻辑或 (||) 或逻辑与 (&&) 运算符混合使用而未加括号以消除歧义的情况。

import {Component, signal, Signal} from '@angular/core';@Component({  template: `    <button [disabled]="hasPermission() && task()?.disabled ?? true">      Run    </button>  `,})class MyComponent {  hasPermission = input(false);  task = input<Task|undefined>(undefined);}

这有什么问题？

在没有消除歧义的括号的情况下，很难理解是先计算 ?? 还是先计算 ||/&&。这在 TypeScript 和 JavaScript 中被视为错误，但在 Angular 模板中历来是被允许的。

我应该怎么做？

在这些情况下，请始终使用括号来消除歧义。如果您不确定代码的原始意图，但想保持行为不变，请将括号放在 ?? 运算符周围。

import {Component, signal, Signal} from '@angular/core';@Component({  template: `    <button [disabled]="hasPermission() && (task()?.disabled ?? true)">      Run    </button>  `,})class MyComponent {  hasPermission = input(false);  task = input<Task|undefined>(undefined);}
