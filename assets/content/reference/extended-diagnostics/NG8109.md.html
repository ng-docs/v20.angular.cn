Signals 必须在模板插值中调用。

edit

此诊断检测模板插值中未调用的 signals。

import {Component, signal, Signal} from '@angular/core';@Component({  template: `<div>{{ mySignal }}/div>`,})class MyComponent {    mySignal: Signal<number> = signal(0);}

那样有什么问题？

Angular Signals 是零参数函数 (() => T)。当执行时，它们会返回 signal 的当前值。 这意味着当在模板插值中使用它们来渲染它们的值时，它们旨在被调用。

我应该怎么做？

当你在模板插值中使用 signal 来渲染它的值时，请确保调用该 signal。

import {Component, signal, Signal} from '@angular/core';@Component({  template: `<div>{{ mySignal() }}/div>`,})class MyComponent {  mySignal: Signal<number> = signal(0)}

配置要求

必须启用 strictTemplates 才能发出任何扩展诊断。 interpolatedSignalNotInvoked 除了 strictTemplates 之外没有额外的要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "interpolatedSignalNotInvoked": "suppress"      }    }  }}

请参阅 扩展诊断配置 以获取更多信息。
