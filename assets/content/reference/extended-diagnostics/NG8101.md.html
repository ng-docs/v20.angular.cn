无效的 Banana-in-Box

edit

此诊断检测到用于双向绑定的反向 “banana-in-box” 语法。

<user-viewer ([user])="loggedInUser" />

那有什么问题？

就目前而言，([var]) 实际上是一个名为 [var] 的事件绑定。作者可能打算将其作为名为 var 的变量的双向绑定，但就目前而言，此绑定不起作用。

我应该怎么做？

修复笔误。 顾名思义，香蕉 ( 应该放在方框 [] 里面。 在这种情况下：

<user-viewer [(user)]="loggedInUser" />

配置要求

必须启用strictTemplates才能发出任何扩展诊断。 invalidBananaInBox 除了 strictTemplates 之外没有其他要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "invalidBananaInBox": "suppress"      }    }  }}

有关更多信息，请参阅扩展诊断配置。
