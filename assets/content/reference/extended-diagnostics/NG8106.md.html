后缀不支持

edit

此诊断会在属性（Attribute）绑定中使用了 .px、.% 和 .em 后缀时进行检测。

<img [attr.width.px]="5">

那有什么问题？

这些后缀仅在样式绑定中可用。当绑定到属性（Attribute）时，它们没有任何意义。

我应该怎么做？

与其使用仅在样式绑定中受支持的 .px、.% 或 .em 后缀，不如将其移至绑定的值赋值中。

<img [attr.width]="'5px'">

配置要求

strictTemplates 必须启用才能发出任何扩展诊断。suffixNotSupported 除了 strictTemplates 之外没有其他要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "suffixNotSupported": "suppress"      }    }  }}

有关更多信息，请参阅扩展诊断配置。
