文本属性未绑定

edit

此诊断可确保具有“特殊”Angular绑定前缀（attr.、style.和class.）的属性被解释为绑定。

<div attr.id="my-id"></div>

那样有什么问题？

在此范例中，attr.id 被解释为常规 Attribute，并将按原样出现在最终 HTML 中 (<div attr.id="my-id"></div>)。这可能不是开发者的本意。 相反，其本意可能是要设置 id Attribute (<div id="my-id"></div>)。

我应该怎么做？

绑定到attr.、class.或style.时，请确保使用Angular模板绑定语法（[]）。

<div [attr.id]="my-id"></div><div [style.color]="red"></div><div [class.large]="true"></div>

配置要求

必须启用 strictTemplates 才能发出任何扩展诊断。 除了 strictTemplates 之外，textAttributeNotBinding 没有其他要求。

如果我无法避免这种情况怎么办？

可以通过编辑项目的 tsconfig.json 文件来禁用此诊断：

{  "angularCompilerOptions": {    "extendedDiagnostics": {      "checks": {        "textAttributeNotBinding": "suppress"      }    }  }}

有关更多信息，请参阅扩展诊断配置。
