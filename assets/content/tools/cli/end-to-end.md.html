端到端测试

edit

端到端 (E2E) 测试是一种用于断言你的整个应用从开始到结束或“端到端”都能按预期工作的测试形式。E2E 测试与单元测试的不同之处在于，它与代码的底层实现细节完全解耦。它通常用于以模拟用户与应用交互的方式来验证应用。本页是关于如何使用 Angular CLI 开始在 Angular 中进行端到端测试的指南。

建立 E2E 测试

Angular CLI 下载并安装运行 Angular 应用的端到端测试所需的一切。

ng e2e

ng e2e 命令将首先检查你的项目以查找 "e2e" 目标。如果找不到，CLI 将提示你想要使用哪个 e2e 包，并引导你完成建立过程。

Cannot find "e2e" target for the specified project.You can add a package that implements these capabilities.For example:Cypress: ng add @cypress/schematicNightwatch: ng add @nightwatch/schematicsWebdriverIO: ng add @wdio/schematicsPlaywright: ng add playwright-ng-schematicsPuppeteer: ng add @puppeteer/ng-schematicsWould you like to add a package with "e2e" capabilities now?No❯ CypressNightwatchWebdriverIOPlaywrightPuppeteer

如果你在上面的列表中没有找到想要使用的测试运行器，你可以使用 ng add 手动添加一个包。

运行 E2E 测试

既然你的应用已经配置了端到端测试，我们现在可以运行相同的命令来执行你的测试。

ng e2e

请注意，使用任何集成的 e2e 包运行测试都没有什么“特别”之处。ng e2e 命令实际上只是在底层运行 e2e 构建器。你始终可以创建你自己的自定义构建器，命名为 e2e，并使用 ng e2e 运行它。
