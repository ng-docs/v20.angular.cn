建立本地环境和工作区

edit

本指南介绍了如何使用 Angular CLI 为 Angular 开发设置环境。 它包括有关安装 CLI、创建初始工作区和入门应用以及本地运行该应用以验证设置的信息。

无需本地设置即可试用 Angular

如果你是 Angular 新手，不妨从 立即试用！ 开始，它会在你的浏览器中介绍 Angular 的要点。 这个独立的教程利用了用于在线开发的交互式 StackBlitz 环境。 在你准备好之前，你无需建立本地环境。

开始之前

要使用 Angular CLI，你应该熟悉以下内容：

JavaScriptopen_in_new HTMLopen_in_new CSSopen_in_new

你应该熟悉命令行界面 (CLI) 工具的用法，并对命令 Shell 有一个大致的了解。 了解 TypeScript 会有所帮助，但不是必需的。

依赖

要在你的本地系统上安装 Angular CLI，你需要安装 Node.js。 Angular CLI 使用 Node 及其相关的包管理器 npm，在浏览器之外安装和运行 JavaScript 工具。

下载并安装 Node.js，其中也将包含 npm CLI。 Angular 需要 active LTS 或 maintenance LTS 版本的 Node.js。 有关更多信息，请参阅 Angular 的版本兼容性 指南。

安装 Angular CLI

要安装 Angular CLI，请打开终端窗口并运行以下命令：

npm

npm install -g @angular/cli

pnpm

pnpm install -g @angular/cli

yarn

yarn global add @angular/cli

bun

bun install -g @angular/cli

Powershell 执行策略

在 Windows 客户端计算机上，默认情况下禁用 PowerShell 脚本的执行，因此上述命令可能会失败并出现错误。 要允许执行 PowerShell 脚本（npm 全局二进制文件需要这样做），你必须设置以下执行策略：

Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

仔细阅读执行命令后显示的消息，并按照说明进行操作。确保你了解设置执行策略的含义。

Unix 权限

在某些类 Unix 的设置中，全局脚本可能归 root 用户所有，因此上述命令可能会失败并出现权限错误。 使用 sudo 以 root 用户身份执行命令，并在出现提示时输入你的密码：

npm

sudo npm install -g @angular/cli

pnpm

sudo pnpm install -g @angular/cli

yarn

sudo yarn global add @angular/cli

bun

sudo bun install -g @angular/cli

确保你了解以 root 身份运行命令的含义。

创建一个工作区和初始应用

你在 Angular 工作区的上下文中开发应用。

要创建一个新的工作区和初始入门应用，请运行 CLI 命令 ng new 并提供名称 my-app，如此处所示，然后回答有关要包含的功能的提示：

ng new my-app

Angular CLI 安装必要的 Angular npm 包和其他依赖项。 这可能需要几分钟。

CLI 在一个与工作区同名的新目录中创建一个新的工作区和一个小的欢迎应用，随时可以运行。 导航到新目录，以便后续命令使用此工作区。

cd my-app

运行应用

Angular CLI 包含一个开发服务器，供你在本地构建和启动你的应用。运行以下命令：

ng serve --open

ng serve 命令启动服务器，监视你的文件，并在你更改这些文件时重建应用并重新加载浏览器。

--open（或仅 -o）选项会自动打开你的浏览器并访问 http://localhost:4200/ 以查看生成的应用。

工作区和项目文件

ng new 命令创建一个 Angular 工作区 文件夹，并在其中生成一个新的应用。 一个工作区可以包含多个应用和库。 ng new 命令创建的初始应用位于工作区的根目录下。 当你在现有工作区中生成额外的应用或库时，默认情况下它会进入 projects/ 子文件夹。

新生成的应用包含根组件和模板的源文件。 每个应用都有一个 src 文件夹，其中包含其组件、数据和资产。

你可以直接编辑生成的文件，或者使用 CLI 命令添加和修改它们。 使用 ng generate 命令为其他组件、指令、管道、服务等添加新文件。 诸如 ng add 和 ng generate 之类的命令（用于创建或操作应用和库）必须从工作区内执行。 相比之下，诸如 ng new 之类的命令必须在工作区外部执行，因为它们将创建一个新的工作区。

后续步骤

  • 了解有关生成的工作区的 文件结构 和 配置 的更多信息。

  • 使用 ng test 测试你的新应用。

  • 使用 ng generate 生成样板代码，例如组件、指令和管道。

  • 使用 ng deploy 部署你的新应用，使其对真实用户可用。

  • 使用 ng e2e 建立并运行你的应用的端到端测试。
