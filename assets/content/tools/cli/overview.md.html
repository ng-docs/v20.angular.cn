Angular CLI

edit

Angular CLI 是一种命令行界面工具，使你可以直接从命令 Shell 搭建、开发、测试、部署和维护 Angular 应用。

Angular CLI 在 npm 上以 @angular/cli 包的形式发布，并包含一个名为 ng 的二进制文件。调用 ng 命令时，就会使用 Angular CLI。

无需本地安装即可试用 Angular

如果你是 Angular 新手，不妨从立即试用！开始，它在现成的基本在线商店应用的上下文中介绍了 Angular 的要点，供你检查和修改。 这个独立的教程利用了用于在线开发的交互式 StackBlitz 环境。 在准备就绪之前，你无需建立本地环境。

CLI 命令行语言语法

Angular CLI 在选项语法方面大致遵循 Unix/POSIX 约定。

布尔选项

布尔选项有两种形式：--this-option 将标志设置为 true，--no-this-option 将其设置为 false。 你也可以使用 --this-option=false 或 --this-option=true。 如果未提供任何选项，则该标志将保持其默认状态，如参考文档中所列。

数组选项

数组选项可以通过两种形式提供：--option value1 value2 或 --option value1 --option value2。

键/值选项

某些选项（如 --define）需要 key=value 对的数组作为其值。 与数组选项一样，键/值选项可以通过两种形式提供： --define 'KEY_1="value1"' KEY_2=true 或 --define 'KEY_1="value1"' --define KEY_2=true。

相对路径

指定文件的选项可以作为绝对路径给出，也可以作为相对于当前工作目录的路径给出，当前工作目录通常是工作区或项目根目录。
