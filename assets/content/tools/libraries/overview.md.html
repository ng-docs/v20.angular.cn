Angular 库概览

edit

许多应用都需要解决相同的通用问题，例如呈现统一的用户界面、呈现数据以及允许数据录入。 开发人员可以为特定领域创建通用解决方案，这些方案可以调整后在不同的应用中复用。 这样的解决方案可以构建为 Angular 库，并且这些库可以作为 npm 包 发布和共享。

Angular 库是一个 Angular 项目，它与应用的不同之处在于它不能独立运行。 库必须在应用中导入和使用。

库扩展了 Angular 的基础功能。 例如，要向应用添加响应式表单，请使用 ng add @angular/forms 添加库包，然后在你的应用代码中从 @angular/forms 库导入 ReactiveFormsModule。 同样，将服务工作线程库添加到 Angular 应用是将其转变为渐进式 Web 应用 (PWA) 的步骤之一。 Angular Material 是一个大型通用库的示例，它提供了复杂的、可复用的、可适应的 UI 组件。

任何应用开发者都可以使用这些库以及 Angular 团队或第三方发布的其他库。 请参阅 使用已发布的库。

提示： 库旨在供 Angular 应用使用。要将 Angular 特性添加到非 Angular Web 应用，请使用 Angular 自定义元素。

创建库

如果你开发了适合复用的特性，则可以创建自己的库。 这些库可以在你的工作区本地使用，或者你可以将它们作为 npm 包 发布，以便与其他项目或其他 Angular 开发者共享。 这些包可以发布到 npm 仓库、私有 npm 企业仓库或支持 npm 包的私有包管理系统。 请参阅 创建库。

决定将特性打包为库是一项架构决策。这类似于决定特性是组件还是服务，或者决定组件的范围。

将特性打包为库会强制库中的工件与应用的业务逻辑解耦。 这可以帮助避免各种不良实践或架构错误，这些错误可能会使将来难以解耦和复用代码。

将代码放入单独的库中比仅仅将所有内容放在一个应用中更复杂。 它需要在管理、维护和更新库方面投入更多的时间和思考。 当库在多个应用中使用时，这种复杂性会得到回报。
