DevTools 概览

edit

Angular DevTools 是一个浏览器扩展程序，可为 Angular 应用提供调试和分析功能。

从 Chrome Web Store 或 Firefox Addons 安装 Angular DevTools。

你可以通过按 F12 或 Ctrl+Shift+I (Windows 或 Linux) 和 Fn+F12 或 Cmd+Option+I (Mac) 在任何网页上打开 Chrome 或 Firefox DevTools。 打开浏览器 DevTools 并安装 Angular DevTools 后，你可以在“Angular”选项卡下找到它。

有帮助： Chrome 的新标签页不运行已安装的扩展程序，因此 Angular 选项卡不会出现在 DevTools 中。请访问任何其他页面来查看它。

打开你的应用

打开扩展程序时，你会看到两个额外的选项卡：

选项卡 详细信息
组件 允许你探索应用中的组件和指令，并预览或编辑它们的状态。
分析器 允许你分析应用性能，并了解变更检测执行期间的性能瓶颈。

在 Angular DevTools 的右上角，你可以找到页面上运行的 Angular 版本以及扩展程序的最新提交哈希值。

未检测到 Angular 应用

如果在打开 Angular DevTools 时看到错误消息“未检测到 Angular 应用”，则表示它无法与页面上的 Angular 应用进行通信。 最常见的原因是你正在检查的网页不包含 Angular 应用。 请仔细检查你是否正在检查正确的网页以及 Angular 应用是否正在运行。

我们检测到一个使用生产配置构建的应用

如果你看到错误消息“我们检测到一个使用生产配置构建的应用。Angular DevTools 仅支持开发构建。”，则表示在页面上找到了一个 Angular 应用，但它是使用生产优化编译的。 在为生产编译时，Angular CLI 会移除各种调试功能，以最大限度地减少页面上的 JavaScript 量以提高性能。这包括与 DevTools 通信所需的功能。

要运行 DevTools，你需要禁用优化来编译你的应用。ng serve 默认会这样做。 如果你需要调试已部署的应用，请使用 optimization 配置选项 ({"optimization": false}) 在你的构建中禁用优化。
