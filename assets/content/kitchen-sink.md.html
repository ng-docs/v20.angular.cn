警报

警报只是特殊的段落。它们有助于引起注意（不要与呼叫混淆），以引起对某些更紧急事项的关注。它们从上下文中获得字体大小，并提供多种级别。尽量不要使用警报来渲染过多内容，而是用来增强和吸引对周围内容的关注。

在 Markdown 中，使用格式 SEVERITY_LEVEL + : + ALERT_TEXT 来样式化新行上的警报。

注意：使用 Note 来提供非必需于正文的辅助/附加信息。

提示：使用 Tip 来呼吁用户可以执行的特定任务/操作，或直接影响任务/操作的事实。

待办：使用 TODO 来表示您计划很快扩展的不完整文档。您也可以分配 TODO，例如 TODO(emmatwersky): Text。

问题：使用 Question 向读者提出问题，就像一个他们应该能够回答的小测验一样。

摘要：使用 Summary 提供页面或部分内容的兩到三句话的概要，以便读者可以确定这是否是他们需要的地方。

简而言之：如果您能用一两句话提供有关页面或部分的基本信息，请使用 TL;DR（或 TLDR）。例如，TLDR：大黄猫是一只猫。

关键：使用 Critical 来指出潜在的坏事或提醒读者在做某事之前应该小心。例如，警告：运行带 -f 选项的 rm 将在不提示您的情况下删除受写保护的文件或目录。

重要：使用 Important 来获取对理解文本或完成某些任务至关重要的信息。

有帮助：使用最佳实践来呼吁已知成功或优于替代方案的做法。

注意：请注意 开发者！警报可以包含一个 链接 和其他嵌套样式（但请尽量谨慎使用）！.

  1. 安装 Angular CLI

    您可以使用 Angular CLI 来创建项目、生成应用程序和库代码，以及执行各种持续的开发任务，例如测试、打包和部署。

    要安装 Angular CLI，请打开一个终端窗口并运行以下命令：

    npm install -g @angular/cli

  2. 创建工作区和初始应用程序

    您在 Angular 工作区的上下文中开发应用程序。

    要创建新的工作区和初始入门应用程序：

    • 运行 CLI 命令 ng new 并提供名称 my-app，如下所示：

      ng new my-app

    • ng new 命令会提示您输入有关要在初始应用中包含的功能的信息。按 Enter 或 Return 键接受默认设置。

      Angular CLI 会安装必要的 Angular npm 包和其他依赖项。这可能需要几分钟时间。

      CLI 会创建一个新的工作区和一个简单的欢迎应用程序，即可运行。

  3. 运行应用程序

    Angular CLI 包含一个服务器，供您在本地构建和提供应用程序。

    1. 导航到工作区文件夹，例如 my-app。
    2. 运行以下命令：
      cd my-app ng serve --open

    ng serve 命令会启动服务器，监视您的文件，并在您更改这些文件时重建应用程序。

    --open（或简写 -o）选项会自动在您的浏览器中打开 http://localhost:4200/。 如果您的安装和设置成功，您应该会看到一个与下图类似的页面。

  4. 最后一步

    这就是所有的文档组件了！现在：

    开始 编写 出色的 文档！
