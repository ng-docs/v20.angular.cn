自定义构建管道

edit

在构建 Angular 应用时，我们强烈建议你使用 Angular CLI，以利用其依赖于结构的更新功能和构建系统抽象。 这样，你的项目就可以从最新的安全性、性能和 API 改进以及透明的构建改进中获益。

本页探讨了当你需要不使用 Angular CLI 的自定义构建管道时的罕见用例。 下面列出的所有工具都是由 Angular 社区成员维护的开源构建插件。 要了解有关其支持模式和维护状态的更多信息，请查看其文档和 GitHub 仓库 URL。

何时应该使用自定义构建管道？

在某些特定的用例中，你可能希望维护自定义构建管道。 例如：

  • 你有一个使用不同工具链的现有应用，并且你想向其中添加 Angular
  • 你与 module federation 紧密耦合，并且无法采用与捆绑器无关的 native federation
  • 你想使用你最喜欢的构建工具创建一个短期的实验

有哪些选项？

目前，有两个得到良好支持的社区工具，使你能够使用 Vite 插件 和 Rspack 插件 创建自定义构建管道。 它们都使用为 Angular CLI 提供支持的底层抽象。 它们允许你创建灵活的构建管道，并且需要手动维护，没有自动更新体验。

Rspack

Rspack 是一个基于 Rust 的捆绑器，旨在提供与 webpack 插件生态系统的兼容性。

如果你的项目与 webpack 生态系统紧密耦合，并且严重依赖自定义 webpack 配置，则可以利用 Rspack 来缩短构建时间。

你可以在该项目的文档网站上找到有关 Angular Rspack 的更多信息。

Vite

Vite 是一种前端构建工具，旨在为现代 Web 项目提供更快、更精简的开发体验。 Vite 还可通过其插件系统进行扩展，该系统允许生态系统构建与 Vite 的集成，例如用于单元和浏览器测试的 Vitest、用于隔离创作组件的 Storybook 等。 Angular CLI 也使用 Vite 作为其开发服务器。

AnalogJS Vite plugin for Angular 使 Angular 能够与使用 Vite 或构建于 Vite 之上的项目或框架一起使用。 这可以包括直接使用 Vite 开发和构建 Angular 项目，或将 Angular 添加到现有项目或管道。 一个示例是将 Angular UI 组件集成到使用 Astro and Starlight 的文档网站中。

你可以通过其文档页面了解有关 AnalogJS 以及如何使用该插件的更多信息。
