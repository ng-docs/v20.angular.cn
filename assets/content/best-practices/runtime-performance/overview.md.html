运行时性能优化

edit

快速渲染对于 Angular 至关重要，我们在构建此框架时就考虑了很多优化，以帮助你开发高性能的应用。为了更好地理解你的应用的性能，我们提供了 Angular DevTools 和一个关于如何使用 Chrome DevTools 进行性能分析的 视频指南。在本节中，我们将介绍最常见的性能优化技术。

变更检测是 Angular 用来检查你的应用状态是否已更改以及是否需要更新任何 DOM 的过程。从高层次来看，Angular 会从上到下遍历你的组件，以查找变更。Angular 会定期运行其变更检测机制，以便将数据模型的更改反映在应用的视图中。可以手动或通过异步事件（例如，用户交互或 XMLHttpRequest 完成）触发变更检测。

变更检测经过了高度优化且性能良好，但如果应用运行得过于频繁，仍然可能导致速度减慢。

在本指南中，你将学习如何通过跳过应用的某些部分并在必要时才运行变更检测来控制和优化变更检测机制。
