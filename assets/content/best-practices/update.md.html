让你的 Angular 项目保持更新

edit

正如 Web 和整个 Web 生态系统一样，Angular 也在持续改进。Angular 在持续改进的同时，也非常注重稳定性，并力求让更新变得简单直接。让你的 Angular 应用保持更新，就能让你享用前沿的新特性，以及各种优化和缺陷修复。

本文档包含一些信息和资源，以帮助你让你的 Angular 应用和库保持更新。

有关我们的版本控制策略和实践（包括支持和弃用实践以及发布计划）的信息，请参阅 Angular 版本控制和发布。

有帮助：如果你当前使用的是 AngularJS，请参阅 从 AngularJS 升级。 AngularJS 是指 Angular 的所有 v1.x 版本。

获取新版本发布的通知

要获取新版本发布通知，请在 X（原 Twitter）上关注 @angular 或订阅 Angular 博客。

了解新特性

有什么新鲜事？有什么变化？我们在 Angular 博客的 版本发布公告 中分享了你需要了解的最重要信息。

要查看按版本组织的完整更改列表，请参阅 Angular 更改日志。

检查你的 Angular 版本

要检查你的应用的 Angular 版本，请在你的项目目录中使用 ng version 命令。

查找 Angular 的当前版本

Angular 最新的稳定发布版本显示在 npm 的“版本”下。例如，16.2.4。

你还可以使用 CLI 命令 ng update 找到 Angular 的最新版本。 默认情况下，ng update（不带附加参数）会列出可用于你的更新。

更新你的环境和应用

为了使更新变得简单，我们在交互式的 Angular 更新指南 中提供了完整的说明。

Angular 更新指南根据你指定的当前版本和目标版本提供自定义的更新说明。它包括基本和高级更新路径，以匹配你的应用的复杂性。它还包括故障排除信息和任何推荐的手动更改，以帮助你充分利用新版本。

对于简单的更新，你只需要 CLI 命令 ng update。不带附加参数，ng update 会列出可用于你的更新，并提供将你的应用更新到最新版本的推荐步骤。

Angular 版本控制和发布 描述了你可以根据发布版本号预期的更改级别。 它还描述了支持的更新路径。
