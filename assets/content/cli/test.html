CLI

test

命令
ng test
ng t

接受项目名称，该名称在 angular.json 工作区配置文件的 projects 部分中指定。 当未提供项目名称时，它将为所有项目执行。

参数

project

要构建的项目的名称。可以是应用或库。

值类型string

选项

aot

使用预先编译运行测试。

值类型boolean默认值false
browsers

覆盖针对哪些浏览器运行测试。设置为 false 以不使用任何浏览器。

值类型string
code-coverage

输出代码覆盖率报告。

值类型boolean默认值false
code-coverage-exclude

从代码覆盖率中排除的 Globs。

值类型array
configuration
别名c

一个或多个命名的构建器配置，以逗号分隔的列表形式在 angular.json 的 “configurations” 部分中指定。 构建器使用命名的配置来运行给定的目标。 更多信息，请参阅 https://angular.dev/reference/configs/workspace-config#alternate-build-configurations。

值类型string
define

定义全局标识符，当在任何 JavaScript 或 TypeScript 代码（包括库）中找到它们时，它们将被指定的常量值替换。该值将直接使用。字符串值必须加引号。Angular 元数据中的标识符（如组件装饰器）将不会被替换。

值类型array
exclude

要排除的文件 Globs，相对于项目根目录。

值类型array
external-dependencies

排除列出的外部依赖项，使其不被捆绑到捆绑包中。相反，创建的捆绑包依赖于这些依赖项在运行时可用。

值类型array
help

在控制台中显示此命令的帮助消息。

值类型boolean
include

要包含的文件 Globs，相对于项目根目录。 有 2 种特殊情况：

  • 当提供目录路径时，将包含所有以 “.spec.@(ts|tsx)” 结尾的 spec 文件
  • 当提供文件路径，并且存在匹配的 spec 文件时，将改为包含该文件。
值类型array默认值**/*.spec.ts
inline-style-language

用于应用的内联组件样式的样式表语言。

值类型string允许值css, less, sass, scss默认值css
karma-config

Karma 配置文件的名称。

值类型string
main

主入口点文件的名称。

值类型string
poll

启用并定义文件监视轮询时间周期，以毫秒为单位。

值类型number
polyfills

要在构建中包含的 polyfills 列表。可以是文件的完整路径，相对于当前工作区或模块说明符。例如：'zone.js'。

值类型array
preserve-symlinks

解析模块时不要使用真实路径。如果未设置，则当 NodeJS 选项 --preserve-symlinks 设置时，将默认为 true。

值类型boolean
progress

在构建时将进度记录到控制台。

值类型boolean默认值true
reporters

要使用的 Karma 报告器。直接传递给 karma 运行器。

值类型array
source-map

为脚本和样式输出源码映射。更多信息，请参阅 https://angular.dev/reference/configs/workspace-config#source-map-configuration。

值类型boolean默认值true
ts-config

TypeScript 配置文件的名称。

值类型string
watch

源文件更改时重新运行测试。

值类型boolean默认值true
web-worker-ts-config

Web Worker 模块的 TypeScript 配置。

值类型string
