CLI

update

命令
ng update

通过运行以下命令，对核心框架和 CLI 的当前稳定版本执行基本更新。

ng update @angular/cli @angular/core

要更新到下一个 Beta 版或预发布版本，请使用 --next 选项。

要从一个主版本更新到另一个主版本，请使用以下格式

ng update @angular/cli@^<major_version> @angular/core@^<major_version>

我们建议你始终更新到最新的补丁版本，因为它包含自初始主版本发布以来我们发布的所有修复程序。 例如，使用以下命令获取最新的 10.x.x 版本并使用它进行更新。

ng update @angular/cli@^10 @angular/core@^10

有关更新应用程序的详细信息和指南，请参阅交互式 Angular 更新指南。

参数

packages

要更新的软件包的名称。

值类型array默认值

选项

allow-dirty

当存储库包含已修改或未跟踪的文件时，是否允许更新。

值类型boolean默认值false
create-commits
别名C

为更新和迁移创建源码控制提交。

值类型boolean默认值false
force

忽略对等依赖版本不匹配。

值类型boolean默认值false
from

要从中迁移的版本。仅在更新单个软件包时可用，并且仅在使用“migrate-only”时可用。

值类型string
help

在控制台中显示此命令的帮助消息。

值类型boolean
migrate-only

仅执行迁移，不更新已安装的版本。

值类型boolean
name

要运行的迁移的名称。仅在更新单个软件包时可用。

值类型string
next

使用预发布版本，包括 Beta 版和 RCs。

值类型boolean默认值false
to

应用迁移的版本上限。仅在更新单个软件包时可用，并且仅在使用“migrate-only”选项时可用。需要指定“from”。默认为检测到的已安装版本。

值类型string
verbose

显示有关执行期间内部操作的其他详细信息。

值类型boolean默认值false
