completion

命令
ng completion

建立自动补全功能会配置你的终端，这样在输入时，按下 <TAB> 键就会显示可用的各种命令和选项。这让你无需死记硬背就能轻松发现和使用 CLI 命令。

自动建立

首次使用 CLI 时（v14+），CLI 应该会提示并询问是否为你建立自动补全功能。只需回答“是”，CLI 就会处理剩下的事情。

如果你之前拒绝了提示，它就不会再次询问。但是你可以运行 ng completion 来自动执行相同的操作。

这将修改你的终端环境以加载 Angular CLI 自动补全功能，但无法更新你当前的终端会话。可以重启终端，或者直接运行 source <(ng completion script) 以在你当前的会话中启用自动补全功能。

手动建立

有些用户可能具有高度自定义的终端设置，其配置文件可能会以固定的结构检入到源码控制中。ng completion 始终只会将 Angular 的设置附加到当前 Shell 的现有配置文件中，或者在不存在配置文件时创建一个。如果你想更精确地控制此配置文件的位置，你可以通过让你的 Shell 在启动时运行以下命令来手动建立它：

source <(ng completion script)

这等效于 ng completion 将自动建立的内容，并在需要时为高级用户提供更高的环境灵活性。

平台支持

Angular CLI 支持 MacOS 和 Linux 操作系统上的 Bash 和 Zsh shell 的自动补全。在 Windows 上，支持 Git Bash 和使用 Bash 或 Zsh 的 Windows Subsystem for Linux。

全局安装

自动补全的工作原理是配置你的终端，以便在启动时调用 Angular CLI 来加载建立脚本。这意味着终端必须能够找到并执行 Angular CLI，通常是通过全局安装将二进制文件放在用户的 $PATH 中。如果你收到 command not found: ng，请确保 CLI 已全局安装，你可以使用 -g 标志来完成全局安装：

npm install -g @angular/cli
