CLI

new

命令
ng new
ng n

创建并初始化一个新的 Angular 应用，该应用是新工作区的默认项目。

为可选配置提供交互式提示，例如添加路由支持。 所有提示都可以安全地使用默认值。

  • 新的工作区文件夹被赋予指定的项目名称，并在顶层包含配置文件。

  • 默认情况下，新的初始应用（与工作区同名）的文件放在 src/ 子文件夹中。

  • 新应用的配置会出现在 angular.json 工作区配置文件的 projects 部分中，位于其项目名称下。

  • 你在工作区中生成的后续应用都位于 projects/ 子文件夹中。

如果你计划在工作区中拥有多个应用，你可以使用 --no-create-application 选项创建一个空工作区。 然后，你可以使用 ng generate application 创建一个初始应用。 这允许工作区名称与初始应用名称不同，并确保所有应用都位于 /projects 子文件夹中，与配置文件的结构匹配。

参数

name

新工作区和初始项目的名称。此名称将用于根目录和整个项目中的各种标识符。

值类型string

选项

ai-config

指定为哪些 AI 工具生成配置文件。这些文件通过遵循最佳实践来改进 AI 工具的输出。

值类型数组允许值claude, copilot, cursor, gemini, jetbrains, none, windsurf
collection
别名c

用于生成初始应用的原理图集合。

值类型string
commit

为新仓库配置初始 Git 提交。

值类型boolean默认值true
create-application

在新工作区中创建一个新的初始应用项目。如果为 false，则创建一个没有初始应用的空工作区。然后，你可以使用 ng generate application 命令在 projects 目录中创建应用。

值类型boolean默认值true
defaults

禁用具有默认选项的交互式输入提示。

值类型boolean默认值false
directory

应在其中创建新工作区和项目的目录。如果未指定，则将在当前目录中创建工作区。

值类型string
dry-run
别名d

运行并报告活动，但不写出结果。

值类型boolean默认值false
force

强制覆盖现有文件。

值类型boolean默认值false
help

在控制台中显示此命令的帮助信息。

值类型boolean
inline-style
别名s

将初始应用的根组件的样式直接包含在 app.component.ts 文件中。默认情况下，会创建一个单独的样式表文件（例如 app.component.css）。

值类型boolean
inline-template
别名t

将初始应用的根组件的 HTML 模板直接包含在 app.component.ts 文件中。默认情况下，会创建一个单独的模板文件（例如 app.component.html）。

值类型boolean
interactive

启用交互式输入提示。

值类型boolean默认值true
minimal

生成一个不带任何测试框架的最小 Angular 工作区。这旨在用于学习目的和简单的实验，而不是用于生产应用。

值类型boolean默认值false
new-project-root

将在工作区内创建新项目的路径，相对于工作区根目录。默认情况下，新项目在 projects 目录中创建。

值类型string默认值projects
package-manager

用于安装依赖包的包管理器。

值类型字符串允许值bun, cnpm, npm, pnpm, yarn
prefix
别名p

应用于初始项目的生成的选择器的前缀。例如，如果前缀是 my-app 并且你生成了一个名为 my-component 的组件，则选择器将是 my-app-my-component。

值类型string默认值app
routing

在初始应用项目中启用路由。这将建立必要的文件和模块，以管理应用中不同视图之间的导航。

值类型boolean
skip-git
别名g

不要在新工作区中初始化 Git 仓库。默认情况下，会初始化 Git 仓库以帮助你跟踪对项目的更改。

值类型boolean默认值false
skip-install

跳过自动安装软件包。你将需要在以后手动安装依赖项。

值类型boolean默认值false
skip-tests
别名S

跳过单元测试文件 spec.ts 的生成。

值类型boolean默认值false
ssr

为服务端渲染 (SSR) 和静态站点生成 (SSG/预渲染) 配置初始应用。

值类型boolean
standalone

基于独立 API 创建一个应用，无需 NgModule。

值类型boolean默认值true
strict

启用更严格的类型检查和更严格的捆绑包预算设置。此设置有助于提高可维护性并预先捕获错误。有关更多信息，请参阅 https://angular.dev/tools/cli/template-typecheck#strict-mode

值类型boolean默认值true
style

要在初始项目中为组件创建的样式表文件的类型。

值类型string允许的值css, less, sass, scss
view-encapsulation

为初始项目中的组件设置视图封装模式。这决定了组件样式的范围和应用方式。选项包括：Emulated（默认值，样式限定于组件）、None（样式是全局的）和 ShadowDom（样式使用 Shadow DOM 封装）。

值类型string允许的值Emulated, None, ShadowDom
zoneless

创建一个不使用 zone.js 的初始应用。

值类型boolean
