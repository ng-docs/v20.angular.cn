analytics

命令
ng analytics

你可以允许 Angular 团队向 Google 发送命令行工具的使用统计数据，以帮助他们确定特性和改进的优先级。 除非你明确选择加入，否则 Angular 团队不会收集使用情况统计信息。安装 Angular CLI 时，系统会提示你是否允许全局收集使用情况统计信息。 如果你选择“否”或跳过提示，则不会收集任何数据。

收集哪些内容？

使用情况分析包括每次执行的命令和选定的标志。 使用情况分析可能包括以下信息：

  • 你的操作系统（macOS、Linux 发行版、Windows）及其版本。
  • 包管理器名称和版本（仅限本地版本）。
  • Node.js 版本（仅限本地版本）。
  • Angular CLI 版本（仅限本地版本）。
  • 运行的命令名称。
  • 工作区信息，应用和库项目的数量。
  • 对于原理图命令（add、generate 和 new），原理图集合和名称以及选定的标志列表。
  • 对于构建命令（build、serve），构建器名称、捆绑包（初始和惰性加载）的数量和大小、编译单元、构建和重建所花费的时间以及基本的 Angular 特定 API 使用情况。

仅报告 Angular 拥有和开发的原理图和构建器。 第三方原理图和构建器不会向 Angular 团队发送数据。
