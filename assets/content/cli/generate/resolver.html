CLI

生成解析器

命令
ng generate resolver
ng generate r

在你的项目中创建一个新的解析器。解析器用于在路由激活之前预先获取数据，以确保在组件显示之前必要的数据是可用的。这可以通过防止延迟和加载状态来改善用户体验。此原理图使用指定的名称和选项生成一个新的解析器。

参数

name

新解析器的名称。这将用于创建解析器的类和 spec 文件（例如，my-resolver.resolver.ts 和 my-resolver.resolver.spec.ts）。

值类型string

选项

flat

在当前项目的顶层创建新的解析器文件。如果设置为 false，将创建一个以解析器名称命名的新文件夹来包含这些文件。

值类型boolean默认值true
functional

将解析器创建为函数 ResolveFn 而不是类。对于基本场景，函数式解析器可能更简单。

值类型boolean默认值true
project

要在其中创建解析器的项目的名称。如果未指定，CLI 将从当前目录确定项目。

值类型string
skip-tests

跳过为新解析器生成单元测试文件 spec.ts。

值类型boolean默认值false
type-separator

在生成的文件名中使用在类型之前的分隔符字符。例如，如果将选项设置为 .，则文件名将为 example.resolver.ts。

值类型string允许的值-, .默认值-
