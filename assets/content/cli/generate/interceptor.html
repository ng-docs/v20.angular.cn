CLI

generate interceptor

命令
ng generate interceptor

在你的项目中创建一个新的拦截器。拦截器用于在 HTTP 请求和响应到达其目的地之前对其进行拦截和修改。这使你可以执行诸如添加身份验证标头、处理错误或记录请求之类的任务。此原理图会为新的拦截器生成必要的文件和样板代码。

参数

name

新拦截器的名称。这将用于创建拦截器的类和 spec 文件（例如，my-interceptor.interceptor.ts 和 my-interceptor.interceptor.spec.ts）。

值类型string

选项

flat

在当前项目的顶层创建新的拦截器文件。如果设置为 false，将创建一个以拦截器名称命名的新文件夹来包含这些文件。

值类型boolean默认值true
functional

将拦截器创建为一个 HttpInterceptorFn 函数而不是一个类。对于基本场景，函数式拦截器可能更简单。

值类型boolean默认值true
project

要在其中创建拦截器的项目的名称。如果未指定，CLI 将从当前目录确定项目。

值类型string
skip-tests

跳过为新拦截器生成单元测试文件 spec.ts。

值类型boolean默认值false
type-separator

用于生成文件名称中类型之前的分隔符字符。例如，如果将选项设置为 .，则文件将命名为 example.interceptor.ts。

值类型string允许的值-, .默认值-
