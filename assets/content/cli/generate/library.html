CLI

生成库

命令
ng generate library
ng generate lib

在你的 Angular 工作区中创建一个新的库项目。库是可复用的组件、服务和其他 Angular 工件的集合，可以在多个应用之间共享。这个原理图简化了使用必要的文件和配置生成新库的过程。

参数

name

新库的名称。此名称将用于项目目录和库代码中的各种标识符。

值类型string

选项

entry-file

库的公共 API 文件的路径，相对于工作区根目录。此文件定义了库的哪些部分可以被导入它的应用访问。

值类型string默认值public-api
prefix
别名p

要添加到此库中生成的组件的选择器上的前缀。例如，如果前缀是 my-lib 并且你生成了一个名为 my-component 的组件，则选择器将是 my-lib-my-component。

值类型string默认值lib
project-root

新库的根目录，相对于工作区根目录。如果未指定，则将在 projects 目录中的子文件夹中使用库的名称创建库。

值类型string
skip-install

跳过自动安装软件包。你将需要稍后手动安装依赖项。

值类型boolean默认值false
skip-package-json

不要自动将依赖项添加到 package.json 文件中。

值类型boolean默认值false
skip-ts-config

不要更新工作区 tsconfig.json 文件以添加新库的路径映射。路径映射是在应用中使用库所必需的，但可以在此处禁用以简化开发。

值类型boolean默认值false
standalone

创建一个利用独立 API 的库，从而无需 NgModule。这可以简化库的结构及其在应用中的用法。

值类型boolean默认值true
