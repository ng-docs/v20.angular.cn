CLI

generate guard

命令
ng generate guard
ng generate g

在你的项目中创建一个新的路由守卫。路由守卫用于在路由激活之前检查某些条件，以控制对应用某些部分的访问。本原理图使用指定的名称、类型和选项生成一个新的守卫。

参数

name

新路由守卫的名称。这将用于创建守卫的类文件和 spec 文件（例如，my-guard.guard.ts 和 my-guard.guard.spec.ts）。

值类型string

选项

flat

在当前项目的顶层创建新的守卫文件。如果设置为 false，将创建一个以守卫名称命名的新文件夹来包含这些文件。

值类型boolean默认值true
functional

将守卫生成为函数而不是类。对于基本场景，函数式守卫可能更简单。

值类型boolean默认值true
implements
别名guardType

指定要创建的守卫类型。你可以选择以下一个或多个：CanActivate（控制对路由的访问）、CanActivateChild（控制对子路由的访问）、CanDeactivate（在离开路由之前请求确认）、CanMatch（确定是否可以匹配路由）。

值类型array允许值CanActivate、CanActivateChild、CanDeactivate、CanMatch默认值CanActivate
project

要在其中创建守卫的项目名称。如果未指定，CLI 将从当前目录确定项目。

值类型string
skip-tests

跳过为新守卫生成单元测试文件 spec.ts。

值类型boolean默认值false
type-separator

用于在生成的文件名中类型之前的分隔符字符。例如，如果将选项设置为 .，则文件名将是 example.guard.ts。

值类型string允许值-, .默认值-
