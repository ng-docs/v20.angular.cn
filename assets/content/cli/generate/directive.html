CLI

生成指令

命令
ng generate directive
ng generate d

在你的项目中创建一个新的指令。指令用于扩展 HTML 元素和组件的行为或外观。它们允许你操纵 DOM，添加自定义属性（Attribute），以及响应事件。这个原理图为新的指令生成必要的文件和样板代码。

参数

name

新指令的名称。这将用于创建指令的类文件和 spec 文件（例如，my-directive.directive.ts 和 my-directive.directive.spec.ts）。

值类型string

选项

export

自动从指定的 NgModule 导出该指令，使其可用于应用中的其他模块。

值类型boolean默认值false
flat

在当前项目的顶层创建新的指令文件。如果设置为 false，将创建一个以指令名称命名的新文件夹来包含这些文件。

值类型boolean默认值true
module
别名m

指定应该在其中声明指令的 NgModule。如果未提供，CLI 将尝试在指令的路径中找到最近的 NgModule。

值类型string
prefix
别名p

要添加到指令选择器中的前缀。例如，如果前缀是 app 且指令名称是 highlight，则选择器将是 appHighlight。

值类型string
project

应该在其中添加指令的项目名称。如果未指定，CLI 将从当前目录确定项目。

值类型string
selector

用于此指令的 HTML 选择器。如果未提供，将根据指令的名称生成一个选择器（例如，appHighlight）。

值类型string
skip-import

不要自动将新的指令导入到其最近的 NgModule 中。

值类型boolean默认值false
skip-tests

跳过为新指令生成单元测试文件 spec.ts。

值类型boolean默认值false
standalone

生成一个独立的指令。独立指令是自包含的，不需要在 NgModule 中声明。它们可以独立使用，也可以直接导入到其他独立组件或指令中。

值类型boolean默认值true
type

将自定义类型附加到指令的文件名。例如，如果将类型设置为 directive，则文件名将是 example.directive.ts。

值类型string
