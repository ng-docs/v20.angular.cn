CLI

生成服务

命令
ng generate service
ng generate s

在你的项目中创建一个新的服务。服务用于封装可复用的逻辑，例如数据访问、API 调用或工具函数。这个原理图简化了生成包含必要文件和样板代码的新服务的流程。

参数

name

新服务的名称。这将用于创建服务的类文件和 spec 文件（例如，my-service.service.ts 和 my-service.service.spec.ts）。

值类型string

选项

flat

在项目顶层或给定路径下创建文件。如果设置为 false，将创建一个以服务名称命名的新文件夹来包含这些文件。

值类型boolean默认值true
project

应该添加服务的项目名称。如果未指定，CLI 将从当前目录确定项目。

值类型string
skip-tests

跳过为服务生成单元测试文件 spec.ts。

值类型boolean默认值false
type

将自定义类型附加到服务的文件名。例如，如果将类型设置为 service，则文件名将为 my-service.service.ts。

值类型string
