CLI

生成组件

命令
ng generate component
ng generate c

创建一个新的 Angular 组件。组件是 Angular 应用的基本构建块。每个组件都由一个 TypeScript 类、一个 HTML 模板和一个可选的 CSS 样式表组成。使用此原理图在你的项目中生成一个新组件。

参数

name

新组件的名称。这将用于创建组件的类、模板和样式表文件。例如，如果提供 my-component，文件将命名为 my-component.ts、my-component.html 和 my-component.css。

值类型string

选项

change-detection
别名c

配置组件的变更检测策略。

值类型string允许值Default, OnPush默认值Default
display-block
别名b

将 :host { display: block; } 添加到组件的样式表，确保组件渲染为块级元素。这对于布局很有用。

值类型boolean默认值false
export

自动从指定的 NgModule 导出组件，使其可供应用中的其他模块访问。

值类型boolean默认值false
export-default

在组件的 TypeScript 文件中使用默认导出而不是命名导出。

值类型boolean默认值false
flat

直接在项目的 src/app 目录中创建组件文件，而不是为它们创建一个新文件夹。

值类型boolean默认值false
inline-style
别名s

将组件的样式直接包含在 component.ts 文件中。默认情况下，会创建一个单独的样式表文件（例如 my-component.css）。

值类型boolean默认值false
inline-template
别名t

将组件的 HTML 模板直接包含在 component.ts 文件中。默认情况下，会创建一个单独的模板文件（例如 my-component.html）。

值类型boolean默认值false
module
别名m

指定应该在其中声明组件的 NgModule。如果未提供，CLI 将尝试在组件的路径中找到最近的 NgModule。

值类型string
ng-html

生成组件模板文件时使用 '.ng.html' 文件扩展名，而不是 '.html'。

值类型boolean默认值false
prefix
别名p

要添加到组件选择器的前缀。例如，如果前缀是 app 且组件名称是 my-component，则选择器将是 app-my-component。

值类型string
project

应在其中添加组件的项目的名称。如果未指定，CLI 将从当前目录确定项目。

值类型string
selector

用于此组件的 HTML 选择器。如果未提供，将根据组件名称生成一个选择器（例如，app-my-component）。

值类型string
skip-import

不要自动将新组件导入到其最近的 NgModule 中。

值类型boolean默认值false
skip-selector

跳过为组件生成 HTML 选择器。

值类型boolean默认值false
skip-tests

跳过生成单元测试文件 spec.ts。

值类型boolean默认值false
standalone

生成一个独立组件。独立组件是自包含的，不需要在 NgModule 中声明。它们可以独立使用，也可以直接导入到其他独立组件中。

值类型boolean默认值true
style

指定要为组件创建的样式表类型，或者 none 以跳过创建样式表。

值类型string允许值css, less, none, sass, scss默认值css
type

将自定义类型附加到组件的文件名。例如，如果将类型设置为 container，则文件将命名为 my-component.container.ts。

值类型string
view-encapsulation
别名v

设置组件的视图封装模式。这决定了组件样式的范围和应用方式。

值类型string允许值Emulated, None, ShadowDom
