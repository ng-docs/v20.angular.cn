CLI

生成管道

命令
ng generate pipe
ng generate p

在你的项目中创建一个新的管道。管道用于转换数据，以便在模板中显示。它们接受输入值并应用特定的转换，例如格式化日期、货币或过滤数组。此原理图会为新管道生成必要的文件和样板代码。

参数

name

新管道的名称。这将用于创建管道的类文件和 spec 文件（例如，my-pipe.pipe.ts 和 my-pipe.pipe.spec.ts）。

值类型string

选项

export

自动从指定的 NgModule 导出此管道，使其可用于应用中的其他模块。

值类型boolean默认值false
flat

在当前项目的顶层创建新的管道文件。如果设置为 false，将创建一个以管道名称命名的新文件夹来包含这些文件。

值类型boolean默认值true
module
别名m

指定应该在其中声明管道的 NgModule。如果未提供，CLI 将尝试在管道的路径中查找最近的 NgModule。

值类型string
project

要在其中创建管道的项目的名称。如果未指定，CLI 将从当前目录确定项目。

值类型string
skip-import

不要自动将新管道导入到其最近的 NgModule 中。

值类型boolean默认值false
skip-tests

阻止为新管道生成单元测试文件 spec.ts。

值类型boolean默认值false
standalone

生成一个独立的管道。独立管道是自包含的，不需要在 NgModule 中声明。它们可以独立使用，也可以直接导入到其他独立的组件、指令或管道中。

值类型boolean默认值true
type-separator

用于生成的文件名中类型之前的分隔符字符。例如，如果将选项设置为 .，则文件名将是 example.pipe.ts。

值类型string允许值-, .默认值-
