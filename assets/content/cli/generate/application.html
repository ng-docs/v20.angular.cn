CLI

生成应用

命令
ng generate application
ng generate app

在你的工作区内生成一个新的 Angular 应用。本原理图会建立你项目的基本结构，包括根组件、模块和配置文件。你可以自定义应用的各个方面，例如路由、样式和测试。

参数

name

新应用的名称。此名称将用于项目目录以及整个应用代码中的各种标识符。

值类型string

选项

inline-style
别名s

将根组件的样式直接包含在 app.component.ts 文件中。只有 CSS 样式可以内联包含。默认情况下，会创建一个单独的样式表文件（例如 app.component.css）。

值类型boolean
inline-template
别名t

将根组件的 HTML 模板直接包含在 app.component.ts 文件中。默认情况下，会创建一个单独的模板文件（例如 app.component.html）。

值类型boolean
minimal

生成一个不带任何测试框架的最小化项目。这适用于学习目的和简单的实验，不适用于生产应用。

值类型boolean默认值false
prefix
别名p

要添加到此应用中生成的组件的选择器上的前缀。例如，如果前缀是 my-app 并且你生成了一个名为 my-component 的组件，则选择器将是 my-app-my-component。

值类型string默认值app
project-root

相对于工作区根目录，将在其中创建新应用文件的目录。如果未指定，则将在 projects 目录中的子文件夹中使用应用的名称创建应用。

值类型string
routing

生成一个已配置路由的应用。这将建立必要的文件和模块，以管理应用中不同视图之间的导航。

值类型boolean默认值true
skip-install

跳过自动安装软件包。你将需要稍后手动安装依赖项。

值类型boolean默认值false
skip-package-json

不要将依赖项添加到 package.json 文件中。

值类型boolean默认值false
skip-tests
别名S

跳过生成单元测试文件 spec.ts。

值类型boolean默认值false
ssr

为服务器端渲染 (SSR) 和静态站点生成 (SSG/预渲染) 配置应用。

值类型boolean默认值false
standalone

创建一个利用独立 API 的应用，从而无需 NgModule。这可以简化你的应用结构。

值类型boolean默认值true
strict

为应用启用更严格的捆绑包预算设置。这有助于保持你的应用捆绑包大小较小并提高性能。有关更多信息，请参阅 https://angular.dev/tools/cli/template-typecheck#strict-mode

值类型boolean默认值true
style

要为应用中的组件创建的样式表文件的类型。

值类型string允许值css, less, sass, scss默认值css
view-encapsulation

设置应用组件的视图封装模式。这将决定组件样式的范围和应用方式。

值类型string允许值Emulated, None, ShadowDom
zoneless

生成一个不使用 zone.js 的应用。

值类型boolean默认值false
