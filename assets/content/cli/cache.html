CLI

cache

命令
ng cache

默认情况下，Angular CLI 会在磁盘上保存许多可缓存的操作。

当你重新运行相同的构建时，构建体系会恢复先前构建的状态，并复用先前执行的操作，从而减少构建和测试你的应用和库所需的时间。

要修改默认缓存设置，请将 cli.cache 对象添加到你的工作区配置中。 该对象位于文件顶层 cli.cache 下，在 projects 部分之外。

{  "$schema": "./node_modules/@angular/cli/lib/config/schema.json",  "version": 1,  "cli": {    "cache": {      // ...    },  },  "projects": {},}

欲知详情，请参阅缓存选项。

缓存环境

默认情况下，磁盘缓存仅对本地环境启用。环境的值可以是以下之一：

  • all - 允许在所有机器上使用磁盘缓存。
  • local - 仅允许在开发机器上使用磁盘缓存。
  • ci - 仅允许在持续集成 (CI) 系统上使用磁盘缓存。

要将环境设置更改为 all，请运行以下命令：

ng config cli.cache.environment all

欲知详情，请参阅缓存选项中的 environment。

Angular CLI 检查 CI 环境变量的存在与否及其值，以确定它在哪个环境中运行。

缓存路径

默认情况下，.angular/cache 用作存储缓存结果的基本目录。

要将此路径更改为 .cache/ng，请运行以下命令：

ng config cli.cache.path ".cache/ng"

子命令

此命令具有以下子命令

  • clean
  • disable
  • enable
  • info

选项

help

在控制台中显示此命令的帮助消息。

值类型boolean
